## 🎯 1. 비즈니스 모델 (One-Liner)

> **"호텔 매니저가 1회 $200을 내면, 6언어 채널에 호텔 영상이 노출되고, 6개월 동안 예약 0건이면 100% 환불해드립니다."**

이 한 문장이 모든 페이지에 녹아 있어야 합니다.

---

## 💎 2. 핵심 가치 제안 (Value Proposition)

호텔 매니저가 $200을 결제하는 이유 — **3가지 차별화 강점**:

### 강점 1: 6언어 채널 동시 노출
- 한국어 (호텔이야)
- 영어 (Kotel)
- 일본어 (ホテルだ)
- 중국어 (世界就是家)
- 스리랑카어 (Sinhala 채널)
- 베트남어 (Vietnamese 채널)

→ 다른 마케팅 회사가 따라 하기 어려운 **유일한 자산**

### 강점 2: 6개월 예약 보장 환불
- 1회 $200 결제 → 6개월 동안 예약 0건 시 **100% 환불** (세금 제외)
- 호텔이 안심하고 결제할 수 있는 강력한 명분
- 이걸로 영업 진입 장벽 대폭 낮춤

### 강점 3: 별도 이벤트 페이지 트래픽 송출
- 대표님이 직접 광고비 지출하여 호텔로 트래픽 유입
- 단순 영상 노출이 아닌 **"실제 예약 발생"** 을 위한 추가 마케팅

### 부가 가치
- **3일 내 영상 제작** → 노출 날짜 확정 통보 (이전 약속한 7일 → 3일로 단축)
- **1회 결제 영구 영상 자산** (6개월 후에도 영상은 채널에 남음)

## 💰 3. 가격 정책

### 재계약 정책 명확화 (2026-04-29 결정)
- ❌ "재계약 시 20% 할인" 같은 정책 **없음**
- ✅ 동일 가격 $200 유지
- 이유: 가격 할인이 아닌 **누적 성과로 설득**
- 메일 문구 예: "지난 6개월 X명 노출 / Y건 예약. 계속 노출을 원하시면 재계약 가능합니다."

## 🔄 4. 환불 정책 (6개월 보장)

### 환불 조건
- 결제일 기준 **6개월 동안 예약 0건** 발생 시
- **100% 환불** (세금 제외)
- 환불 신청은 매니저가 marketing.html에서 직접 가능

### 환불 제외 사항
- 세금 (PayPal 수수료 / 한국 부가세 등)
- 6개월 이전 자발적 해지 (조기 해지는 환불 없음)
- 매니저의 호텔 정보 오류로 인한 노출 실패 (매니저 책임)

### 자동화 트리거
- D-30 (결제 5개월 차): 누적 성과 메일 자동 발송
- D-Day (결제 6개월 차):
  - 예약 0건 시 → "환불 신청 안내" 메일
  - 예약 발생 시 → "재계약 권유" 메일 (할인 없음)

- 매니저가 사장에게 보고서 보냄 → 사장이 "다른 호텔도 추천해줘" → 자연 영업

## 🏨 7. 호텔 매칭 정책 (Agoda 매칭)

### 핵심 원칙 (2026-04-26 결정 — 변경 불가)
> **"가입 신청한 호텔은 100% 활성화시킨다."**
>
> 매칭 = 매출의 시작점. 매칭 실패 = 매출 0.
> 단 한 명의 매니저도 놓치지 않는다.

### 3단계 안전망 (이미 구현 완료, 라이브 작동 중)

```
[1차] 자동 매칭 (Google Places + Agoda 동시 검색)
  ✅ 성공 (예상 70~80%) → 즉시 활성화
  ❌ 실패 → 2차로

[2차] 매니저 수동 입력 (Agoda URL 백업)
  - "호텔을 못 찾으셨나요? 아고다 URL로 입력하기" 백업 필드
  ✅ 성공 (예상 10~15% 추가) → 활성화
  ❌ 실패 → 3차로

[3차] 관리자 수동 매칭 (admin.html → Agoda Matching 탭)
  ✅ 옵션 ①: hotelId 직접 입력 → 활성화
  ✅ 옵션 ②: Agoda 등록 안내 메일 자동 발송
  ✅ 옵션 ③: 1:1 컨택 (대표님이 직접 매니저에게 연락)
```

### 호텔 등급 정책
- **3성급 이상만 등록 가능** (Agoda 자동 등급 표시)
- 1~2성급 / 게스트하우스는 영상 품질 유지 위해 제외
- APA 호텔은 전면 제외 (대표님 정책)

## 📱 8. 사용자 여정 (User Journey)

### Stage 0: 비회원 진입
```
index.html (메인 랜딩)
  → 결제 버튼 클릭
  → "결제는 가입 후 가능" 모달
  → signup.html로 이동
```

### Stage 1: 가입 ~ 결제 전
```
signup.html (가입)
  → 이메일 인증
  → hotel-info.html (호텔 정보 입력)
    → Agoda 매칭 (3단계 안전망)
    → status=pending or review
  → 관리자 승인 (admin.html)
    → status=approved
  → sales.html (결제 유도) ⭐ 신설 예정
    → 가치 어필 + PayPal 결제 버튼
```

### Stage 2: 결제 후 (성과 페이지)
```
결제 완료
  → status=paid
  → marketing.html (성과 대시보드) ⭐ 신설 예정
    - 핵심 헤드라인: 조회수 / 예약 건수
    - 채널별 성과 (6언어)
    - 6개월 보장 D-Day 카운트
    - 📄 PDF 보고서 다운로드 (매니저 핵심 기능)
    - 결제 일시 + 인보이스 다운로드
    - 호텔 정보 (작게)
```

### Stage 3: 자동화 사이클
```
Day 0: 결제 완료 메일 (인보이스 첨부)
Day 0~3: 영상 제작 → 노출 날짜 확정
Day X: 영상 첫 노출 메일 (6개 채널 링크)
매월 결제일: 월간 리포트 메일 + PDF 첨부 ⭐ (주간 → 월간으로 변경)
D-30 (5개월 차): 누적 성과 + 재계약 안내 메일
D-Day (6개월 차):
  - 예약 0건: 환불 신청 안내
  - 예약 발생: 재계약 권유 (동일 가격 $200)
```

## 📌 15. 의사결정 원칙 (Decision Principles)

비즈니스 결정 시 이 우선순위를 따른다:

1. **매출 보호 우선** — 매출 발생 흐름을 막는 것은 절대 도입 X
   - 예: Agoda 매칭 실패해도 100% 활성화시킨다
2. **매니저 부담 최소화** — 매니저가 매일 안 들어와도 되게
   - 예: 주간 → 월간 리포트
3. **사장 보고 가능성** — 매니저가 사장에게 보고할 수 있는 도구 제공
   - 예: PDF 보고서
4. **글로벌 표준** — 한국 전용 기능은 도입 X
   - 예: USD 결제, 영어 우선 UI
5. **신뢰성 > 자동화** — 자동화가 깨지면 수동 처리 가능해야
   - 예: Agoda 매칭 3단계 안전망
