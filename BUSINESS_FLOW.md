# TravelWinners B2B — 사용자 여정 (Business Flow)

> **이 문서는 호텔 매니저가 가입부터 6개월 보장 종료까지 거치는 전체 여정을 시각화합니다.**
>
> 각 단계마다:
> - 사용자가 보는 페이지

> - 발송되는 알림 메일
> - status 변화
> - 관련 코드 위치
>
> 마지막 갱신: 2026-04-29

## 📍 Stage 0: 비회원 진입

### 사용자 행동
1. 호텔 매니저가 외부 채널(SNS / 검색 / 추천)로 `gohotelwinners.com` 진입
2. 메인 페이지에서 가치 제안 확인
3. "결제하기" 버튼 클릭

### 시스템 동작
- 비로그인 상태 감지 → **"결제는 가입 후 가능" 모달** 표시
- 모달에서 "회원가입" 버튼 → signup.html로 이동

### 페이지
- **index.html** (라이브)

### 향후 개선
- index.html 결제 버튼 클릭 시 모달 추가 (현재는 직접 회원가입 페이지로 안내)
- A/B 테스트로 전환율 측정

## 📍 Stage 1: 가입 ~ 호텔 등록

### 1A. 회원가입 (signup.html)

**사용자 입력**:
- 이메일 / 비밀번호
- 이름 / 휴대폰 / 직책
- 호텔명
- ☑️ 마케팅 수신 동의 (선택)

**시스템 동작**:
1. Supabase Auth → 인증 메일 자동 발송
2. profiles 테이블에 매니저 정보 저장
3. 마케팅 동의 시 → 영업 DB에 `marketing_consent=true` 저장

**발송 메일**:
- ✉️ 이메일 인증 메일 (Supabase Auth 자동)

**다음 단계**: 매니저가 메일 링크 클릭 → verify-email.html

---

### 1B. 이메일 인증 (verify-email.html)

**시스템 동작**:
- Supabase Auth 토큰 검증 → 계정 활성화
- profiles.email_confirmed = true

**다음 단계**: 자동 redirect → hotel-info.html

---

### 1C. 호텔 정보 입력 (hotel-info.html) — Agoda 3단계 안전망 ⭐

```
[1차] 자동 매칭
  매니저: 호텔명 입력
       ↓
  시스템: Google Places + Agoda 병렬 검색
       ↓
  결과 표시 (Agoda 등록 뱃지 포함)
       ↓
  매니저: 검색 결과에서 자기 호텔 선택
       ↓
  자동 채움: 주소, 사진, 평점, Agoda hotelId, 등급
       ↓
  ✅ 매칭 성공 → status = pending → Stage 2로

[2차] 자동 실패 시 수동 입력
  매니저: "호텔을 못 찾으셨나요? 아고다 URL로 입력하기" 클릭
       ↓
  매니저: Agoda 호텔 페이지 URL 붙여넣기
       ↓
  시스템: URL 파싱 → hotelId 추출
       ↓
  ✅ 매칭 성공 → status = pending
  ❌ 매칭 실패 → status = manual_pending → Stage 2 Agoda Matching 탭으로

[3차] 관리자 수동 처리 (별도 큐)
  → Stage 2에서 처리
```

**관련 파일**:
- `hotel-info.html` (UI)
- `api/agoda-search.js` (자동 매칭)
- `api/agoda-hotel.js` (Agoda 데이터 조회)

**DB 컬럼**:
- `hotels.agoda_match_status` (auto_matched / manual_pending / manual_matched / agoda_registration_pending / rejected)
- `hotels.agoda_url`
- `hotels.agoda_hotel_id`
- `hotels.agoda_city_id`

**발송 메일** (향후 추가 예정):
- ⏳ "호텔 정보 등록 완료, 검토 중입니다" (BACKLOG P1)

## 📍 Stage 2: 관리자 검증

### 2A. 정상 매칭 호텔 처리 (admin.html → Hotels 탭)

**관리자 행동**:
1. Admin Console → Hotels 탭 진입
2. status=pending 호텔 목록 확인
3. View 클릭 → 호텔 상세 정보 확인 (Agoda 링크 검증)
4. 결정:
   - ✅ 승인 → status = approved
   - ❌ 거절 → status = rejected (사유 입력)
   - 🔄 재검토 요청 → status = review

**발송 메일** (향후 추가 예정):
- ⏳ 승인 시: "호텔 승인 완료, 결제 진행하세요" + sales.html 링크 (BACKLOG P1)
- ⏳ 거절 시: "호텔 등록이 거절되었습니다 - 사유" (BACKLOG P1)

---

### 2B. 매칭 실패 호텔 처리 (admin.html → Agoda Matching 탭) ⭐

**관리자가 보는 정보**:
- 매니저 정보 (이메일 / 휴대폰)
- 입력한 호텔명 + Google Places 정보
- 자동 매칭 실패 사유

**관리자 옵션**:
1. **hotelId 직접 입력**: Agoda Partner 백오피스 또는 검색에서 hotelId 조회 후 입력
2. **Agoda 등록 안내 메일 발송**: 매니저에게 "Agoda Partner Hub 가입 가이드" 자동 발송
3. **1:1 컨택**: 대표님이 직접 매니저에게 연락

**관련 API**:
- `api/admin-send-agoda-invite.js` (Agoda 등록 안내 메일 발송)

**핵심 원칙**:
> "가입 신청한 호텔은 100% 활성화시킨다"
> 매칭 = 매출의 시작점. 단 한 명도 놓치지 않는다.

## 🔄 status 라이프사이클

```
pending          → 매니저가 호텔 정보 입력 완료, 검토 대기
review           → 관리자가 추가 검토 중
manual_pending   → Agoda 매칭 실패, 관리자 수동 처리 대기
approved         → 관리자 승인, 결제 대기
rejected         → 관리자 거절
paid             → 결제 완료
producing        → 영상 제작 중
published        → 영상 노출 중 (6개월 카운트 시작)
refundable       → 6개월 보장 만료, 환불 신청 가능 (예약 0건)
refunded         → 환불 완료
expired          → 6개월 만료, 재계약 안 함
```
