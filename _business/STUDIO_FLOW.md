# 스튜디오 서비스 로직도 (전체 흐름 한 장)

> **2026-07-17 대표님 지시**: *"지금 우리가 스튜디오에 서비스 형태 로직도를 전체 한번 정리해서
> 너가 기록하고 있어야 됨. **지금처럼 할 때마다 헷갈리면 안 됨.**"*
>
> **왜 이 문서가 생겼나** — 2026-07-17 하루에 클로드가 **확정문서를 안 읽어서 5번 틀렸다**:
> ① 지역 세부 목적을 「출구」라 함 (대표님: 기본자료 보는 곳)
> ② D-062 확정률 배지를 순위에 붙여 **거꾸로** 만듦 (100%가 전부 1.4건짜리)
> ③ 노출 열쇠를 `hotels.published_at` 으로 씀 (D-062: **`publications`**)
> ④ 가격대 축을 물음 (D-066에 이미 «코드 없음=정상» 확정)
> ⑤ 「확인 필요」를 코드 오타로만 봄 (7-E-2: **cid 어긋남 = 수수료 샘**)
> → 답이 **6개 문서에 흩어져** 있었다. **한 장으로 모은다.**
>
> 🔴 **이 문서는 지도다. 판정은 원문이 한다.** 어긋나면 원문이 맞다. 원문을 고치고 여기도 고친다.

## 0. 한 장 요약 — 소재 하나가 도는 길

```
   [키워드] 발굴                [전략] 관리              [올리기] 발행           [성과표·호텔] 회수
       │                          │                          │                        │
 나라→도시→지역               기획대기                   드라이브 대기            발행 완료
 어떤 동네·성급·언제           ↓ [원고 시작]              ↓ 로봇이 읽음            ↓
 어느 호텔                     원고작성                   코드로 매칭              성과가
       │                          ↓ 코드 매칭(자동)         ↓                        다음 기획
       └─[＋ 전략으로]──────→   발행예정  ──────────→   발행완료 ──────────→      근거로
                                  ↑                                                   │
                                  └───────────────────────────────────────────────────┘
```
**한 바퀴가 닫힌다.** 발굴 → 관리 → 발행 → 회수 → 다시 발굴.

## 1. 축 (무엇으로 가르나) — **여기서 제일 많이 틀렸다**

| 축 | 값 | 누가 정하나 | 근거 |
|---|---|---|---|
| **타겟** | 언어 + 시장 (한국어·한국 / 일본어·일본 …) | `channels.language` **자동** | D-065 |
| **나라 → 도시 → 지역** | 오사카 → 난바 | `hotels.district`(좌표봇) | D-065 |
| **성급** | 3·4·5성 — **내림**(3.5→3) | 아고다 파일 `star_rating` | 대표님 2026-07-17 |
| **유형** | 호텔·리조트 / **료칸 따로** | `hotels.property_type` | 대표님 2026-07-17 |
| ~~가격대~~ | **폐기** | — | 대표님 2026-07-17 |
| ~~채널~~ | **축 아님** → 전개면(D-067) | 원고 링크 `cid` | D-065 264줄 |
| ~~평점~~ | **폐기** | — | 대표님 2026-07-17 |

🔴 **타겟 = 언어+시장.** 채널로 쪼개면 같은 조사를 3번 한다(한국어 채널 3개) → 앵커 잣대가 깨진다.
🔴 **성급은 내림.** 3.5성을 4성이라 소개하면 실망, 3성이라 소개하면 **기대보다 좋아 예약**한다.
🔴 **아파트·호스텔·게하는 성급 축에 안 넣는다** — 콘텐츠 유형이 다르다. **장부엔 남는다.**
🔴 **료칸은 유형 축** — 「오사카 료칸」 3.1로 **그 말로 찾는 사람이 있다.** 찾는 말이 있으면 축이 된다.
🔴 **가격·평점은 바뀐다.** 우리가 안 쌓은 남의 값은 축이 못 된다. (아고다 파일 가격 = **304만 행 전부 빈칸**, 실측)

## 2. 키워드 메뉴 — 발굴

### 화면 4단계
```
s1 나라 → s2 도시 → s3 도시 상세(지역 목록) → s4 지역 세부
```
**목적 = 기본 자료 보는 곳** (대표님 2026-07-17). 여기에 큐를 두지 않는다. **출구 하나뿐**(D-065 0-H).

### 지역 세부(s4) 7블록 — 자료 → 결론 → 실행
| | 블록 | 답하는 것 | 재료 |
|---|---|---|---|
| 4-0 | 이 동네가 뭔가 | 성급으로 판정 | `v_district_star` |
| 4-1 | 성급 분포 | **뭘 만드나** | 〃 (아고다 재고 800m vs 우리 vs 예약) |
| 4-1b | 예약 패턴 **[접힘]** | **언제·어떻게** | `v_district_month` · `v_district_pattern` |
| 4-4 | 지역 검색어 | 무슨 말로 찾나 | `keyword`+`trend` (이름 포함 판정) |
| 4-4b | 호텔 매출 순위 | 돈이 어디서 나나 | `v_district_hotel` (**확정만**) |
| 4-3 | 콘텐츠 추천 | **그래서 뭘** | 위 전부 조합 → **[＋ 전략으로]** |
| 4-14 | 호텔 목록(장부) | 우리가 아는 전부 | `v_district_hotel` (**네 갈래**) |

### 🔴 두 블록은 다른 물건이다 — 섞지 마라
| | 4-4b 매출 순위 | 4-14 호텔 목록 |
|---|---|---|
| 무엇 | **돈** | **장부** |
| 취소만 있는 호텔 | **뺀다** | **넣는다** — *"예약했다가 취소했구나"* 도 정보(대표님) |
| 갈래 | `bucket='revenue'` | revenue / cancelled_only / no_booking / 아직 안 본 곳 |

### 배지 — **순위 자리와 호텔 한 곳 자리는 다르다**
| 자리 | 배지 | 근거 |
|---|---|---|
| **순위·추천(키워드)** | **인기**(확정 10건+) / **주의**(예약 10건+인데 확정<10 & 확정률<50) / 없음(표본<10) | 대표님 2026-07-17 |
| **호텔 한 곳(호텔 메뉴)** | 안정≥85 / 보통70~84 / 주의50~69 / 경고<50 | D-062 ⑤-3 |
🔴 **취소가 많은 건 인기의 부산물이다** — 소테츠 확정 23·취소 24인데 난바 매출 3등.
   실측: 확정률 **100%가 33곳인데 전부 평균 1.4건**짜리. 순위는 **규모가 먼저**다.

### 재사용 (노출 이력이 오면 켜짐 · 지금 재료 0건)
`아직못씀`(183일 안) → **`곧 가능`(3달 안 · 지금 기획하면 딱)** → `지금가능` / `유료 계약`(쿨다운 없음)

## 3. 전략 메뉴 — 관리 (D-066 · 2층)

```
┌ 방향 판 (위층) ─── 🔴 아직 없음
│ 타겟별 · 나라+성급 · 1년 뷰(착수 3~4달 크게 + 연간 미리보기)
│ [확정]/[보류]/[확정취소]   ([바꾸기] 폐기 — 근거 없는 방향 금지)
│ 누르면 → 키워드 도시 상세로 점프
└──────────────
┌ 칸반 큐 (아래층) ── ✅ 있음
│ 기획대기 → 원고작성 → 발행예정 → 발행완료
└──────────────
```

- **코드 = `TW-XXXX` · 채널 무관** (D-066). 기획은 채널을 안 정한다 — **원고의 cid가 정한다.**
- **세 역할** (D-067 §8-1): 기획자 = [＋ 전략으로] 누른 사람 자동 / 원고 담당 = 쓰는 사람(기획대기~원고작성) / **최종 업로더 = 올린 사람(발행완료, 올리기 연동)**. 단계마다 주인공이 바뀐다.
- **담당 권한** (D-067 §8-2): 담당 없음→에디터 이상 [내가 맡기] / **담당 있으면 남이 못 맡음**(이름만 표시) / **변경(재배정)=최고관리자만** / **원고작성 담당자=코드복사·분석만**(놓기 없음, 확정됨). 정석=화면+서버 둘 다 막음.
- **자체기획 발행예정** = "올린 사람" 표시 안 함 (발행완료 때 최종업로더로만, D-067 §8-3).
- **입구 2개**: 전략 방향 · 키워드. **직접 추가 폐기**
- **키워드 화면 표시**: `＋전략으로` → `전략에 있음·코드·단계·기획자` → `발행함`
  또 누르면 **막지 않고 알려준다** — *각도가 다르면 다른 영상*(대표님)
- **확인필요 ↔ 발행예정 동기화** (D-067 §8-4·8-5): **문제 원고는 전략에 안 뜬다 → 올리기 [확인필요]에서만**(추천·자체기획 공통, 채널명·파일명·사유 표시). **정상 원고만** 발행예정 ⟷ 올리기 코드로 동기화(담당·발행상태). 고쳐서 정상화 → 올리기 확인필요서 사라짐 + 발행예정 나타남.

## 4. 올리기 메뉴 — 발행 · **원고 네 갈래**

```
드라이브 (채널마다 대기·완료·확인필요 3폴더) → 로봇이 하루 4번(06·11·16·21시) 읽음
```

| 갈래 | 무엇 | 판정 | 근거 |
|---|---|---|---|
| 🟢 **추천으로 만든 것** | `TW-0001` 있고 cid도 맞음 | 카드에 붙음 · 발행예정 자동 | D-066 |
| ⚪ **그냥 만든 것** | **코드 없음** | **오류 아님.** 자체 제작 → 바로 발행 | D-066 |
| 🔴 **확인 필요 · 채널 어긋남** | **폴더 채널 ≠ 링크 cid** | **거부** | **7-E-2** |
| 🟠 **확인 필요 · 코드 오타** | 코드 있는데 맞는 카드 없음 | 확인 | D-066 |

> **확인필요 표시 (D-067 §8-4·8-7)**: 올리기 [확인필요]에 **채널명(cid 판정)·파일명·사유**. 전략엔 안 뜸. 같은 파일 고쳐 재투입(file_id 추적)=자동 갱신 / 다른 파일=채널명 보고 [지우기]. 사유 3종: 코드 오타·같은 채널 코드 중복·아고다 링크 없음.

### 🔴 7-E-2 — 채널은 아고다 링크가 정한다
> 원고에 든 아고다 파트너 링크가 채널의 **유일한 근거**다. **수수료가 그 cid 로 잡히기 때문이다.**

| 근거 | 우선순위 |
|---|---|
| 원고 링크의 `cid` | **1순위** |
| 호출 시 넘긴 `cid` | 2순위 |
| 원고 본문의 채널명 | 3순위 (링크 없을 때만) |

- 링크 3개 cid 가 서로 다르면 **거부** (수수료가 갈린다)
- 넘긴 cid ≠ 링크 cid → **거부**
- `channel_cid_map` 에 없는 cid → **경고**

> ⚠️ 2026-07-10 사고: `1931762`(2차 계정)를 채널 cid 로 붙여넣음. 이 규칙이 그 자리에서 막았다.

**왜 제일 위험한가** — 폴더는 호텔이야인데 링크 cid가 호텔이곳이면 **영상은 호텔이야에 올라가고 돈은 호텔이곳으로 간다.**

**CID 확정값**(대표님 직접 검증): 여행능력자들 유튜브 **1913282** · 여행능력자들 웹 **1919025** · 호텔이야 **1932026** · 호텔이곳 **1946819**
