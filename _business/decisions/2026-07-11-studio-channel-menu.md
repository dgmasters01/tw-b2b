# 2026-07-11 스튜디오 채널 메뉴 (UX/UI) — 채널 자산·규격 관리 마스터

> 압축본: `_os/charter/decisions-index.md` (D-064)
> 결정자: 대표님 / 정리: Claude
> 스튜디오 6메뉴 중 여섯 번째 "채널". 라이브 코드(channels·channel_cid_map·youtube.js·_lib/youtube-rules.js·_content/youtube/*.md) 검증 기반.

## 목적
채널 메뉴 = **콘텐츠를 올리는 "채널 자산"과 "채널별 규격"을 한곳에서 등록·관리하는 마스터**. 채널의 이름·언어·플랫폼·아고다 CID·활성·순서 + **자동 생성 규격**(제목·파일명·본문·키워드·나레이션)을 관리한다. 스튜디오의 올리기·성과표·호텔이 전부 이 채널 위에서 돈다.
- **경계**: 성과표 "채널별"=채널 성과 보기 / 채널 메뉴=채널 자체를 등록·설정·CID·규격 관리.

## 2층 구조
1. **채널 기본 레지스트리** — 코드·이름(한/영)·언어·플랫폼·아고다 CID(다수, old/new)·활성(is_active)·표시순서(display_order). → `channels` + `channel_cid_map`.
2. **채널 규격** — 자동 생성 규칙 §1~§5. → 문서 `_content/youtube/{채널}.md`.

## 규격 항목 (라이브 호텔이야.md v1 기준)
- §1 제목: 슬롯 순서(지역→도시→특징)·글자수(60자 이내)·제목 해시태그 없음·성급 처리(3성 이하 제목 제외).
- §2 파일명: 형식·주키워드 5~6개·확정 제목으로 덮어쓰기.
- §3 본문(더보기) 블록 순서: 요약 → 호텔블록 → 조사기준 → 챕터 → 한눈에 → 고지 → 해시태그 10개. **링크=TOP1 맨 위 / 챕터=TOP3 맨 위(전 채널 통일)**. 숏폼은 챕터 없음.
- §4 키워드(태그란 500자): 슬롯 템플릿·지역 키워드 40%.
- §5 오프닝 나레이션: 언어=채널 언어·톤.

## 구현 방식 = A안 (확정)
규격을 **화면 폼으로 편집 → 문서 저장(github-commit)**. render(youtube.js)는 지금처럼 그 문서를 읽음 → 자동 생성·전 메뉴 적용에 바로 연결. 개발 최소, 지금 작동 유지.
- **필수 개조 1건(기술, 클로드 자율)**: `_lib/youtube-rules.js`의 하드코딩 `CHANNELS=['여행능력자들','호텔이야','호텔이곳']` 배열과 `detectChannel` 하드코딩 제거 → **채널 DB 기반**으로. 채널 레지스트리를 돌며 각 채널 규격 문서를 읽고, 채널 판별도 DB 이름/CID로. 그래야 새 채널이 **코드 수정 없이** 자동 인식·생성.
- 이 개조로 2층이 이어짐: 기본(DB)=드라이버(어떤 채널이 있나), 규격(문서)=내용(어떻게 뽑나).

## 새 채널 규격 만들기 = Claude 대화로 작성 → .md 등록 → 자동 통합 (확정 · 심플화)
스튜디오 안에 위저드를 만들지 않는다. 규격을 만드는 어려운 부분은 **이미 잘 되는 Claude 대화창**이 맡는다.
1. **Claude 대화창에서 작성** — 샘플 워드 파일을 Claude 채팅에 넣고 대화로 정리(제목·본문·태그·키워드 하나씩) → 최종 `{채널}.md` 산출. (기존 3채널을 만든 그 방식 그대로)
2. **채널 메뉴에서 .md 등록** — 완성된 `{채널}.md`를 업로드. 파일 §0 채널정보(채널명·언어)와 규격 내 아고다 링크(CID)를 자동 인식.
3. **자동 통합 (등록하면 시스템이 알아서)** — ① channels 등록 + channel_cid_map에 CID 매핑 ② `_content/youtube/{채널}.md` 배치 + 활성화 ③ render(loadRules DB기반) 인식 → 원고 올리면 즉시 자동 생성.
- 채널 메뉴 = 규격 .md **등록소 + 자동 통합**. 드래프팅·정리는 Claude 대화가 담당 → 스튜디오 UI가 훨씬 단순.
- 권한: 등록 = owner/admin만, 에디터 조회만.

## 권한 (확정)
- 채널 생성·규격 저장·CID 편집 = **owner 또는 admin만**. **에디터는 조회만**(생성·저장 불가).
- **두 겹 방어**: ① UI에서 버튼 미표시(에디터·매니저) ② 서버 API에서 역할 검증(우회 호출 거부). 버튼만 숨기면 뚫리니 서버 게이트 필수.

## CID 관리
채널별 CID 추가/폐기(신규 전환 시 old 유지·new 추가). CID = 아고다 예약↔채널 연결 키(성과표·호텔의 뿌리). CID 확정값은 대표님 화면 검증 완료(여행능력자들 유튜브 1913282·웹 1919025·호텔이야 1932026·호텔이곳 1946819).

## 전파 (단일 진실 → 각 메뉴)
규격/기본을 한곳(규격 문서 + 채널 DB)에서 고치면 → render가 읽어 **올리기(복사 4카드 자동)** · **성과표(채널명·언어 표기)** · **호텔(채널별 노출·예약)** 에 자동 반영.

## 데이터·연결
- 채널 기본 = `channels`(code·name·name_en·language·platform·has_agoda_api·agoda_site_id·is_active·display_order·notes)
- CID = `channel_cid_map`(cid·channel_code·cid_label old/new·is_active)
- 규격 = `_content/youtube/{채널}.md`, render=`_lib/youtube-rules.js`(loadRules·detectChannel)·`_lib/youtube-render.js`
- 키워드 실측 = `_content/youtube/keywords`
- 시드 채널(라이브): TW 여행능력자들·HT 호텔이야·KT Kotel·TC 호텔닷컴·JP ホテルだ·ZH 世界就是家 (+ VN·HG 등 CID)
