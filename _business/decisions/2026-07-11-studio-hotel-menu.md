> 스튜디오 6메뉴 중 세 번째 "호텔".

## 목적 (재정의 — 가장 중요)
호텔 메뉴는 **호텔 하나에 대한 전체를 모아 전략적으로 판단하는 대시보드**다. 단순 재사용 관리가 아니라, "이 호텔을 콘텐츠에 어떻게(다시) 쓸지"를 노출·성과·취소·확정률을 보고 결정한다. 특히 취소 분석(예약이 언제 잡히고 단계별로 얼마나 살아남나)이 핵심.

## 목록
- 검색: 호텔명·hid
- 각 호텔: **나라·도시** 뱃지 · **성급** · **유형(호텔/리조트)** · hid · 재사용 상태(아직못씀/지금가능/노출없음)
- 필터: 나라·도시·성급·유형 드롭다운 + "지금 쓸 수 있는 것만"

## 재사용 금지 = 6개월 (확정)
- 마지막 노출일 + 6개월 = 재사용 가능일.

## 상세 (호텔 클릭 시) — 위에서 아래 순서
### 기본 정보
나라·도시·성급·유형·hid + 재사용 상태(가능일)

### ① 노출 이력 (여러 건)
한 호텔은 여러 채널·여러 번 노출된다. 각 노출 = **정확한 날짜(YYYY-MM-DD)** · 채널 · 영상 제목 · 노출 순위(TOP1/2/3). 목록으로 쌓임.

### ② 성과 요약 — 드릴다운 3단계 + 전환 퍼널
- 지표 4개: 조회수 · 클릭 · 예약 · 확정률 (전체 통합값)
- **드릴다운**(지표 클릭 → 채널별 → 다시 클릭 → 최종):
  - **조회수**: 전체 → 채널별 조회수 → **영상별 조회수**(영상·게시일·조회수)
  - **클릭**: 전체 → 채널별 클릭 → **영상별 클릭수**
  - **예약**: 전체 → 채널별 예약 → **개별 예약**(예약일·금액·투숙기간·박·인원·객실·국가·상태 = 매니저 형태)
  - 조회수·클릭은 영상별까지, 예약은 개별 예약까지(한 칸 더 깊음)
- **전환 퍼널**: 조회 → 클릭(전환율) → 예약(전환율) → 확정(확정률). "얼마나 봐서 얼마나 눌러 얼마나 예약했나"
- 조회수·클릭은 유튜브 연동·추적 링크 후 채워짐(지금 "연동 후"), 예약은 실데이터

### ③ 채널별 예약
채널마다 예약 건수·확정률. 어느 채널이 이 호텔에 효과적인가.

### ④ 예약·방문·취소·노쇼
전체 집계(예약/방문/취소/노쇼 + 방문율) + [세부 전체 보기](채널·기간 필터·페이지, 건수 많아도 안 무너지게). 세부 = 매니저 제공 형태.

### ⑤ 취소·확정률 전략 (확정 2026-07-11)
> 라이브 코드(manager-dashboard.html·sql/04-bookings-agoda.sql) 검증 기반. 상상 금지.

**확정률 정의 = 옵션 A · 실투숙 기준 (대표님 확정).**
확정 = 체크아웃완료. **취소·노쇼 둘 다 미확정**("실제 몸이 왔나"). 매출 KPI(노쇼 포함, D-061)와 라벨 충돌 방지 위해 노쇼는 별도 표기한다.

**⑤-1 리드타임별 확정률 (지금 구현 가능).**
- 리드타임 = `checkin_date − booked_at` (일). manager-dashboard에 동일 로직 존재(취소 제외 valid만, d≥0).
- 구간: 0~3(임박) / 4~7 / 8~14 / 15~30 / 31~60 / 61+(원거리).
- 구간별: 예약수 · 확정 · 취소 · 노쇼 · 확정률(%).
- 가설: 임박 예약↑ 확정률 / 원거리 예약↑ 취소. **실데이터가 가설과 다르면 그게 인사이트.**

**⑤-2 취소 패턴 (대체안 — 지금 구현 가능).**
- ⚠️ **라이브 확인 결과: 아고다 원천(`bookings_agoda`)에 취소일 컬럼 없음.** `booked_at`·`checkin_date`·`booking_status`·`is_cancelled`만 존재, `updated_at`은 DB 갱신시점이라 실제 취소시점 아님. 아고다 임포트도 취소일 파싱 안 함.
- ⇒ "체크인 D-몇일 전에 취소가 몰리나"(진짜 취소 타이밍) **지금 불가.**
- **대체 = 리드타임 구간별 상태 구성비**(확정/취소/노쇼 스택). "언제 잡은 예약이 잘 깨지나."
- 진짜 취소 타이밍은 아고다 리포트에 취소일이 있으면 `cancelled_date` 컬럼 1개 추가로 **⑤-2 승격**(조건부, 아래 재정리 조건 참조).

**⑤-3 판단 힌트 — TOP3 배지 (임계값 잠정).**
- 호텔 확정률 → 자동 배지: **안정 ≥85 / 보통 70~84 / 주의 50~69 / 경고 <50**.
- 주의="TOP3 넣을 때 취소 유출 감안", 경고="노출 대비 실투숙 낮음, TOP3 신중".
- 임계값 85/70/50은 **잠정**(추후 대표님 조정 가능). 배지는 호텔 목록·상세 헤더에 자동 표시.

> **⚠️ 재정리 조건 (대표님 지시):** 위 수치·구간은 현 데이터 기준 설계다. **대표님이 실제 아고다 예약파일을 업로드하면 ⑤ 전체를 실데이터로 재정리**한다. 이때 취소일 컬럼이 존재하면 ⑤-2를 진짜 취소 타이밍(체크인 D-day 기준)으로 승격 검토.

### ⑥ 재사용
마지막 노출 + 6개월 → 재사용 가능일.
