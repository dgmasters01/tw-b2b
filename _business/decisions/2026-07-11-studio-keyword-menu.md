# 2026-07-11 스튜디오 키워드 메뉴 (UX/UI) — 분석·추천 엔진 + 수요 트렌드 인텔리전스

> 압축본: `_os/charter/decisions-index.md` (D-065)
> 결정자: 대표님 / 정리: Claude
> 스튜디오 6메뉴 중 다섯 번째 "키워드". 라이브 코드(_content/youtube/키워드-실측.md v1·keywords/{도시}.csv·youtube.js §4) 검증 기반.

## 목적
키워드 메뉴 = **"다음에 뭘 만들지"를 데이터로 뽑아주는 분석·추천 엔진** + **수요 트렌드 인텔리전스(선행 지표)**. 키워드 나열이 목적이 아니라, 실측(수요·경쟁)으로 발굴하고 월별로 누적해 계절·전년 트렌드를 만들어 "어느 나라·도시·위치·성급 호텔이 언제 필요한지 미리" 알려주는 자리. **사람은 확인·미세조정만**(수동 채택 없음).

## 구조 (메인 + 3단계 드릴)
### 메인(홈) = 국가 개요
- **월 선택**: 상단 월 스트립(최근·앞 달 빠른 접근) + **연·월 선택기**(데이터 쌓일수록 과거 조회·여러 해 비교). 월 의미 2축 = **지난 달=실측 복기 / 앞 달=겨냥 준비**(기본 동작).
- **국가 개요 순위**: 이번 달 어느 나라에 집중할지(착수 우선도=수요 상승+미개척 기회). 나라별 신호·미개척 수·추천 콘텐츠 수.
- **선행 준비**(여기가 제자리 — 여러 나라 가로지름): 착수=피크−3달(리드타임 3달), 4개월 앞까지. 겨냥 월별 콘텐츠·나라·도시·상태(늦음/지금 착수/여유). → 미리 만들어 예약 건다.

### 도시 개요 (나라 클릭)
그 나라 도시별(오사카·도쿄·후쿠오카…): 이번 달 수요·미개척·추천 수·월별 추이. 도시 클릭 → 상세.

### 도시 상세 (도시 클릭)
- **① 콘텐츠 추천**: 순위 + 근거 + **넣을 키워드 세트**(자수 포함). [이걸로 만들기].
- **② 키워드 순위 자동 정리**: 기회점수순, 월별 추이 스파크라인, **보는 용**(수동 채택 없음). 채택 후보 상위 ~30(500자 범위) + [더 보기]. "기준 월" 명시.
- **③ 자동 채택 태그란**: 규칙 적용 500자 자동 완성. 미세조정만. [복사]·[§4 저장].
- **④ 미개척 기회**: 순위 + 키워드. **클릭하면 펼쳐져 키워드별 노출량(월간 수요)** 표시 → 얼마나 큰 기회인지 판단.
- **⑤ 그 도시 수요 트렌드**: 월별, **여러 해(2024·2025·2026) 겹쳐** 계절 패턴 확인.
- **연관 키워드**: 자동완성 파생·호텔명 축약형 별도 섹션(누르면 채택 추가).

### 네비게이션
상단 경로(breadcrumb) 클릭으로 상위 단계 이동("일본›오사카"에서 일본=도시개요, 이번 달 개요=메인). 홈으로 언제든 복귀.

## 실측 방법론 (라이브 키워드-실측.md v1)
- **기회점수 = 수요 ÷ log10(경쟁)**. 수요=구글/유튜브 자동완성 순위, 경쟁=유튜브 검색결과 estimatedResults(±20% 노이즈).
- 살아있는/죽은 키워드 구분. 화이트리스트 토큰 `[나라][도시][지역1~3][역명][N성급]`, 도시·지역·역명 중 하나 필수 포함.
- 도시 대표어(오사카 호텔 226만) = 못 이겨도 비용 0이라 넣음. 성급 어형 = 자동완성 없어도 AI검색용 1개 유지.
- 띄어쓰기/붙여쓰기 페어(1.5배 벌어지면 둘 다, 붙여쓰기 경쟁 500 미만이면 죽은 어형 제외). 호텔명 규칙.

## 데이터 (핵심 전제)
- **실측 = 월별 스냅샷 누적**(지금은 단발 CSV → 매달 실측 돌려 쌓아야 트렌드·선행·여러 해 비교 가능).
- 도시별 CSV 존재: 교토·나고야·도쿄·오사카·후쿠오카(`_content/youtube/keywords/`).
- **회수 = 키워드 메뉴 시계열**(성과표 아님 — 성과표엔 키워드별 조회 없음, 조회수는 영상 단위).

## 리드타임 (확정)
콘텐츠를 **3달 미리** 만들어 예약 건다. 착수 = 겨냥(피크) − 3달. 선행 뷰 = 4개월 앞까지.

## 출구 · 연동
- **출구**: [이걸로 만들기] → 전략 "만들 콘텐츠 기획 큐" 등록(주제·키워드·근거 포함).
- **연동**: 채널 규격 §4(태그란 템플릿) · 올리기(원고 태그 render §4) · 전략(추천→기획) · 호텔(도시·호텔명 키워드). 실측 소스=구글/유튜브 자동완성(수요)·검색 estimatedResults(경쟁).

## 전략 기획 큐 미리보기 (전략 메뉴 소유 · 여기선 출구로 미리봄)
- **카테고리 = 진행 상태**: 기획 대기 → 원고 작성 → 발행 예정 → 발행 완료.
- **항목 필드**: 주제·나라·도시·성급·넣을 키워드·목표 발행월(리드타임 반영)·채널·담당·우선순위.
- 발행 완료 → 성과표/호텔로 성과 회수.

## 보강 확정 (2026-07-12 재점검 — 타겟 축 + 조사 운영)

### 타겟(언어+시장) = 조사의 첫 축 (신규 핵심)
- 조사 단위 = **언어 + 시장** 한 쌍. "방콕 조사"가 아니라 "일본어로 방콕 조사". 같은 도시도 타겟마다 완전히 따로 쌓임(섞이지 않음).
- **중국어 분리 필수**: 대만·홍콩=중국어 번체, 중국 본토=중국어 간체. 본토는 유튜브 밖(차단)→조사법 후속. "중국어" 단일 묶음 금지.
- **타겟 목록 = 채널에서 자동 생성**(하드코딩 금지, D-064 원칙 계승). 채널 추가 = 타겟 자동 추가.
- **두 종류**: ①운영 중 타겟(채널 있음, 매달 자동 조사) ②탐색용 타겟(채널 없이 조사만=정찰, "채널 없음·탐색용" 배지). 조사 좋으면 채널로 승격→상시 자동 편입. 조사가 채널 신설의 선행 근거.
- **타겟 선택 UI = 드롭다운+검색**(무한 확장, D-060 "가로나열 금지" 계승). 현재 타겟 하나 크게+자주쓰는 것 빠른전환+검색+[새 타겟]. 칩 나열 폐기.

### 조사 운영 방식
- **주기 = 매달 자동 + 급할 때 수동**. 매달 1일 등록된 모든 (타겟×도시) 자동 조사. 수동=특정 도시 [지금 다시 조사]. 같은 달 재조사=그 달 스냅샷 덮어씀(월 1점 원칙).
- **조사 날짜 항상 표시**: 각 데이터에 "마지막 조사: YYYY-MM-DD"(자동·수동 모두 갱신).
- **권한**: 새 도시 조사·새 타겟 추가 = owner·admin만(유튜브 요청 발생). 에디터=조회만.

### 새 도시 첫 조사 화면 (빈→중→결과)
- 1단계 빈: "아직 조사 전"+[조사 시작](owner/admin만 보임)+예상 2~3분.
- 2단계 중: 진행 표시(수집→경쟁→기회점수).
- 3단계 결과: 추천·키워드·미개척 채워짐+마지막 조사 날짜. 그 도시=다음 달부터 자동 조사 합류.

### 자동 태그 손질 = 저장해 지키기
- 자동 500자 태그를 사람이 손보면 **저장해서 유지**. 매달 자동은 "새 추천"으로 옆에 참고용만(손질본 안 덮음). 자동 추천 중 손질본에 없는 게 있으면 "새 키워드 N개" 표시(반영은 사람 판단). 손질 자유도=자유, 500자 초과 시 화면 경고.

### 화면 진입 순서 (타겟 축 반영 최종)
① 맨 위 타겟 선택(드롭다운)+월 → ② 국가 개요(홈·선행 준비) → ③ 도시 개요(나라 클릭) → ④ 도시 상세(①추천②키워드순위③자동태그④미개척⑤수요 트렌드) → [이걸로 만들기]→전략 큐.
- **네비 = 상단 경로(홈›타겟›나라›도시) 클릭 + [메인으로] 버튼**. [이걸로 만들기] 안 눌러도 자유 복귀·다른 도시 선택 가능.

# 🔴 전면 개정 (2026-07-14) — 실측 검증으로 뿌리 교체

> 아래 개정이 위 원본과 충돌하면 **개정이 우선**. 원본은 이력으로 보존.
> 근거 전문: `_business/decisions/2026-07-14-keyword-menu-redesign-wip.md`
> 개정 사유: 구글 트렌드·예약·구글플레이스 **실측**으로 검증한 결과, 기존 공식과 전제가 틀린 것이 확인됨.

## ① 기회점수 공식 교체 (근본 결함 수정)

**문제**: 자동완성 순위를 "수요"로 썼는데, 그건 수요가 아니라 **"그 접두어를 친 사람 중에서의 조건부 순위"**였음.
실측 대조 (구글 트렌드 유튜브 검색, 133주):
| 키워드 | 기존 기회점수 | 실제 검색 |
|---|---|---|
| 히요리 호텔 오사카 난바 | **3.58 (1위·초록)** | **0.9** (133주 중 3주) |
| 난바역 근처 호텔 | 2.21 | 데이터 없음 |
| 오사카 호텔 (대표어) | **1.28 (꼴찌·회색)** | **46.2** (133주 전부) |
→ **정확히 역순.** 검색 0인 키워드를 1위로 추천하고 있었음.

**확정**:
- **수요 = 구글 트렌드 실측** (`property:youtube`, geo=타겟시장).
- **자동완성 순위 = 수요 지표에서 제외.** "어떤 키워드가 존재하나"(연관어 발굴)용으로만 사용.
- 경쟁 = 유튜브 검색결과 영상 수 (기존 유지, 자릿수만 신뢰).

## ② 분석 축 = 나라 → 도시 → 지역. 호텔은 **출력**

- **축(질문 = 사람들이 뭘 찾나)**: 나라 → 도시 → **지역**. 트렌드로 수요·계절 측정.
- **출력(답 = 우리가 골라 보여주는 것)**: 그 지역의 **여러 호텔**. 호텔 마스터 = 후보 공급원(계층 아님).
- **태그만(비용 0)**: 호텔명·역명·측정 안 되는 지역.

**사업 근거**: 우리 수익 = 도시의 여러 호텔을 발굴·추천해 예약. 호텔명 검색자는 **이미 그 호텔로 정한 사람** → 우리가 낄 자리 없음. 도시·지역 검색자가 우리 고객.

**계층이 아니라 "이름값"이 가름** (실측): 난바 42.5 vs 우메다 5.7 (같은 지역 계층인데 8배). → **지역 목록 미리 만들지 말 것. 재봐서 살아있는 것만 축으로.**

**호텔 지역 판정 = 좌표** (검증 완료): 구글 플레이스로 주소·좌표 채움 → `hotels.district`. 도시 중심 30km 초과 = `manual_check`(동명 호텔 오매칭, 10건 중 1건 실제 검출).

## ③ 계절 근거 = 구글 트렌드 (예약 기반 폐기)

- **폐기**: 예약 건수 기반 계절 곡선. 2024는 U자(12월 183 피크)인데 2025는 계속 내리막(2월 155→12월 79). 비중(%)으로 봐도 정반대. → 여행 계절이 아니라 **우리 채널 활동**을 그림. **대표님 지시로 삭제.**
- **채택**: 구글 트렌드 유튜브 검색. 도시마다 계절이 명확히 다르고 2년 반복됨 → 진짜 신호.
  - 오사카 = 겨울 피크(11월 58.0, 1월 56.8) / 봄 바닥(4월 36.0)
  - 다낭 = 여름 피크(8월 51.8, 6월 48.4) / 겨울 바닥(1월 34.1)
- **조회 기간 = 2024-01 ~ 현재** (2년 반). 용도가 다름:
  - **계절 모양**: 2024+2025 평균 (반복되면 진짜)
  - **인기도 판단**: 2025 vs 2026 (최근)
  - **2024 = 기준선** — 없으면 "진짜 하락"과 "반짝 후 정상화"를 구별 못 함 (방콕 67→50→32 진짜 하락 / 타이베이 17→31→17 반짝 정상화 / 세부 47→14→13 이미 붕괴)
- **비행시간 제한 없음** (대표님 지시): 데이터가 지지 안 함(타이베이 2.5h 하락, 나트랑 5h 상승). 타겟만 좁힘.

## ④ 착수 시점 = 검색 피크 − 3달

- 리드타임 근거 = **우리 예약 실측** (확정 3,766건): 중앙값 19일, 3달 안 **92%**.
- 도시별 보정: 치앙마이 25일·88% = 4달 전 / 호치민 12일·96%, 방콕 14일·92% = 3달 전 여유.
- 메인 기본 = "이번 달 수요"가 아니라 **"지금 착수 = 3달 뒤 노출 겨냥"**.
- 월 선택 = 현재 달부터 **6개월** + 연·월 지정(과거·미래) + 지난달 복기.
- 적용 예: 오사카 피크 11~1월 → 8~10월 발행 → 7월 = 착수. 다낭 피크 6~9월 → 3~5월 발행 → 7월 = **올해는 늦음**.
- ⚠️ 원본의 "오사카 단풍철 10월", "다낭 건기 10~11월" 은 **Claude 날조**. 실측과 정반대.

## ⑤ ④미개척 → **확장 여지** 로 재설계 (대표님 지적)

기존 "영상 있으면 중복 → 건너뜀"은 **가장 확실한 돈줄을 버리는 것**. 우선순위 재정의:
1. **검증됨 + 남은 호텔 많음 → "더 캐기"** (1순위·가장 안전). 영상 있고 예약도 붙음 = 되는 게 증명됨. 호텔만 바꿔 2편·3편. 오사카 = 예약 붙는 검증 호텔 **255개**, TOP3면 **85편 여지**.
2. **미개척** (2순위·도박). 수요 크지만 될지 모름.
3. **영상 있는데 예약 안 붙음** (3순위). 각도 바꾸거나 접기. 그대로 반복 금지.

⚠️ **지금은 이 판정 불가**: hotels에 **영상 노출 컬럼 없음**. 대표님이 수동 정리해 주실 **노출 이력(날짜·채널·TOP3 호텔)** 입력 전까지는 "확장 여지 = 남은 검증 호텔 수"로만 표시. **없는 걸 있다고 하지 말 것.**

## ⑥ 추천 알고리즘 (5단계 거름망 · 순서가 핵심)

1. **계절 게이트** — 검색 피크 − 3달 = 착수 월. 겨냥 월이 범위 밖이면 **탈락**. (수요·경쟁은 다음 달에도 잡지만, 놓친 계절은 1년을 기다림)
2. **수요 크기** — 트렌드 실측. 측정 안 되면 "모름"으로 후순위.
3. **인기도 추세** — 2025 vs 2026, 2024 기준선.
4. **확장 여지** — ⑤ 참조.
5. **경쟁** — 영상 수.

**전환(우리 예약)은 순위에 쓰지 않음** — 우리가 안 간 도시는 영원히 0이라 **새 발굴을 막음**. "검증됨" 배지로만 표시.

## ⑦ 신규 3가지

1. **검색** — 나라·도시·지역 아무거나. 조사 안 된 곳도 검색 → [조사하기] → 그때 조사·저장. (목록에 갇히지 않음)
2. **[왜?] 근거 카드** — 모든 추천에. 실측 4종(계절 곡선·인기도 3년·경쟁/확장여지·전환) + 조사일. **근거 칸이 비면 추천 불가 = 날조 구조적 방지.**
3. **호텔 후보** — `hotels.district` 기준, 예약 붙는데 안 다룬 호텔.

## ⑧ "없음" vs "모름" 구별 (필수)

구글 트렌드는 **측정 바닥**이 있음(5 미만 안 잡힘). 우메다 5.7 = "검색 없음"이 **아니라 "우리가 못 잼"**. 화면에 "없음"이라 쓰면 그것도 거짓 → **"측정 한계 아래"**로 표시.

## ⑨ 업데이트 주기

- **매달 1일, 전 타겟 × 도시 자동 조사.** (계절은 천천히 변해 매주 봐도 판단 안 바뀌고 호출만 늚)
- 착수 임박 도시(겨냥 −3~4달) 우선. 검색으로 새 도시 찍으면 그 자리에서 조사.

## ⑩ 구글 트렌드 기술 제약 (구현 시 필수)

- 호출: `/trends/api/explore` → TIMESERIES 토큰 → `/trends/api/widgetdata/multiline`. pytrends는 막힘(직접 호출로 우회).
- ⚠️ `time:"today 24-m"` = **400 에러**. **날짜 범위 지정("2024-01-01 2026-07-15")** 사용. `today 12-m`, `today 5-y` 는 OK.
- ⚠️ **키워드마다 독립 정규화** → "나트랑 72 > 방콕 32" 읽으면 **틀림**. 같은 키워드의 시간 변화만 비교 가능. **도시 비교하려면 한 쿼리에 묶을 것(최대 5개).**
- 상대지수(0~100), 절대 검색수 아님.

## ⑪ 키워드 전체 측정 = 앵커 기법 (2026-07-14 검증)

**문제**: 구글 트렌드는 한 번에 5개까지만. 묶음이 다르면 잣대도 달라 서로 비교 불가.
**해법**: **모든 묶음에 대표어(앵커)를 포함** → 앵커 값이 같아져 이어붙일 수 있음.
- 실측 검증: 묶음1 오사카호텔=46.23 / 묶음2 오사카호텔=46.23 → **보정배율 1.000 (완벽 일치)**
- **키워드 40개 = 트렌드 10회** (묶음당 앵커1+신규4). 월 1회. **DB 저장 후 화면은 호출 0.**
- ⚠️ **구글 트렌드도 429(호출 제한) 있음** — 2026-07-14 실제로 걸림. 봇에 **재시도 + 간격(15~25초)** 필수.

- **제목**: 최상위 수요어 (오사카면 "호텔"+"숙소" 둘 다)
- **본문/설명**: 그다음 (숙소 추천 16.7 등)

## ⑭ 지역 계절 ≠ 도시 계절 (2026-07-14 실측)
- **난바**: 11월 단일 피크 66.3 / 4월 바닥 20.6 → **진폭 3.2배**
- **오사카**: 11·12·1월 완만 (58.0/55.0/56.8) / 4월 36.0 → **진폭 1.6배**
- → **살아있는 지역은 계절을 따로 재야 함.** 도시 곡선으로 갈음하면 뾰족한 피크를 놓침.
- 측정 안 되는 지역(우메다 0.6 등)은 도시 곡선 차용 + **"도시 기준이라 정확도 낮음" 명시**.

## 🚨 Claude 날조 정정 이력 (3건 — 모두 실측으로 반박됨)
1. "오사카 단풍철 10월 노출에 딱" → 실측: 오사카 피크는 **11~1월(겨울)**. 10월은 상승 진입.
2. "다낭 건기 10~11월 오름" → 실측: 다낭 피크는 **6~9월(여름)**. 10월 겨냥은 늦음.
3. "난바 42.5 ≈ 오사카 46.2, 둘 다 큼" → **각자 정규화된 값 비교 = 틀림.** 같이 재면 난바 10.8 (1/4).
