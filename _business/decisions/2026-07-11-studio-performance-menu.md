# 2026-07-11 스튜디오 성과표 메뉴 (UX/UI) — 전 채널·영상·호텔 종합 조망

> 압축본: `_os/charter/decisions-index.md` (D-063)
> 결정자: 대표님 / 정리: Claude
> 스튜디오 6메뉴 중 두 번째 "성과표". 라이브 코드(manager-dashboard·publications·bookings_agoda·channels) 검증 기반.

## 목적
성과표는 **전 채널·영상·호텔의 성과를 한 화면에서 조망**하는 종합 대시보드다. 정렬·필터로 "뭐가 잘 되고 뭐가 새는가"를 빠르게 가려낸다. 기준 3축 = 조회수 · 영상 수 · 실예약(**구독자 수 절대 금지**).
- **경계**: 성과표 = 전체 조망 / 호텔 메뉴(D-062) = 호텔 1개 심층. 겹치지 않고 이어진다. 성과표에서 문제를 찾고 → 호텔 상세(특히 ⑤ 취소·확정률)로 드릴다운.

## 용어 (중요 — 대표님 확정)
- **예약금액** = 손님이 호텔에 낸 돈 = **호텔 매출**. 우리 매출 아님. 호텔에게 "우리가 이만큼 예약을 만들어줬다"를 증명하는 실적 숫자. (확정·취소 제외 기준)
- **수수료** = **우리 수익** = 아고다 제휴 커미션. 대표님(owner)만 봄.
- 이전 명칭 "확정매출" 폐기 → **예약금액**으로 통일. KPI·표·추세·모든 드릴 단계 일괄 적용.

## 구성 (위 → 아래)
### 1. 기간 선택
이번 달 · 최근 3개월 · 전체 · 기간 지정. 모든 지표가 이 기간 기준.

### 2. KPI 요약 스트립 (전체 합계)
영상 수(발행됨) · 조회수 · 클릭 · 예약 · 취소율 · 확정률 · 예약금액 + **[수수료 = owner만]**.
- 조회수·클릭 = 연동 후 채워짐 → 지금은 정직하게 `—`(회색).

### 3. 렌즈 3개 (같은 데이터, 관점 전환)
채널별(기본) · 영상별 · 호텔별. **나라·언어는 렌즈 아님 = 필터**(렌즈는 행의 성격 자체가 바뀌는 것, 나라·언어는 걸러보는 속성).

### 4. 메인 표
- 정렬: 열 클릭. 필터: 채널·언어·나라.
- 열: 영상 · 조회수(`—`) · 예약 · **취소(건수·취소율)** · 확정률 · 예약금액 · **[수수료 owner]**.
- **취소는 정렬용 신호**로만 얹는다(누가 제일 새나 즉시 발견). 깊은 분석(왜·리드타임·단계·TOP3)은 **호텔 ⑤가 담당**. 취소율 정렬 → 행 클릭 → 호텔 ⑤로.

### 5. 드릴다운 — 4단계 (펼쳐보기, 인라인)
**채널 → 영상 → 소개된 호텔(TOP1·2·3) → 개별 예약.**
- 채널 클릭 → 영상별 펼침.
- 영상 클릭 → **그 영상이 소개한 호텔들**(원고의 TOP1·2·3 파트너 링크 hid 기준). 영상 하나에 호텔 여러 개.
- 호텔 클릭 → 그 호텔의 **개별 예약**(예약번호·예약일·투숙기간·박·국가·금액·상태·[수수료 owner] = 매니저 D-061·호텔 ④ 컴포넌트 재사용). 취소 예약은 수수료 `—`.
  - **리프 화면 구성(확정)**: ①경로(breadcrumb) 채널 › 영상 › 호텔(순위) › 개별예약 — "어느 호텔 예약인지" 항상 표시 / ②호텔 요약 한 줄(순위뱃지·호텔명·성급·도시·hid + 예약·취소·확정률·예약금액·[수수료 owner] + [호텔 상세→D-062] 링크) / ③개별 예약 표(N건 중 일부 + [전체 보기]).
  - 호텔명 → 호텔 메뉴(D-062) 상세로 이동.

### 6. 월별 추세 (표와 동거)
토글: 예약 · 조회수(연동 후) · 예약금액 · **[수수료 owner]**. 지금값(표)과 흐름(추세)을 나란히.

## 권한
**수수료 KPI·열·추세 토글 = 대표님(owner)만.** 에디터·매니저 화면엔 렌더 자체가 안 됨(D-053·D-061 원칙). 권한 판정으로 렌더만 분기.

## 데이터·연결
- 영상 수 = publications(발행됨)
- 예약·취소·확정률·예약금액 = bookings_agoda + 집계 뷰(v_manager/admin_channel_stats·hotel_summary·video_summary)
- 수수료 = v_admin_* (owner)
- 채널 = channels / CID = channel_cid_map
- 영상↔호텔 = publications ↔ hotels(hid), TOP1·2·3
- 조회수 = D-059 연동 후 / 클릭 = 추적링크(gohotel.win/h/) 후
- **영상 단위 예약 귀속** = 추적링크 연동 후 정확. 연동 전엔 "채널 예약 중 이 영상 구간"으로 잠정.
- 확정률 정의 = 실투숙 기준(D-062 ⑤ 옵션 A와 동일)

## 후속 / 재정리 조건
- 조회수·클릭·영상별 취소 귀속은 유튜브 연동·추적링크 후 실값 채움.
- 대표님 아고다 예약파일 업로드 시 예약금액·취소·확정률 실데이터로 재정리(D-062 ⑤와 동일 조건).
