# 2026-07-11 스튜디오 성과표 메뉴 (UX/UI) — 전 채널·영상·호텔 종합 조망

> 압축본: `_os/charter/decisions-index.md` (D-063)
> 결정자: 대표님 / 정리: Claude
> 스튜디오 6메뉴 중 두 번째 "성과표". 라이브 코드(manager-dashboard·publications·bookings_agoda·channels) 검증 기반.

## 목적
성과표는 **전 채널·영상·호텔의 성과를 한 화면에서 조망**하는 종합 대시보드다. 정렬·필터로 "뭐가 잘 되고 뭐가 새는가"를 빠르게 가려낸다. 기준 3축 = 조회수 · 영상 수 · 실예약(**구독자 수 절대 금지**).
- **경계**: 성과표 = 전체 조망 / 호텔 메뉴(D-062) = 호텔 1개 심층. 겹치지 않고 이어진다. 성과표에서 문제를 찾고 → 호텔 상세(특히 ⑤ 취소·확정률)로 드릴다운.

## 용어 (중요 — 대표님 확정)
- **예약금액** = 손님이 호텔에 낸 돈 = **호텔 매출**. 우리 매출 아님. 호텔에게 "우리가 이만큼 예약을 만들어줬다"를 증명하는 실적 숫자. (확정·취소 제외 기준)
- **수수료** = **우리 수익** = 아고다 제휴 커미션. 대표님(owner)만 봄.
- 이전 명칭 "확정매출" 폐기 → **예약금액**으로 통일. KPI·표·추세·모든 드릴 단계 일괄 적용.

## 구성 (위 → 아래)
### 1. 기간 선택
이번 달 · 최근 3개월 · 전체 · 기간 지정. 모든 지표가 이 기간 기준.

### 2. KPI 요약 스트립 (전체 합계)
영상 수(발행됨) · 조회수 · 클릭 · 예약 · 취소율 · 확정률 · 예약금액 + **[수수료 = owner만]**.
- 조회수·클릭 = 연동 후 채워짐 → 지금은 정직하게 `—`(회색).

### 3. 렌즈 3개 (같은 데이터, 관점 전환)
채널별(기본) · 영상별 · 호텔별. **나라·언어는 렌즈 아님 = 필터**(렌즈는 행의 성격 자체가 바뀌는 것, 나라·언어는 걸러보는 속성).
- **렌즈 공통 드릴 원칙(확정)**: 세 렌즈 모두 **같은 펼쳐보기 드릴 컴포넌트를 공유**하고 진입 레벨만 다르다. 하나 만들어 세 곳에서 재사용.
  - 채널별 = 채널 → 영상 → 소개된 호텔 → 개별 예약 (4단계)
  - 영상별 = 영상 → 소개된 호텔 → 개별 예약 (3단계, 채널 레벨만 빠짐). 채널 넘나드는 영상 성과 랭킹(효자 영상/문제 영상 발굴), 행에 채널 뱃지·게시일·소개호텔수 표시.
  - 호텔별 = 호텔 → 개별 예약 (2단계, 바로 호텔 리프). 호텔 기준 성과·취소, 호텔명 → 호텔 메뉴(D-062) 상세.
  - 어느 렌즈든 펼치면 동일한 **호텔 요약바 + 개별 예약 표**(리프)로 수렴.

### 4. 메인 표
- 정렬: 열 클릭. 필터: 채널·언어·나라.
- 열: 영상 · 조회수(`—`) · 예약 · **취소(건수·취소율)** · 확정률 · 예약금액 · **[수수료 owner]**.
- **기본 정렬 = 예약 건수 내림차순**(기존 집계 뷰 v_*_channel_stats booking_count desc 관례와 동일) + **순위 번호(#1·#2·#3…)**: 잘되는 게 앞으로, 약한 건 아래로. 정렬 기준 바꾸면 순위 재계산.
- **대상 순위 ≠ 시간 순위**: "어느 채널/영상/호텔이 잘되나"는 이 리스트가, "어느 **날/기간**이 강했나"는 **추세 그래프**가 담당(일·주·월 토글, 특정 날 클릭 → 그날 예약). 둘을 리스트에 섞지 않는다.
- **취소는 정렬용 신호**로만 얹는다(누가 제일 새나 즉시 발견). 깊은 분석(왜·리드타임·단계·TOP3)은 **호텔 ⑤가 담당**. 취소율 정렬 → 행 클릭 → 호텔 ⑤로.

### 5. 드릴다운 — 4단계 (펼쳐보기, 인라인)
**채널 → 영상 → 소개된 호텔(TOP1·2·3) → 개별 예약.**
- 채널 클릭 → 영상별 펼침.
- 영상 클릭 → **그 영상이 소개한 호텔들**(원고의 TOP1·2·3 파트너 링크 hid 기준). 영상 하나에 호텔 여러 개.
- 호텔 클릭 → 그 호텔의 **개별 예약**(예약번호·예약일·투숙기간·박·국가·금액·상태·[수수료 owner] = 매니저 D-061·호텔 ④ 컴포넌트 재사용). 취소 예약은 수수료 `—`.
  - **리프 화면 구성(확정)**: ①경로(breadcrumb) 채널 › 영상 › 호텔(순위) › 개별예약 — "어느 호텔 예약인지" 항상 표시 / ②호텔 요약 한 줄(순위뱃지·호텔명·성급·도시·hid + 예약·취소·확정률·예약금액·[수수료 owner] + [호텔 상세→D-062] 링크) / ③개별 예약 표(N건 중 일부 + [전체 보기]).
  - **호텔별 리프 특칙**: 여러 영상에서 온 예약을 통합하므로 **유입 영상/채널 열** + 상단 **"유입 영상별 요약" 블록**(이 호텔 × 각 영상: 예약·확정률·예약금액). 요약 카드 클릭 = 아래 예약 목록을 그 영상만 필터 / 영상 이름 클릭 = 영상별 렌즈로 점프(그 영상 전체). 유입 영상 단위는 추적링크 연동 후 정확, 연동 전엔 채널까지.
  - 호텔명 → 호텔 메뉴(D-062) 상세로 이동.

### 6. 월별 추세 (표와 동거)
토글: 예약 · 조회수(연동 후) · 예약금액 · **[수수료 owner]**. 지금값(표)과 흐름(추세)을 나란히.

## 권한
**수수료 KPI·열·추세 토글 = 대표님(owner)만.** 에디터·매니저 화면엔 렌더 자체가 안 됨(D-053·D-061 원칙). 권한 판정으로 렌더만 분기.

## 데이터·연결
- 영상 수 = publications(발행됨)
- 예약·취소·확정률·예약금액 = bookings_agoda + 집계 뷰(v_manager/admin_channel_stats·hotel_summary·video_summary)
- 수수료 = v_admin_* (owner)
- 채널 = channels / CID = channel_cid_map
- 영상↔호텔 = publications ↔ hotels(hid), TOP1·2·3
- 조회수 = D-059 연동 후 / 클릭 = 추적링크(gohotel.win/h/) 후
- **영상 단위 예약 귀속** = 추적링크 연동 후 정확. 연동 전엔 "채널 예약 중 이 영상 구간"으로 잠정.
- 확정률 정의 = 실투숙 기준(D-062 ⑤ 옵션 A와 동일)

## 출처별 성과 비교 (2026-07-12 · D-066 연동)
- 콘텐츠 출처(전략 추천/키워드 발굴/원고 유입) 꼬리표를 발행 후 성과까지 유지 → **출처별로 묶어 편수·평균조회·예약·확정률 비교**. "데이터로 고른 게 진짜 더 잘 됐나" 검증(전략 시스템 ROI).
- **펼쳐보기 = 성과표 공통 드릴다운 재사용**: 출처 누르면 개별 영상 리스트→영상 누르면 호텔→개별 예약. 새 컴포넌트 X.
- 위치=성과표 메뉴 내 섹션. 전략 메뉴에서 궁금하면 여기로.

## 접속 국가(고객 국적) 축 — 신규 (2026-07-12 · 대표님 발견)
- 아고다 원본 `사이트 주소 위치` = **예약자가 접속한 국가**(고객 국적·거주지). 숙소 `국가`와 다른 별개 축. **DB 미보유 → 재임포트 시 필수 적재**.
- 이걸로 **"어느 채널 → 어느 나라 고객 → 어디 호텔 예약"** 삼각 관계가 실측됨(그동안 채널 언어로 추정만 했던 것).
- **2025 실측**: 한국 hotel2ya 92% KR / 일본 hoteluda 93% JP → 채널↔고객국 일치 확인. **대만 world-hotel = TW 64% + HK 17%**(D-065 대만·홍콩 타겟 분리 판단을 데이터가 뒷받침). **베트남 reviewkhachsan = VN 43% · KR 23%**(타겟 흐림 → 점검 필요).
- **주 흐름**: KR 고객 → 일본342·베트남227·태국141·대만105 / JP 고객 → 베트남60·태국40. "자국민에게 해외 호텔" 모델 확증.
- **확정률 차이(기획 판단 재료)**: KR→태국 69%·이탈리아 74%·호주 71% vs **KR→일본 45%**(건수 1위지만 절반 취소·무료취소 중복예약 추정), JP→베트남 35%. **건수≠수익, 확정률로 효율 판단**.
- ⇒ 성과표에 **접속 국가별 성과** 섹션 추가(채널×접속국×호텔국 교차·확정률). 키워드 타겟(D-065) 실측 검증에도 사용.
