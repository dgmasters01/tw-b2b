# 2026-07-11 스튜디오 올리기 메뉴 전체 설계 (UX/UI)

> 압축본: `_os/charter/decisions-index.md` (D-060)
> 결정자: 대표님 / 정리·기록: Claude
> 상태: 화면(UX/UI) 설계 확정. 실제 개발·드라이브 실연동은 후속 작업.

이 문서는 2026년 7월 11일 대화에서 화면 하나씩 맞춰가며 확정한 "스튜디오 올리기 메뉴"의 전체 설계다.

---

## 0. 스튜디오 전체 틀

- **메뉴 6개(확정)**: 올리기 · 성과표 · 호텔 · 키워드 · 전략 · 채널
- **페이지 한/영 토글**: 화면 글자(버튼·라벨)만 한국어/영어로 바뀐다.
  - **고정(번역 안 함)**: 채널명("호텔이야" 등), 사람 이름.
  - **복사 내용의 언어 = 그 채널의 언어**: 한국 채널 원고는 복사하면 한국어, 일본 채널은 일본어, 영어 채널은 영어. 화면 언어와 무관. (한국 채널에 영어 제목이 올라가는 사고 방지)

---

## 1. 원고 넣기 — 두 경로

1. **직접 올리기** — 화면에서 docx를 골라 [패키지 만들기]. 그 자리에서 바로 등록.
2. **구글 드라이브** — 폴더에 원고를 넣어두면 자동으로 가져옴. (A안 = 읽기 전용, 서버가 파일을 만들지 않음)

원고에 아고다 파트너 링크(cid·hid 붙은 것)가 들어 있으면 코드가 자동으로 읽는다. 그래서 화면에서 링크를 따로 넣는 단계는 없앤다. (링크는 워드 하이퍼링크 저장소 word/_rels 에서 읽음)

---

## 2. 구글 드라이브 흐름

### 폴더 구조 (대표님 계정 소유)
```
유튜브 업로드 자동화
 ├ 여행능력자들 (TW) : 대기 · 완료 · 확인필요
 ├ 호텔이야 (HT)     : 대기 · 완료 · 확인필요
 └ 호텔이곳 (HG)     : 대기 · 완료 · 확인필요
```

### 순서
```
0. 드라이브 연결(한 번만) → 1. 대기에 원고 넣기 → 2. 하루 4번 자동 읽음
→ 3. 올리기 목록에 카드로 뜸 → 4. 완료/확인필요 자동 이동 → 5. 맡기·복사·발행
```

- **연결(0단계)**: ①드라이브에 폴더 만들기 ②서비스 계정 키 등록 ③연결 확인. 1·2는 비밀 키라 대표님이 직접(화면 안내 제공).
- **읽는 시각**: 하루 4번 고정 = 06·11·16·21시. 원고가 없으면 그냥 넘어감(헛돌지 않음).
- **폴더 3개면 충분**: 대기 / 완료 / 확인필요. (보관 폴더는 군더더기라 없앰)
- **완료 = 최종 완성본만.** 우리가 실제로 쓰는 건 이것뿐.
- **안 쓸 원고**는 지우면 구글 드라이브 휴지통으로(30일 복구 가능) — 안전망 이미 있음.

### 확인필요 리스트 정리 규칙
- 리스트에는 **"아직 손봐야 할 미해결 문제"만** 남는다.
- 고쳐서 「대기」에 다시 넣으면 다음 확인 시각에 처리 → 완료로 가고 **리스트에서 자동으로 빠짐**.
- 화면 위에 **"다음 확인 16:00"** 안내(시스템은 실시간 감시가 아니라 그 시각에만 폴더를 봄. 실시간 "재처리 대기" 표시는 하지 않음 — 실제 동작과 어긋나므로).
- 넣었는지 확인은 드라이브 「대기」 폴더를 직접 보면 됨.
- 각 항목에 [지우기](= 휴지통). 원인 표시: 파일명 형식/파트너 링크 없음/cid 불일치/중복 등.

## 3. 올리기 목록 (핵심 화면)

### 보기 3종
- **전체 보기** — 모두가 봄. 여러 채널 한 목록에 섞여 나옴.
- **내 작업** — 내가 맡은 것만.
- **미지정** — 아직 아무도 안 맡은 것만(담당자가 바로 집어감).

### 필터 (모두 드롭다운 = 무한 확장, 가로 나열 안 함)
- 채널 드롭다운 / 담당자 드롭다운(관리자는 특정 사람 골라보기) / 언어 드롭다운

### 각 원고에 새기는 정보
- **채널 뱃지 + 언어**
- **출처**: 자동(드라이브) / 수동(+올린 사람 "○○ 올림")
- **중복 딱지**: 같은 원고가 이미 있는데 [그래도 추가]로 넣은 것. "수동 · ○○ 올림 · 겹침"이면 누가 중복을 만들었는지 저절로 드러남.
- **상태(3종만)**: 대기 · 맡음 · 발행됨. (실시간으로 알 수 없는 "작업 중" 표현은 안 씀)
  - 맡음: 누가 · 맡은 날짜 · 며칠째. 오래되면(예: 3일) "방치" 강조.
- **올린 사람(uploaded_by)** 저장.

---

## 4. 발행 카드 (원고 하나 펼침)

- 맡기 전(대기): [내가 맡기]만. 맡아야 복사·발행이 열림.
- 맡은 뒤: **복사 4개 + 번호** → ①제목 ②설명란 ③태그 ④파일명.
  - 챕터·해시태그·아고다 링크는 **②설명란 안에 구조대로 포함**(따로 복사 안 함).
  - ③태그엔 글자 수 / 유튜브 한도(500자) 표시.
- 발행: 유튜브 주소 붙여넣고 [발행 완료]. [놓기]로 찜 해제(다른 사람이 맡게).
- **발행 후 주소 수정 + 수정 이력**(D-058): 오타 난 주소 고치기, 이전→새 주소·누가·언제 기록, video 재연결.

---

## 5. 중복 방지 (양방향)

- **드라이브로 들어온 중복** → 배치가 "이미 등록됨" 감지 → 확인필요로: "이미 처리된 원고입니다 · 대기 폴더에서 지워주세요".
- **수동 업로드 중복** → [패키지 만들기] 누르는 즉시 경고: "이미 등록된 원고입니다" + 겹치는 원본 + "드라이브 대기에도 있으면 거기서도 지우세요" + [취소]/[그래도 추가].
- 어느 길로 들어와도 같은 영상이 두 번 만들어지지 않는다. 일부러 [그래도 추가]한 것은 목록에 "중복" 딱지로 남는다.

---

## 6. 후속 작업 (별도)

- 드라이브 자동화 실연동(BL-YT-DRIVE-WATCH): 화면 설계 완료 → 개발 → **마지막에** 대표님이 폴더 만들고 키 등록. 지금은 급하지 않음.
- 조회수·클릭은 유튜브 연동·추적 링크 후 채워짐(D-059). 성과표의 조회수/클릭 칸은 그전까진 "연동 후"로 비어 있음.
