# 분석 페이지 날짜 표기 원칙 (채널명 단일화 관련)

**기록일:** 2026-07-12
**배경:** 분석 페이지(booking-analytics) 채널별 탭을 실시간 API 전환(3번 채널명 단일화) 진행 중.
**제기:** 이지형 대표님

## 핵심 원칙 — 날짜 2종을 반드시 분리

이름만 바꿔도 "예약 데이터 업데이트 날짜"가 바뀌면 안 된다. 그러면 실제 예약 통계가 갱신된 것처럼 착시가 생겨 대표님이 헷갈린다.

| 날짜 종류 | 의미 | 언제 바뀌나 | 표기 |
|---|---|---|---|
| **예약 데이터 기준일** | 아고다 예약 자료를 실제로 적재/갱신한 날 (현재 2026-04-15) | 새 예약 데이터를 실제로 넣었을 때만 | "예약 데이터: YYYY-MM-DD" |
| **화면 조회 시각** | 페이지를 지금 연 시각 (이름·이름변경은 여기 속함) | 페이지 열 때마다 / 이름 변경 시 | (표기 안 하거나 "조회: 방금" 정도) |

## 반드시 지킬 것
- 채널 **이름 변경**은 "예약 데이터 기준일"을 **절대 바꾸지 않는다** (이름은 라벨일 뿐, 통계 갱신 아님).
- 분석 페이지 상단 "데이터: 2026.04.15" 는 **예약 데이터 기준일**이며, 실시간 전환 후에도 이 날짜는 **실제 예약 데이터를 새로 넣은 날**로만 갱신한다.
- 실시간으로 이름·집계를 불러와도, 화면에는 "예약 데이터 기준일"을 명확히 표기해 "이 숫자는 언제 자료 기준인지"를 대표님이 항상 알 수 있게 한다.
- → 이렇게 하면 대표님은 "4월 15일 이후 자료를 업데이트해야겠다"를 날짜만 보고 판단할 수 있다.
