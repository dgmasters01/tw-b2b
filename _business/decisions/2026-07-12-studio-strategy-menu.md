# 2026-07-12 스튜디오 전략 메뉴 (UX/UI) — 만들 콘텐츠 기획 큐 + 방향 판

> 압축본: `_os/charter/decisions-index.md` (D-066)
> 결정자: 대표님 / 정리: Claude
> 스튜디오 6메뉴 중 마지막 "전략". 목업 기반 하나씩 확정. 6메뉴 설계 완결.

## 목적
전략 메뉴 = **만들기로 정한 콘텐츠를 발행까지 끌고 가며 진행 상태를 한눈에 보는 실행 큐**. 5메뉴 사이 빈자리(키워드=발굴/올리기=발행/성과표·호텔=회수/채널=자산)에서 "정한 걸 발행까지 관리"를 채운다. **2층 구조 = 방향 판(위) + 칸반 큐(아래)**.

## 세 층위 (혼동 방지 — 핵심)
- **세부(추천·키워드·근거) = 키워드 도시 상세 하나에만.** 전략은 세부 화면 새로 안 만듦.
- **큐 카드 = 요약 + 진행 관리**(넣을 키워드 요약·근거 한 줄·연결 원고·상태). 세부는 [분석 다시 보기]로 키워드 도시 상세 점프.
- **방향 판 = 세부 없는 간판.** 누르면 세부로 가는 문.

## 방향 판 (위층)
- **타겟(언어)별**: 상단 언어 선택 → 그 채널 방향만(드롭다운, 채널서 자동).
- **키워드 자동 제안**: 3달 앞 피크 근거로 월별 방향 제안. 대표님은 확인·조정만.
- **버튼 = [확정]·[보류]·[확정 취소], [바꾸기] 폐기**: 확정=이 방향 밈(칸반 강조 시작)/보류=지금 안 밈/확정취소=제안 중으로 되돌림(가역). **[바꾸기] 없앰** — 없는 방향 손으로 만들면 근거 없고 세부로 갈 데 없음, 키워드 "수동 채택 없음" 원칙 위배.
- **방향 단위 = 나라+성급**(도시·타겟까지 X — 너무 좁으면 큐 카드와 겹침). 도시 여럿 포함.
- **누르면 키워드 도시 상세로 점프**(그 나라·성급 걸러진 채, 세부는 거기서).
- **기간 = 1년 뷰**: 착수 구간(가까운 3~4달, 카드 크게·확정 대상) + 연간 미리보기(나머지 작게·계절 흐름·데이터 쌓이며 또렷·가까워지면 착수 구간으로 승격). 키워드 리드타임 3달 반영.

## 칸반 큐 (아래층)
- **4상태**: 기획 대기 → 원고 작성 → 발행 예정 → 발행 완료.
- **카드 필드**: 주제·고유코드(TW-XXXX)·나라·도시·성급·타겟·목표월·우선순위·기획한 사람·원고 담당·출처.
- **상태 이동 = 실제 작업 따라 자동(+손으로도)**: 기획대기→원고작성=[원고 시작](담당 수동 표시 — 시스템은 "작성 중" 감지 못함)/원고작성→발행예정=드라이브 완료 폴더 유입+고유 코드 매칭(자동)/발행예정→발행완료=올리기 발행(자동).
- **원고-카드 매칭 = 고유 코드**(대표님 안, 문서생성 원안 폐기): 카드마다 코드, 담당이 **어디서든** 원고 쓰고 코드 한 줄 넣음, 드라이브 유입 시 코드로 정확 매칭(이름 추측 X·두 번 일 X). **코드 없음=오류 아님=큐 밖 원고**(예전 방식 바로 발행+원하면 큐 새 카드 기록)/**코드 있는데 안 맞음=확인 필요**(오타).
- **두 역할**: 기획한 사람([이걸로 만들기] 누른 사람 자동)+원고 담당(따로 배정). 대표님이 원고 안 써도 "기획=대표님" 남음.
- **담당 배정 접속자별**: 관리자·대표님=[담당 지정]+[내가 맡기] 둘 다/에디터=[내가 맡기]만(남 배정 잠김). D-060 맡기/놓기 방식.
- **칸반 카드 담당 한눈에**: 미지정=노란색 강조("담당 없음")/배정=직원·대표님 표시.
- **화면 넘침 대응**: 필터(타겟·담당·목표월)+정렬(우선순위·목표월)/진행 중 3칸 집중/발행 완료=최근 몇 개만+전체는 발행이력·성과표/모바일=한 칸씩 좌우.

## 입구 · 권한 · 출구
- **입구 2개(둘 다 데이터 기반)**: 전략 방향/키워드에서. **직접 추가 폐기**(데이터 없이 손으로 만드는 건 "데이터 보고 판단" 원칙 위배). 원고 유입(코드 없음)=기획 아닌 완성 원고라 예외 표시.
- **[이걸로 만들기] 권한 = 관리자·에디터 다 가능**(조사=유튜브 요청이라 owner/admin만/만들기=이미 조사된 것 고르기라 유튜브 요청 없음→개방). 직원 눌러도 **바로 큐 진입(승인 없음)**, 기획한 사람만 기록.
- **출구**: 발행 완료 → 성과표(조망)·호텔(심층)·발행이력(전체 URL)로 성과 회수. 큐엔 최근만. 회수 성과가 다음 기획 근거로 순환.

## 카드 상세
주제·코드·현재 상태/타겟·목표월·우선순위/기획한 사람·원고 담당/넣을 키워드(딸려옴)/전략 근거(분석 멘트)/연결 원고(코드 묶임·드라이브 열기)/발행 상태(완료 시 유튜브 URL·성과 링크)/**[키워드 분석 다시 보기]**(도시 상세로 — 판단 근거 언제든 재확인).

## 출처 배지 (전략 검증 루프)
큐 카드에 출처: **전략(어느 방향)·키워드에서·원고 유입**. 발행 후 성과까지 꼬리표 유지 → 성과표 "출처별 성과 비교"로 "데이터로 고른 게 진짜 더 잘 됐나" 검증.

## 전체 흐름
방향 판(방향 확정) → 방향의 추천 도시 → 도시 상세 분석(판단 근거=키워드 도시 상세) → [이걸로 만들기] → 큐 카드(기획 대기) → [원고 시작]→원고 작성 → 드라이브 완료·코드 매칭→발행 예정 → 올리기 발행→발행 완료 → 성과표·호텔·발행이력.

## 키워드 vs 전략 추천 (관계)
다른 게 아니라 **같은 데이터(키워드 분석) 다른 범위**: 키워드=제약 없이 넓게 탐색/전략=정한 방향으로 좁힘. 같은 도시 데이터·분석 동일.

## 파급 보강 (다른 메뉴)
- **키워드(D-065)**: 도시 상세에 **전략 분석 멘트(AI 요약)** 추가 + 키워드 기본 12개+더보기 30 + **성급 필터**(방향=나라+성급 점프 대응) + [이걸로 만들기] 권한 관리자·에디터. 도시 상세=전략·키워드 두 경로 공유.
- **성과표(D-063)**: **출처별 성과 비교** 섹션(전략/키워드/원고유입, 편수·평균조회·예약·확정률, 펼치면 개별 영상→호텔→예약 공통 드릴).

## 후속 (개발)
6메뉴 설계 완료 → 실제 구현 착수. 하나씩 서비스로 만들어 즉시 사용.
