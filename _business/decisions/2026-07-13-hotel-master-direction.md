# 호텔 마스터(통합 명단) 정석 방향 결정 (2026-07-13)

> 결정자: 대표님 / 정리: 클로드. 후속 작업 = **BL-HOTEL-MASTER** (새 대화창에서 착수).
> 이 문서 하나로 새 세션이 배경·결정·순서를 다 파악할 수 있게 정리.

## 배경 (왜 필요한가)
아고다 예약의 호텔이 아고다ID·이름으로 뒤죽박죽이라, 어느 한 기준으로도 호텔을 정확히 못 묶는다.
- 라이브 실측: **아고다ID 3,903 / 이름 3,200 / 이름+도시 3,192**개.
- 한 호텔이 아고다ID **14개**로 쪼개지거나(예: Red Roof Inn Osaka Namba), 이름 2개로 갈림(ZONK Hakata / ZONK HOTEL Hakata · Galleria 10 리브랜딩).
- 성과표 호텔별·호텔 메뉴·키워드·매니저 대시보드가 전부 "호텔 하나를 정확히 식별"하는 위에서 돌아야 함 → 뿌리가 필요.
- (채널/CID는 이미 정리 완료 — `channel_cid_map` 14개, 예약 100% 정합. 뿌리 중 호텔만 남음.)

## 결정 1 — 호텔 마스터 = 기존 `hotels` 테이블 확장 (새 테이블 만들지 않음)
- `hotels`에 이미 `agoda_hotel_id`·`property_type`(호텔/리조트)·`agoda_match_status`·`hotel_name_local` 등이 있음 → **새로 만들지 않고 이걸 키운다** (중복·혼동 방지 = 대표님 강조).
- 명단은 **하나**. 우리 고유 호텔 번호 하나에 아고다ID·이름 여럿을 매핑. 필드: 대표이름·나라·도시·성급·유형·운영상태(영업/폐업)·옛이름 이력.
- **"따로 만들어 나중에 합치기"가 아니라 "명단 하나를 짓고 칸을 순서대로 채운다"** (대표님 확인: 집 한 채, 방을 순서대로).

## 결정 2 — 채우는 순서 (대표님 우선순위 = 스튜디오 수익이 먼저)
- **먼저 (스튜디오용):** 아고다 예약을 호텔별로 통합 → 우리 호텔 번호 부여. 성과표·키워드·콘텐츠가 이 위에서 즉시 돎. **매니저 없이 예약 데이터만으로 됨.**
- **나중 (매니저 영업 시작 때):** 매니저 가입 호텔의 `agoda_hotel_id` 채우고 예약 매칭 확정(`agoda_match_status`) → 매니저 대시보드 예약·고객분석 살림.
- 같은 명단·같은 번호라 나중 결합 작업 없음, 중복 없음.

## 결정 3 — 정석 방식 (완벽주의 회피)
1. **자동 1차 통합**: 아고다ID + 이름 정규화 + 도시로 컴퓨터가 대부분 자동으로 묶음.
2. **예약 많은 큰 호텔(상위 100~200)부터** 대표님이 눈으로 확정 (여기만 맞으면 실질 정확).
3. **검증 로봇**이 애매한 묶음만 표시 → 점진 정리.
4. 이름변경=옛이름 이력, 폐업=운영상태(아고다 예약만으론 자동판별 어려워 "N개월 예약없음" 힌트+확인).

## 매니저 시스템 영향 (디테일 체크 결과 — 충돌 아님)
- 매니저 시스템은 `hotels` 위에서 돎. 등록·결제·문서·문의 = 정상.
- 예약 매칭(`agoda_hotel_id`·`agoda_match_status`)은 **설계 완비 / 실행 미완**: 가입 호텔 3개 다 `agoda_hotel_id`=None, match=pending/None → **매니저 대시보드 예약·고객분석 현재 빈 상태.**
- 호텔 마스터는 이 미완성을 채우는 것 = 매니저 시스템이 기다리던 다리. 마스터 서면 매니저 예약도 살아남.

## 스튜디오 독립성 (확인됨)
- 스튜디오(성과표·키워드·콘텐츠)는 **아고다 예약 + CID만 있으면 돎**. 매니저 0명이어도 수익. 매니저 완성 안 기다림.
- gohotelwinners 매니저 서비스(대외 영업)는 아직 개발 중 / 스튜디오는 개발 즉시 사용.

## 성과표 현 상태 (이번 세션까지 완료)
- 목적 배너·기간(글로벌표준 8+비교+기간지정)·KPI(증감)·채널별 렌즈(실데이터)·순위표(배지/# 제거, 취소 일반열)·월별 추세·예약 데이터 기준일 배지 = **라이브 완료**.
- 호텔별 렌즈 = 설계 확정(예약 기준 전체+소개 배지, 클릭 시 호텔 메뉴 D-062 세부 재사용) / **구현은 호텔 마스터 후**.
- 영상별 렌즈 = 추적링크 연동 후.
