# 스튜디오 호텔 메뉴 — 최종 UI/UX 확정 + 서비스 방향 (2026-07-13)

> **성격:** D-062(호텔 메뉴=전략 분석 대시보드)의 **구현 레벨 최종 확정**.
> 이 문서는 "무엇을 · 어떻게 보여주고 · **왜** 그렇게 보여주는지"(서비스 방향)를 남긴다.
> 라이브 반영 완료(2026-07-13). 이후 딴 채팅에서 이어갈 때 이 문서가 근거다.

---

## 0. 이 메뉴의 목적 (왜 존재하나)

호텔 메뉴 = **"이미 콘텐츠에 내보낸 호텔 하나의 성과를 모아 전략적으로 판단하는" 대시보드.**
"다음에 뭘 만들지"(키워드·전략 메뉴)와 다르다. 여기는 **이미 노출된 호텔의 성과 확인**이 목적.
핵심 질문: *"이 호텔을 콘텐츠로 (다시) 어떻게 쓸까 — 얼마나 봐서 얼마나 예약됐고, 취소는 어떤가."*

**대원칙 — 정직한 데이터 표기:** 지금 있는 것(아고다 예약=실데이터)과 아직 없는 것(조회·클릭=유튜브 연동 후, 영상별 예약 귀속=추적링크 후)을 **섞지 않고 명확히 구분**한다. 없는 값을 그럴듯하게 지어내지 않고 "연동 후"로 표기한다. 연동되면 같은 자리에서 자동으로 실데이터로 바뀐다.

## 1. 목록 화면

### 현황표(상단 카드 스트립)
- **무엇:** 전체 · 나라·도시 · 호텔·리조트 · 평균 확정률 · 높음(≥85) · 낮음(<50).
- **어떻게:** 카드 = **필터**(전체·높음·낮음·노출안함은 눌러서 거름). **나라·도시 / 호텔·리조트는 지표(숫자)로만** — 클릭 없음.
- **왜:** 나라·도시로 거르는 건 아래 드롭다운에서 하면 "결과 N개"가 바로 뜬다(중복 제거). 규모(전체·나라·유형)를 먼저, 성과(확정률·높음·낮음)를 뒤에 둬서 "얼마나 넓게 깔렸나 → 그중 뭐가 잘되나" 순으로 읽히게.

### 필터·정렬
- **필터:** 나라·도시·성급·유형 드롭다운 + 호텔명·hid 검색. 나라 고르면 "전체 9 → 결과 3개" 자동 표시.
- **정렬:** 추천순 · 확정률순 · 노출순. **추천순(임시) = 노출횟수×2 + 확정률** ("많이 노출됐고 잘 확정되는 호텔" 위로). ⚠️ 임시 공식 — 키워드(수요)·전략 연동되면 "다음에 밀 만한 호텔"로 정식 반영 예정.
- **왜:** 36개국·수천 호텔로 늘어도 "먼저 좁히고 정렬"이 되게. 검색+상위N 방식으로 화면 안 덮음.

### 목록 한 줄
- **무엇:** 호텔명(한글) · 국가(**한글** "대한민국") · 도시 · 성급 · 유형 · 영문명(검색용, 작게) · [소개](대표채널·최고순위·노출N회) → [결과](예약·확정·취소) · 확정률 pill(색: ≥85 초록/≥50 노랑/<50 빨강).
- **왜:** "소개 → 결과" 화살표로 **인과(노출이 예약을 만든다)**를 한 줄에 압축. 확정률 색으로 문제 호텔을 한눈에.

## 2. 상세 (호텔 클릭 시) — 위→아래 순서와 이유

상세는 **인라인 펼침**(목록 줄 아래, 한 번에 하나). 순서 = 돈·계약 → 노출 → 성과 → 취소 타이밍 → 개별 근거.

### ① 유료 계약 카드 (맨 위)
- **무엇:** 결제한 호텔이면 진행 중 계약 카드(회차 "N차 계약" · 운영중/대기 · 선결제일[$·PayPal] · 노출 시작[실제 게시일] · 보장 기한[D-day] · 환불보장 배너) + 종료된 계약 리스트(만료·해지). 미결제면 "소개만 · 유료 계약 없음" 회색 배지.
- **어떻게:** 매니저 결제 화면(D-061)을 그대로 **미러링**하되, **인보이스·영수증·계약서 버튼은 제외**(그건 매니저가 자기 문서 받는 용도, 대표님 성과 화면엔 불필요).
- **왜 맨 위:** "이 호텔이 돈 내는 고객인가"가 성과를 보는 맥락을 정한다. 유료 고객이면 보장 기한(환불 리스크)·재계약 관리가 최우선.
- **왜 종료 리스트:** 만료·해지된 계약도 "계약중 성과"가 남아 **재계약 영업 근거**("이만큼 나왔으니 다시 하시죠")가 된다.
- **데이터·자동화:** 아고다 hid → `hotels.agoda_hotel_id` → `hotels.id` → `v_manager_payments`. 가입 때 아고다 링크로 hid 자동 채움(실패 시 매니저가 링크 붙여넣고 **우리가 확인·승인 후 최종 가입** = 오매칭 방지 게이트, D-053). 결제는 PayPal→v_manager_payments 자동 적재. **→ 가입·결제가 일어나면 카드가 자동으로 "무료→운영중"으로 켜짐.** (지금 결제 0건이라 전부 무료 표기 = 정상)

### ② 노출 이력 (줄 클릭 → 세부 펼침)
- **무엇:** 이 호텔이 나간 영상 목록 = 게시일 · 채널 · 영상 제목 · 이 호텔의 순위(TOP1/2/3). 줄 클릭 시 세부.
- **세부(지금 실데이터):** 상태(발행/대기) · 콘셉트(도시·성급·가격대) · **같이 나간 호텔 TOP1/2/3**(이 영상에 묶여 나간 다른 호텔, 지금 호텔은 강조) · 유튜브 링크(발행 시).
- **세부(연동 후):** 이 영상 조회·클릭(유튜브) · 이 영상 예약(추적링크).
- **"같이 나간 호텔" 클릭 → 그 호텔 전체 상세를 팝업(오버레이)으로.** ✕·바깥·ESC로 닫으면 보던 화면 그대로.
- **왜 팝업(갈아타기 아님):** 목적이 "목록에서 그 호텔 다시 안 찾고 **바로 전체를 분석**". 갈아타면 돌아올 길이 없어 두 번 일이 된다. 팝업은 맥락(보던 호텔)을 안 잃고, 팝업 안에서 또 파고들 수 있다.
- **왜 "같이 나간 호텔":** 한 영상 = 호텔 3개 묶음. "이 호텔을 TOP1로 넣은 영상엔 어떤 호텔이 TOP2·3로 같이 나갔나"(교차 노출)를 보고 전략 판단.
- **데이터:** `publications`(=올리기 메뉴에서 만든 원고). hid_top1/2/3로 이 호텔 매칭. **노출 이력 = 올리기 메뉴의 원고를 "호텔 관점"으로 본 것.** 단 "이 영상이 만든 예약 N건"은 아고다에 영상 정보가 없어 **추적링크 연동 후**에만 가능.

### ③ 성과 종합 — "얼마나 봐서 얼마나 예약됐나"
- **무엇:** 돈 카드 3개(예약금액[초록·손님이 호텔에 낸 돈] · 수수료[대표님만·자물쇠·우리 수익] · 확정률[파랑]) + **전환 칸**(조회 › 클릭 › 예약 › 확정) + 방문·취소·노쇼 요약.
- **전환 칸 드릴다운:** 칸 클릭 → 그 칸 **바로 아래**에 펼침. 예약·확정 → 채널별 → 채널 누르면 개별 예약(매니저 컬럼). 조회·클릭 → "연동 후 채널별→영상별" 안내.
- **왜 전환 칸:** "얼마 봐서 → 얼마 눌러 → 얼마 예약 → 얼마 확정"의 **전환 퍼널**을 한 줄로. 새는 지점을 눈으로 찾게.
- **왜 수수료 대표님만:** 수수료(아고다 커미션)는 우리 수익 → 매니저·에디터에 비노출, owner 화면에서만(D-053).

### ④ 예약·취소 타이밍 (리드타임별)
- **무엇:** 임박(0~7일) · 근접(8~30일) · 먼(31일+) 구간별 확정률·건수. 구간 클릭 → 그 구간 예약만 아래 펼침 + 전체 보기.
- **왜:** "체크인 며칠 전 예약이 잘 깨지나"를 본다. **진짜 취소일(타이밍)은 아고다 원천에 취소일 컬럼이 없어 지금 불가** → 리드타임(예약↔체크인 간격)으로 대체. 아고다 파일에 취소일이 들어오면 진짜 취소 타이밍으로 승격.

### ⑤ 개별 예약 (매니저 컬럼)
- **무엇:** 예약번호·예약일·체크인·체크아웃·박·국가(한글)·기기·금액·상태(방문/고객취소/기타취소) + 수수료(대표님만).
- **왜:** 매니저·admin(D-061·D-053)과 **같은 컴포넌트** 재사용(형태 통일, 수수료 플래그만 다름). 취소건은 아고다가 세부 미제공 → 금액·국가·기기 "—".

## 3. 지금 실데이터 vs 연동 후 (정직 표기 총정리)

| 항목 | 지금 | 켜지는 조건 |
|---|---|---|
| 예약·확정·예약금액·확정률·리드타임·개별예약 | ✅ 실데이터 | — |
| 유료 계약 카드 | 구조 완료(전부 무료 표기) | 가입+아고다매칭+결제 → 자동 |
| 노출 이력·같이 나간 호텔·콘셉트 | ✅ 실데이터 | — |
| 조회·클릭 | "연동 후" | 유튜브 API 연동(D-059) |
| 영상별 예약 귀속("이 영상 예약 N건") | "추적링크 후" | 추적링크(gohotel.win/h/…) |
| 진짜 취소 타이밍(체크인 D-day) | 리드타임으로 대체 | 아고다 파일에 취소일 컬럼 |

## 4. 데이터·연결 요약
- 노출 이력 = `publications`(hid_top1/2/3) — 올리기 메뉴 원고를 호텔 관점으로.
- 예약·확정·취소·금액·리드타임 = `bookings_agoda`.
- 유료 계약 = 아고다 hid → `hotels.agoda_hotel_id` → `hotels.id` → `v_manager_payments`.
- 채널 코드→이름 = HT 호텔이야·HG 호텔이곳·TW 여행능력자들·JP ホテルだ·ZH 世界就是家·KT Koreahotelguide·VN reviewkhachsan.
- 국가 = 영문 저장, 화면은 한글 변환(COUNTRY_KO). 언어 토글(한/영)은 화면 확정 후 일괄 적용 예정(값 유지·라벨만 전환).
