# 예약된 작업 — 수작업 소개 이력 파일 → DB 반영 (BL-EXPOSURE-IMPORT)

**기록일**: 2026-07-14 / 대표님 지시로 예약해둔 작업. 대표님이 파일 주면 착수.

## 배경 (왜 필요한가)
- 현재 "소개함/미소개"는 **영상 원고 데이터(v_content_hotel_exposure, name_in_script)가 있는 호텔만** 소개함으로 판정.
- 대표님은 여러 채널(6개 언어)에서 이미 많은 호텔을 소개했으나, 그 노출 데이터가 시스템에 없음 → 지금 다 "미소개"로 뜸.
- 대표님이 **수작업으로 소개 이력을 정리**해서 파일로 제공 예정. 본문·제목은 못 주고, **노출날짜 · 채널 · TOP3 호텔(3곳)**만.

## 대표님이 줄 파일 형식 (예상)
한 줄 = 한 영상. 컬럼: `노출날짜 | 채널 | 호텔1 | 호텔2 | 호텔3`
- 예: `2024-11-03 · 호텔이야 · Red Roof Inn 오사카 · Caesar Metro Taipei · APA 호텔 남바`
- 엑셀/CSV 등 편한 형식. 호텔은 최대 3곳(TOP3).

## 클로드가 할 작업 (파일 받으면)
1. 파일 파싱: 각 행의 노출날짜·채널·호텔명 3개 추출.
2. **호텔명 → 마스터(hotels.hotel_code) 자동 매칭**: 이름 정규화로 hotels와 대조(오늘 만든 마스터 활용). 채널명 → channel_code 매핑.
3. 매칭 결과:
   - 확실히 맞음 → 소개 이력으로 DB 반영(노출 기록 심기).
   - 애매함 → **확인함(hotel-review)에 올려** 대표님이 "이 호텔 맞아" 확인.
4. 반영되면: 해당 호텔이 **소개함으로 전환** + 성과표 소개 배지 + (가능하면 채널·날짜별 노출로 영상별 성과 일부 연결).

## 반영 위치 (설계 시 결정)
- 소개 이력을 어디에 심을지: 기존 v_content_hotel_exposure 계열 뷰/테이블 구조 확인 후 결정. 원고(name_in_script) 없이 hid·채널·날짜만 있는 노출 레코드를 넣는 방식 설계 필요.
- 소개함 판정 로직(content-performance byHotel의 exposedHids)이 이 새 데이터도 읽게 연결.

## 정직한 한계 (대표님께 전달됨)
- 호텔명 표기 차이(한글/영문/띄어쓰기)로 100% 자동매칭은 안 됨. 잘 맞는 건 자동, 애매한 건 확인함 경유.
- 본문·제목·조회수는 이 파일로 안 들어옴 → 노출 사실·날짜·채널·TOP3만 반영. 영상별 성과의 세밀한 부분은 추적링크 별도.

## 선행 조건
- 대표님이 파일 제공 → 그때 착수. 파일 없으면 이 작업 대기.
