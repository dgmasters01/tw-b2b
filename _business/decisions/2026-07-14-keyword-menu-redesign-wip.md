## 담당자 친화 원칙 (신규)

- 수요·경쟁 막대로 직관 표시.
- [이걸로 만들기] 누르면 "N월 발행 목표로 담김" 미리보기.
- 조사 신선도 배지("3일 전 조사"). 담당자는 조사 권한 없음(owner/admin만) → 조사 UI 숨김.

## 키워드 분석 = 2단계 펼침 (대표님 확정)
- 리스트: 신호로 훑기 (수요/경쟁 막대 + 점수 + 신호점).
- 키워드 클릭 → 요약 펼침 (작은 추이선 "뜨는 중" + 수요/경쟁 세부 + 묶음 근거).
- 그 안 "월별 추이" 클릭 → 큰 차트 (축·값, 기회점수/수요순위/경쟁수 토글). 여러 달 쌓이면 여러 해 겹침.
- 요약 스탯: 살아있는 N · 죽은 M · 이길만함 K · 채택자수/500. 죽은 키워드 "왜 뺐나".

## 키워드 묶음 근거 (신뢰 확보)
- 토큰: 도시·지역·역명·성급 중 하나 필수. 각 키워드 어느 토큰으로 통과했는지 표시.
- 안전장치: 남의 호텔명 제외(트래픽 유출 방지), 띄어쓰기/붙여쓰기 페어(1.5배 갈리면 둘 다).

## 타겟(채널) 선택 — 채널 많아도 안 깨짐
- 하나 크게 보이다 누르면 드롭다운: 검색 + 자주쓰는(빠른전환) + 언어별 그룹 + 운영중/탐색용 구분 + [새 타겟].
- 칩 가로나열 폐기(D-060 계승). 30개·100개여도 OK.

## 겨냥 시점 = 3달 리드타임 (대표님 핵심 지적 → 데이터 검증)
- 메인 기본 = "지금 수요"가 아니라 **"지금 착수 = 3달 뒤 노출 겨냥"**.
- 월 선택: 현재 달부터 6개월 스트립 + 연·월 지정(과거·미래 아무 달) + 지난달 복기.
- 과거 달 = 성과 복기 + 여러 해 겹쳐 계절 확인.

## 착수 기준 = 예약 리드타임 데이터 기반 (핵심 발견 · bookings_agoda 분석)
- **전체(확정 3,766건)**: 중앙값 19일, 1달내 64%, 3달내 92%. 3달+ 일찍 예약 8%뿐.
- 유튜브 SEO 램프업 + 임박 예약 → **여행 시즌 3달 전 발행**해야 예약 90% 커버.
- 기준선:
  - 지금 착수 = 여행 3달 후
  - 곧 늦음 = 2달 후 (예약 시작)
  - 착수 지남 = 1달 이내 (예약 64% 진행 중)
  - 여유 = 4달+ 후
- **도시별(40건+, 중앙값·3달내%)**: 대부분 3달내 92~97% → 3달 공통.
  - 치앙마이 25일·88% = 더 일찍(한 달 앞당김)
  - 호치민 12일·96% / 방콕 14일·92% = 임박 예약(3달 여유)
  - 오사카22·95 / 교토21·97 / 도쿄20·96 / 후쿠오카18·96 / 다낭18·94 / 하노이18·97 / 타이베이17·95 / 세부16·94
- **방식(제안)**: 전체 3달 기준 = 기본 잣대. 도시 상세에서 그 도시 예약 패턴 표시 + 착수 권장 자동 조정. 담당자는 규칙 안 외움.

## 도시별 예약 리드타임 적용안 (2026-07-14 확정 — "노출 경향" → "예약 리드타임"으로 전환)
착수·겨냥 판단 근거를 막연한 경향이 아니라 예약 리드타임 데이터로. 도시별 표는 통째로 안 박고 세 곳에 분산:
- **② 도시 개요**: 각 도시에 예약 성향(더 일찍/보통/임박) + 착수 권장(3달/4달 전) 배지.
- **③ 도시 상세**: 맨 위 "이 도시 예약 패턴" 카드 — 중앙값·3달%·확정 건수 + "왜 이 착수 시점인지" 데이터 설명. (치앙마이 = 4달 전, 12% 놓침 방지)
- **① 메인 선행 준비**: 표 없이 뒤에서 도시별 리드타임 자동 반영 → 착수 상태(늦음/지금착수/여유)만 정확해짐.
