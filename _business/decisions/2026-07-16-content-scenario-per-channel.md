# D-068 — 콘텐츠 1개 : 시나리오 N개 (채널별). 코드는 시나리오의 것 (2026-07-16)

> 결정자: 대표님 / 정리·기록: 클로드
> 상태: **빠져 있던 조각을 박음.** D-066(전략 큐)·D-067(다채널 전개)·2026-07-12(고유코드)가 서로 다른 층을 말하고 있었다.

## 대표님 발언 (원문에 가깝게)
> *"모든 채널에 내보낸다고 연결하지 말고, **그냥 콘텐츠로 작성만** 하면 되지 않을까? 우리가 **각 채널별로 시나리오가 들어가면 시나리오에 고유번호를 넣어주기로 했잖아.** 추천의 경우, 그래서 파악하기로 했잖아."*
> *"유튜브에 **채널별로 업로드되는 내용을 개별로** 뽑잖아. 그러면 **한 개의 호텔이지만 다양한 채널에서 예약이 들어온 것도 알 수 있잖아.**"*

## 이미 기록돼 있던 것 (클로드가 못 찾고 대표님께 되물었음)
`2026-07-12-content-code-all-manuscripts.md` (D-060·D-066 보강):
1. **코드 = 모든 원고의 고유 식별자.** 원고 하나당 코드 하나.
2. **코드 체계 = `채널코드-연번`** (HT-0001 · TW-0002). `content_queue`·`publications` 가 채널별 연번 공간 공유.
3. **자동 발급·자동 부착.** 사람이 손대지 않음.
4. **출처는 코드가 아니라 배지** (일반 / 키워드 / 전략).

→ **코드에 채널이 박히는 건 옳다.** 원고가 채널별이니까. 클로드가 이걸 "충돌"이라 오판했다.

## 🔴 빠져 있던 조각 — 층이 몇 개인가
D-066 원안은 **큐 카드 = 원고 1:1** 을 가정했다(`content_queue.code` · `channel_code` 둘 다 **단수 칸**).
D-067은 *"소재 1개 → 그 언어 전 채널 전부 전개"* 라 했다.
**둘을 겹치면 카드 1개에 코드가 여러 개 필요해진다. 칸이 하나뿐이라 안 들어간다.**

## 확정 — 3층 (2026-07-16)
| 층 | 무엇 | 채널 | 코드 |
|---|---|---|---|
| **① 콘텐츠 (큐 카드)** | *"오사카 4성급, 난바 호텔 3곳을 만든다"* | **없음** | 없음 |
| **② 시나리오 (원고)** | 채널마다 따로 씀 (제목·나레이션·태그가 다름) | **1개** | **`채널코드-연번`** = 여기서 발급 |
| **③ 발행 (publications)** | 유튜브에 실제로 올라간 것 | 1개 | 시나리오 코드 승계 |

- **①에서 채널을 묶지 않는다** (대표님 지시). "전 채널 전개"는 ②에서 시나리오를 N개 만드는 것으로 실현된다. **고르는 화면은 여전히 없다**(D-067 유지) — 자동으로 N개가 생길 뿐이다.
- **코드는 ②의 것이다.** ①에는 코드가 없다. `2026-07-12` 문서의 *"큐 카드에서 발급"* 은 **시나리오를 만들 때 발급**으로 읽는다.
- ①↔② 연결은 **코드가 아니라 카드 id**로 한다. 코드는 ②↔③을 잇는 열쇠다.

## 💡 왜 이 구조라야 하나 — 실측이 증명한다
**한 호텔이 여러 채널에서 예약을 받는다** (2026-07-16 실측, `bookings_agoda` 7,316건):
| | 호텔 수 |
|---|---|
| 예약이 붙은 호텔 | **3,147** |
| 채널 **1개**에서만 | 2,510 (80%) |
| **채널 2개** | **565** |
| **채널 3개 이상** | **72** (최다 **4개**) |

예: `H-00028` Havana Nha Trang — **HT·JP·TW·ZH 4개 채널 · 17건** / `H-00094` Galleria 12 Sukhumvit 방콕 — HT·JP·TW·VN 4채널.

→ **호텔 1개 : 시나리오 N개 : 예약 N갈래.** 코드가 시나리오마다 있어야 *"이 호텔 예약이 어느 채널 어느 원고에서 왔나"* 를 가른다. 카드에 코드 하나면 **4개 채널 예약이 한 덩어리로 뭉개진다.**

## 파급 — 손봐야 할 곳
| 곳 | 지금 | 해야 할 것 |
|---|---|---|
| `content_queue` | `code`·`channel_code` **단수 칸** · 0행 | ①이 되려면 두 칸은 **안 쓴다**(또는 제거). **0행이라 지금 바꾸는 게 가장 쌈** |
| **시나리오 층** | **표 자체가 없음** | ② 신설 필요 (`queue_id` · `channel_code` · `code` · `stage` · 담당) |

## ❌ 하지 말 것
- 큐 카드에 채널을 박는 것 (①에 채널 없음)
- 코드를 카드에 붙이는 것 (코드는 ②의 것)
- **채널을 고르는 화면을 만드는 것** — D-067 유지. 그 언어의 채널 전부에 시나리오가 자동으로 생긴다
- `2026-07-12` 문서를 폐기하는 것 — **살아있다.** 이 문서는 그 위에 층을 나눈 것뿐

- 원고 `HT-0001` 은 **② 시나리오**의 것. 둘은 층이 다르므로 **겹치지 않는다.**

## 보강 2 — **원고 코드 폐기. 코드는 2개다** (2026-07-16 대표님 · 라이브 화면 증거)

> 대표님: *"이 **추천 코드를 채널별로 원고에 붙여 넣으면** 되잖아. 조금씩 시나리오가 다르게 나올 거고, 구글드라이브에 각각의 원고를 넣으면 **여기 고유 채널 cid가 있잖아 그러면 판명할 수 있잖아.** 이렇게 하면 되는 거 아니야? **원고까지 필요없을 것 같은데.**"*

### 실측 — 채널은 **이미 자동으로 판명되고 있다** (라이브 스크린샷 증거)
올리기 화면이 지금 이렇게 말하고 있다:
> *"원고 아고다 링크의 **cid(1932026) 로 채널을 정했습니다: 호텔이야**"*

- 근거 문서: `2026-07-11-studio-upload-menu-full.md:25` — 워드 하이퍼링크(`word/_rels`)에서 cid·hid를 읽는다.
- 드라이브 폴더도 **채널별로 갈려 있다** (TW·HT·HG × 대기·완료·확인필요) — 라이브.
- CID 확정값(대표님 검증): TW 웹 1919025 · 호텔이야 1932026 · 호텔이곳 1946819 · TW 유튜브 1913282.

→ **`R-00123` + cid = (추천 × 채널) 이 유일하게 정해진다.** 원고 코드 `HT-0001` 은 **정보를 하나도 더하지 않는다.**

### 확정 — 코드 **2개**
```
호텔    H-00001    (기존 · 3,182개 발급됨)
추천    R-00001    (신설 · 사람이 워드에 적는 유일한 코드)
원고    폐기        - cid 가 채널을 판명하고, 발행되면 유튜브 영상ID가 곧 그 원고의 이름
```

### `2026-07-12-content-code-all-manuscripts.md` 처리
그 문서의 **목적은 살아있고, 수단만 바뀐다.**
| 그 문서가 원한 것 | 새 방식 |
|---|---|
| 원고 개별 추적 | 발행 후 = `youtube_video_id` / 발행 전 = `R` + 채널 |
| 출처별 비교(일반/키워드/전략) | 배지 그대로 (그 문서 4번 = 코드와 무관하다고 이미 못박음) |
| *"일반 원고는 사람이 절대 손대지 않음"* | **더 잘 지켜진다** — 일반 원고엔 코드가 아예 없다 |
| `채널코드-연번` 으로 사람이 알아봄 | **cid 가 이미 채널을 말해준다.** 연번은 중복 |

→ **폐기 대상 = 코드 체계(2번) · 발급기 · `publications.code` 신설 계획.** 나머지는 유지.

### 흐름 (최종)
```
키워드 [이걸로 만들기] -> 큐 카드 + R-00123 발급
   -> 채널 수만큼 원고를 쓴다 (시나리오는 채널마다 다르다)
   -> 각 원고에 R-00123 한 줄 · 아고다 링크는 원래 넣던 대로
   -> 채널별 드라이브 대기 폴더에 넣는다
   -> 올리기: cid 로 채널 판명(이미 함) + R-00123 읽음(한 줄 추가)
   -> 유튜브 URL 넣고 [발행 완료] -> publications 생성
   -> 전략 큐 카드가 자동으로 "발행 완료" 로
   -> 예약이 채널별로 갈려 들어옴 -> R-00123 으로 다시 합치면 추천 1건의 총 성과
```
**같은 R 이 채널 수만큼 원고에 붙는다.** 중복이 아니라 그게 묶는 끈이다.
