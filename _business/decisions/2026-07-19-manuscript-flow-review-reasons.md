# 2026-07-19 자체 기획 원고 흐름 + 확인 필요 사유 확정 (D-066 보강 · D-067)

> 결정자: 대표님 / 정리·기록: 클로드
> 배경: 전략 큐가 "키워드 추천" 기준으로만 짜여 있어, 코드 없는 자체 기획 원고가 발행 예정에 들어올 때 표시 기준이 없었다. 대표님이 흐름·표시·확인필요 사유를 확정.

## 1. 원고 두 종류 — 단계가 다르다
- **추천 원고 (코드 있음)**: 기획대기 → 원고작성 → 발행예정 → 발행완료 (4단계 전부). 출처=키워드/전략.
- **자체 기획 원고 (코드 없음)**: 이미 완성 원고라 기획대기·원고작성 건너뜀. **드라이브 완료 폴더 유입 → 코드 자동발급(채널-XXXX) → 큐에 바로 「발행 예정」 등록**. 출처=자체 기획(manuscript).
- 즉 자체 기획은 **발행 예정부터 시작**한다.

## 2. 사람이 원고에 넣는 것 = 딱 2개 (나머지 자동)
- **추천 원고**: 코드(TW-XXXX) 한 줄 → [코드 복사] 버튼으로 정확히. + 아고다 링크.
- **자체 기획**: 아고다 링크만. (코드 없음)
- 담당자·도시·성급·채널·TOP1~3은 **사람이 안 적는다** — 시스템이 원고/파일/드라이브에서 자동 추출.

## 3. 자체 기획 카드 표시 (확정)
- **도시·성급·가격** = 원고/파일명에서 자동 추출 (비워두지 않음)
- **채널 배지** = 아고다 링크 cid로 자동 판정 (예: 1932026 → 호텔이야)
- **담당(올린 사람)** = 드라이브에 파일 올린 구글 계정 자동 (원고에 안 적음)
- **최종 업로더** = 발행 완료 시 유튜브 올린 사람 자동 기록 (책임·성과 추적)
- **TOP1~3 호텔** = 원고에서 자동 추출 (조사 근거 대신 "뭘 만든 영상인지")
- **조사·분석 = 없음** ([분석] 버튼 안 붙인다 — 조사 안 거쳤으니 볼 게 없음)
- 출처 배지 = **자체 기획** (키워드/전략과 구분 → 성과표 출처별 비교)

## 4. 추천 카드 vs 자체 기획 카드
| | 추천 (코드 있음) | 자체 기획 (코드 없음) |
|---|---|---|
| 근거 | 조사(참고 호텔·검색어·소개 기준) | 없음 → TOP1~3 호텔로 대체 |
| [분석] | 있음 (도시 상세 팝업) | 없음 |
| 채널 | (발행 후) | 아고다 cid로 바로 |
| 담당 | 지정/맡기 | 올린 사람 자동 |

## 5. 확인 필요 사유 = 3가지 (대표님 확정, 파일명 규격은 폐기)
1. **코드 오타** — 원고에 코드 있는데 큐에 그 코드 카드 없음 → "이 코드에 맞는 카드가 없습니다". (자체 기획은 코드 없으니 해당 없음)
2. **같은 채널 같은 코드 중복** — 하나의 원고를 여러 채널에 쓰는 건 정상. **같은 채널에 같은 코드가 2번**일 때만 확인 필요.
3. **아고다 링크 없음** — 채널 식별 불가 → "원고에 아고다 링크가 없습니다".
- ~~파일명 규격 위반~~ = 폐기 (복잡. 코드 있음/없음 두 갈래로 충분).
- **코드 없음 = 오류 아님** = 자체 기획으로 처리 (확인 필요 아님).

## 6. 확인 필요 → 수정 → 재투입 루프
- 확인 필요 원고는 올리기 [확인필요] 탭에 사유와 함께 뜬다.
- 고쳐서 대기 폴더에 다시 넣으면 다음 확인 때 처리되어 목록에서 빠진다.
- (추후) 코드 오타로 확인 필요면 전략 카드에도 "⚠ 확인 필요" 배지 표시 검토.
