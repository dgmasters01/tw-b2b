# 2026-07-19 자체 기획 원고 흐름 + 확인 필요 사유 확정 (D-066 보강 · D-067)

> 결정자: 대표님 / 정리·기록: 클로드
> 배경: 전략 큐가 "키워드 추천" 기준으로만 짜여 있어, 코드 없는 자체 기획 원고가 발행 예정에 들어올 때 표시 기준이 없었다. 대표님이 흐름·표시·확인필요 사유를 확정.

## 1. 원고 두 종류 — 단계가 다르다
- **추천 원고 (코드 있음)**: 기획대기 → 원고작성 → 발행예정 → 발행완료 (4단계 전부). 출처=키워드/전략.
- **자체 기획 원고 (코드 없음)**: 이미 완성 원고라 기획대기·원고작성 건너뜀. **드라이브 완료 폴더 유입 → 코드 자동발급(채널-XXXX) → 큐에 바로 「발행 예정」 등록**. 출처=자체 기획(manuscript).
- 즉 자체 기획은 **발행 예정부터 시작**한다.

## 2. 사람이 원고에 넣는 것 = 딱 2개 (나머지 자동)
- **추천 원고**: 코드(TW-XXXX) 한 줄 → [코드 복사] 버튼으로 정확히. + 아고다 링크.
- **자체 기획**: 아고다 링크만. (코드 없음)
- 담당자·도시·성급·채널·TOP1~3은 **사람이 안 적는다** — 시스템이 원고/파일/드라이브에서 자동 추출.

## 3. 자체 기획 카드 표시 (확정)
- **도시·성급·가격** = 원고/파일명에서 자동 추출 (비워두지 않음)
- **채널 배지** = 아고다 링크 cid로 자동 판정 (예: 1932026 → 호텔이야)
- **담당(올린 사람)** = 드라이브에 파일 올린 구글 계정 자동 (원고에 안 적음)
- **최종 업로더** = 발행 완료 시 유튜브 올린 사람 자동 기록 (책임·성과 추적)
- **TOP1~3 호텔** = 원고에서 자동 추출 (조사 근거 대신 "뭘 만든 영상인지")
- **조사·분석 = 없음** ([분석] 버튼 안 붙인다 — 조사 안 거쳤으니 볼 게 없음)
- 출처 배지 = **자체 기획** (키워드/전략과 구분 → 성과표 출처별 비교)

## 4. 추천 카드 vs 자체 기획 카드
| | 추천 (코드 있음) | 자체 기획 (코드 없음) |
|---|---|---|
| 근거 | 조사(참고 호텔·검색어·소개 기준) | 없음 → TOP1~3 호텔로 대체 |
| [분석] | 있음 (도시 상세 팝업) | 없음 |
| 채널 | (발행 후) | 아고다 cid로 바로 |
| 담당 | 지정/맡기 | 올린 사람 자동 |

## 5. 확인 필요 사유 = 3가지 (대표님 확정, 파일명 규격은 폐기)
1. **코드 오타** — 원고에 코드 있는데 큐에 그 코드 카드 없음 → "이 코드에 맞는 카드가 없습니다". (자체 기획은 코드 없으니 해당 없음)
2. **같은 채널 같은 코드 중복** — 하나의 원고를 여러 채널에 쓰는 건 정상. **같은 채널에 같은 코드가 2번**일 때만 확인 필요.
3. **아고다 링크 없음** — 채널 식별 불가 → "원고에 아고다 링크가 없습니다".
- ~~파일명 규격 위반~~ = 폐기 (복잡. 코드 있음/없음 두 갈래로 충분).
- **코드 없음 = 오류 아님** = 자체 기획으로 처리 (확인 필요 아님).

## 6. 확인 필요 → 수정 → 재투입 루프
- 확인 필요 원고는 올리기 [확인필요] 탭에 사유와 함께 뜬다.
- 고쳐서 대기 폴더에 다시 넣으면 다음 확인 때 처리되어 목록에서 빠진다.
- (추후) 코드 오타로 확인 필요면 전략 카드에도 "⚠ 확인 필요" 배지 표시 검토.

## 미구현 (로봇=드라이브 워처 BL-YT-DRIVE-WATCH 켜져야 실제 작동)
- 실제 판정·이동·사유 채우기·코드 자동발급·자동 단계전환은 워처 필요.
- 워처 = Vercel env(GOOGLE_DRIVE_SA_KEY·DRIVE_WATCH_FOLDERS) 등록(대표님) 후 작동.

## 8. 담당 규칙 + 확인필요 + 동기화 (07-19 대표님 확정 — 이 설계대로 구현)

### 8-1. 세 종류 담당 (단계마다 주인공이 바뀐다)
- **기획한 사람** = [이걸로 만들기]/[전략으로] 누른 사람 (자동, 끝까지 꼬리표).
- **원고 담당** = 원고 쓰는 사람 (기획대기~원고작성).
- **최종 업로더** = 유튜브 올린 사람 (발행완료, **올리기 연동**).
- 기획대기·원고작성의 주인공 = 원고 담당 / 발행예정·발행완료의 주인공 = 업로더(올리기에서 정함).

### 8-2. 단계별 버튼 (역할 × 담당상태 × 단계)
| 단계 | 담당자(맡은 사람) | 최고관리자 | 다른 사람(에디터·관리자) |
|---|---|---|---|
| 기획대기 | 놓기 · 분석 | 담당 지정 · 분석 · 삭제 | 내가 맡기 · 분석 |
| 원고작성 | **코드복사 · 분석** (놓기 없음) | 담당 변경 · 코드복사 · 분석 · 삭제 | 분석 (표시만) |
| 발행예정 | 분석 | 분석 · 삭제 | 분석 |
| 발행완료 | 성과보기 · 분석 | 성과보기 · 분석 | 성과보기 · 분석 |
- **담당 있으면 다른 사람 못 맡음** — 남이 맡은 카드엔 에디터·관리자에게 [내가 맡기] 안 보임(이름만 표시).
- **담당 변경(재배정) = 최고관리자만.**
- **원고작성엔 [내가 맡기] 없음** (기획대기에서 맡음) · **[놓기]도 없음** (확정됨, 변경은 최고관리자만).
- **원고작성 담당자 = 코드복사·분석만.**
- 정석 = 화면(버튼 안 그림) + 서버(content-queue writer='me'인데 이미 남 담당이면 거부, 최고관리자 아니면) 둘 다 막는다. (현재 미구현 — 서버 안 막힘)

### 8-3. 자체 기획 발행예정 — "올린 사람" 표시 안 함
- 자체 기획 발행예정 카드에서 **"올린 사람" 줄 제거** (대표님 07-19). 업로더는 발행완료 때 "최종 업로더"로만.

### 8-4. 확인필요 = 올리기 [확인필요]에서만 (전략엔 안 넣음)
- **문제 있는 원고(추천·자체기획 공통)는 전부 올리기 [확인필요] 한 곳에서만 본다.** 전략 페이지엔 확인필요 안 띄움(원고작성 배지도 안 함 — A/B 폐기, 대표님 07-19 "올리기에서 무조건 확인").
- 확인필요 표시 = **채널명(cid 판정) · 파일명 · 사유**. 정상 정보(제목·TOP1~3) 못 뽑으니.
- **채널명 표시 필수** — 어느 채널 실수인지 즉시 파악(대표님). cid로 판정된 채널.
- 정상 원고만 제목·채널·TOP1~3 카드로.

### 8-5. 발행예정 ↔ 올리기 동기화 (코드 기준 한 곳 관리, 정상만)
- **정상 원고만 동기화.** 발행예정(content_queue) ⟷ 올리기(publications)를 **코드(TW-XXXX/채널-XXXX)로 잇는다.**
- 올리기에서 업로더 맡음 → 발행예정 카드에도 담당 표시.
- 유튜브 발행 → 양쪽 발행완료 동시.
- 확인필요→정상화 → 올리기 [확인필요]에서 사라짐 + 발행예정에 나타남(양쪽 동기화).

### 8-6. 잘못 넣은 폴더 정리 (대표님 시나리오 07-19)
- **채널은 폴더 위치가 아니라 원고 아고다 cid가 정한다** — 폴더 실수해도 로봇이 cid로 올바른 채널 판정 → 근본 최소화.
- 같은 파일 고쳐 재투입(**file_id 추적**) → 기존 확인필요 자동 갱신(목록에서 빠짐).
- 다른 파일로 새로 넣었으면 → 채널명 보고 잘못된 것 [지우기](올리기 확인필요, 드라이브 휴지통 30일).

### 8-7. 확인필요 사유 (기존 3 + 채널명 표시)
- ①코드 오타 ②같은 채널 같은 코드 중복 ③아고다 링크 없음 (기존).
- + 목록에 **채널명 표시**(cid 판정) 추가.

### 미구현 (드라이브 워처 BL-YT-DRIVE-WATCH 켜져야 실제 작동)
- 8-4·8-5·8-6 전부 워처 필요(원고 유입·확인필요 판정·코드 매칭·동기화·file_id 추적). 워처 = Vercel env(GOOGLE_DRIVE_SA_KEY·DRIVE_WATCH_FOLDERS) 대표님 등록 후.
- 8-2 서버 담당 게이트(남 담당 뺏기 방지)는 워처 없이도 구현 가능.

## 9. 발행완료 카드 디테일 (2026-07-19 대표님 확정 · 프리뷰 `studio-strategy-preview.html` 반영)

### 9-1. 추천·자체기획 카드 = 같은 뼈대 (3덩어리)
- **① 기본**: 제목 · 배지 · 서브메타
- **② 콘텐츠**: 코드줄(TW-XXXX / 채널-XXXX) + **이 영상에 나온 호텔 TOP1·2·3** (추천 카드에도 넣음 — 전엔 빠져 있었음)
- **③ 성과**: 최종 업로더 · 조회 · 예약(준비중) · 수수료(최고관리자만) + 버튼
- **두 카드의 차이 = 딱 2개**: 코드 모양(TW/채널) · 버튼(추천=[분석] / 자체=[원고 열기]). 나머지 전부 동일.

### 9-2. 배지 순서 = ①출처 → ②채널 → ③상태(✓발행됨) 로 통일
- 전엔 추천=출처·발행됨·채널 / 자체=출처·채널·발행됨 으로 어긋나 있었음 → 통일.

### 9-3. 유튜브 = 주소 텍스트 폐기 → **[▶ 유튜브에서 보기] 버튼** (빨강)
- 긴 URL은 잘려서 의미 없음. 버튼줄에 [분석]·[성과 보기]와 나란히. 자체기획도 동일.

### 9-4. 채널명 배지 · 수수료
- 추천 카드에도 **채널명 배지** 추가(발행 후 어느 채널에 올렸나). 여러 채널이면 배지 여러 개.
- **수수료 = 최고관리자(owner) 화면에만** 표시. 다른 역할에겐 서버가 아예 안 보냄(가리는 게 아니라 안 보냄). 지금은 예약과 한 몸이라 "준비중"(추적링크 오면 예약과 함께 켜짐).

### 9-5. 발행완료 하단 담당줄 제거
- 발행완료 주인공 = 최종 업로더(§8-1). "최종 업로더"가 이미 위에 표시되므로 하단 "[최종관리자] 이름" 담당줄 제거(자체기획 카드와 통일).

### 9-6. 펼침 아이콘 = SVG chevron
- `⌄` 문자가 폰트별로 "V"처럼 깨져서 SVG 화살표로 교체.

## 10. [성과 보기] = 이 영상 한 편 성과 화면 (2026-07-19 확정 · perf() 팝업 신설)

> ⚠️ 전엔 [성과 보기]가 **전체 성과표(studio-perf-preview.html)**로 넘어갔음 → **이 영상 한 편 전용** 화면으로 변경(전체로 던지면 그 영상을 다시 찾아야 함).
> 🔴 **성과 보기 = 콘텐츠를 잘 만들기 위한 자리.** 영업(유료 계약 제안) 아님 — 대표님 07-19 *"여기는 콘텐츠를 잘 만들기 위한 스튜디오야."* 유료 계약 제안 넣었다가 뺌.

### 10-1. 화면 5덩어리 (위→아래)
1. **숫자** — 유튜브(영상 조회수) / 아고다=돈(아고다 클릭·예약·확정률·예약금액·수수료). **돈이 맨 위 관점.**
2. **발행 후 흐름** — **발행 기준 주별** 그래프. 조회(막대)는 초반, 예약(점선)은 리드타임으로 뒤따라옴.
3. **이 영상 속 호텔** — 원고 순서 그대로 **TOP1·2·3**(`publications.hid_top1/2/3` = v_content_hotel_exposure rank). 호텔 클릭 → **그 호텔 전체 정보 팝업**.
   - 🔴 **페이지 이동 아닌 겹침 팝업**(성과보기 위에 겹쳐 뜨고 닫으면 성과보기로 복귀). 대표님 07-19 *"호텔 페이지로 넘어가면 불편"*.
   - 팝업 구조(대표님 07-19 확인 · 뺄 것 없음 = 호텔 전체 내용): ①호텔명·나라·성급·유형 ②소개(채널·TOP·횟수)→결과(예약·확정·확정률) ③유료 계약 여부 ④**노출 이력** ⑤성과 종합(예약금액·수수료[owner]·확정률·조회·클릭·예약·확정) ⑥예약·취소 타이밍(리드타임별 임박/근접/먼).
   - 🔴 **노출 이력 = 날짜가 줄 앞에**(대표님 07-19): 발행된 영상=**발행 날짜**, 아직 안 올라간 영상=**미발행(대기)**. 한 호텔의 여러 노출 리듬(언제·간격)을 날짜로 파악. **호텔 메뉴의 호텔 세부 화면도 같은 규칙**(발행 후 날짜).
   - 지금 켜짐 = 예약·확정·금액·확정률(호텔 전체 실데이터) / 준비중 = 조회·클릭(추적링크 후, "연동 후").
4. **다시 소개 가능일** — 발행 +183일(6개월 쿨다운). 이 소재로 또 만들 수 있는 날.

### 10-2. 지금 켜짐 / 준비중 (정직하게 색 구분)
- **지금 켜짐**: 영상 조회수(유튜브 Data API) · 호텔 전체 팝업 · 다시 소개 가능일.
- **준비중(추적링크=영상별 예약 귀속 오면 켜짐)**: 아고다 클릭 · 예약 · 확정률 · 예약금액 · 수수료 · 호텔별 이 영상 기여.
- 🔴 **이 화면 돈 칸 전부를 켜는 열쇠 = 아고다 추적링크(영상 설명란 링크에 추적 꼬리표).** 1순위 후속 과제.

### 10-3. 클릭 = 두 종류 (섞지 말 것)
- **유튜브 썸네일 클릭률(CTR)**: 유튜브 애널리틱스 API(채널 OAuth 인증) 필요. **마케팅 지표.** → 대표님 07-19 **이 화면에서 뺌**(영상 조회수만).
- **아고다 링크 클릭**: 영상 더보기 아고다 추적링크 클릭 = **돈의 앞단계.** 예약·수수료로 이어짐 → 이게 중요.
- 🔴 우리가 연동한 유튜브 API(Data API v3 search)는 **조회수까지만.** CTR·노출수는 안 줌(다른 API).

## 11. 유튜브 주소 수정 = 올리기 [발행됨] 목록 (2026-07-19 확정 · 올리기 손볼 때 구현)

> 배경: 유튜브 주소를 잘못 붙이면 성과가 엉뚱한 영상 조회수를 가져옴. 수정은 **원본이 사는 올리기 한 곳**에서만(전략·성과는 코드로 끌어다 보여줄 뿐). 한 곳에서 고치면 전략·성과 자동 반영.

### 11-1. 올리기에 [발행됨] 탭 신설 (지금 없음 — 대기·확인필요만 있음)
- 발행된 영상이 쌓이므로 **검색·정렬 필수**(1년이면 수백 개).
- **검색**: 제목 · 코드(TW·채널) · 호텔명 · 채널.
- **정렬**: 최신 발행순 · 조회순 · 예약순(최고관리자) · 채널별.
- **거르기**: 채널별 · 기간별.

### 11-2. 주소 수정 권한 = 한 손(최고관리자)만 (대표님 07-19 *"한 사람이 관리하는 게 맞다. 아니면 엉망"*)
- **모두(owner·admin·editor)**: 발행됨 목록 **보기·검색·정렬**.
- **관리자·에디터**: 직접 못 바꿈. 틀린 걸 발견하면 **올바른 새 주소를 붙여넣어 [수정 요청]** → 카드에 "⚠ 수정 요청 중"(중복 방지).
- **최고관리자만**: 요청을 [승인]하면 주소 변경(전략·성과 자동 반영) / [반려] / 요청 없이 **직접 수정**도 가능.
- 요청·승인·반려·수정 전부 **누가 언제** 기록.
- 📌 이 **요청→승인 틀**은 유튜브 주소뿐 아니라 다른 수정(제목 오타 등)에도 재사용. 유튜브 주소부터 시작.
