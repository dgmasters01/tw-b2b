# 결정 D-072 : 다채널 배포 구조 — cid·중복·예약기간 정리

> **제정일**: 2026-07-21
> **누가**: 이지형 대표님 + Claude
> **근거 통찰(대표님)**: *"우리 서비스는 하나의 원고를 다채널로 배포한다. 만들어진 거라도 채널 cid가 다를 거니, 다른 걸로 판단해야 한다. 호텔 예약 기간이 다르고 채널이 다르다. cid는 들어가 있으니 이걸로 구분해야 한다."*
> **상위 규범**: D-067 §8-6(채널은 폴더 아니라 cid가 정한다) · D-068(콘텐츠 코드) · api/youtube.js

## 0. 한 줄 요약

**하나의 콘텐츠를 여러 채널에 배포한다. 채널마다 cid가 다른 건 «정상»이며 «다른 발행»으로 판단한다. cid가 채널을 정한다.**

## 1. 무엇이 «하나»이고 무엇이 «여럿»인가

| 층 | 뜻 | 예 |
|---|---|---|
| **콘텐츠 (하나)** | 호텔 TOP3 + 예약 기간 | 교토 3성급 · 9/10~9/12 |
| **배포 (여럿)** | 그 콘텐츠를 채널마다 올림 | 여행능력자들 / 호텔이야 / 호텔이곳 … |
| **채널마다 다른 것** | 제목·형식·해시태그·**cid** | 같은 내용, 다른 옷 |

같은 호텔·같은 기간을 여러 채널이 다뤄도 **중복이 아니다** — 채널이 다르면 별개 발행이다.

## 2. cid = 채널 도장 (채널 식별자)

- 여행능력자들=1913282 · 호텔이야=1932026 · 호텔이곳=1946819 (웹 1919025).
- **한 원고(한 배포본) = 한 채널 = 한 cid.**
- **채널이 다르면 cid가 다른 게 정상.** 이건 "다른 것"이지 오류가 아니다.
- **채널은 폴더 위치가 아니라 원고의 아고다 cid가 정한다**(D-067 §8-6). 폴더를 잘못 넣어도 cid로 올바른 채널을 판정한다.

## 3. cid 판정 규칙 (언제 «진짜 문제»인가)

| 상황 | 판정 |
|---|---|
| 한 원고 안 링크들의 cid가 **서로 다름** | 🔴 문제 — 수수료가 갈린다. 확인필요 |
| 넣은 cid ≠ 원고 링크 cid (**한 원고 안**) | 🔴 문제 — 확인필요 |
| **채널 A와 채널 B의 cid가 다름** | ✅ 정상 — 다채널 배포. 안 막는다 |
| 모르는 cid | 🔴 문제 — channel_cid_map에 없음 |

즉 "cid 불일치" 오류는 **한 원고 «안»에서 cid가 섞였을 때만** 뜬다. 채널 간 cid 차이는 정상이다.

## 4. 중복 판정 (2026-07-21 수정)

- **키 = `source_filename` + `channel_code`.** (예전엔 파일명만 봐서, 같은 원고를 다른 채널에 올리면 2번째를 중복으로 막던 버그 → 수정 `f4f2552e`.)
- 같은 파일명 **+ 같은 채널** = 덮어쓰기(재투입). 
- 같은 파일명 **+ 다른 채널** = **별개 발행**(막지 않음).
- 같은 유튜브 영상 ID = 중복(그건 진짜 중복).

## 5. 예약 기간 (같은 호텔이라도 다르면 다른 콘텐츠)

- 같은 호텔이라도 **예약 기간이 다르면 다른 콘텐츠**다 (교토 9/10~12 ≠ 교토 10월).
- 콘텐츠 코드 체계는 D-068 참조 (콘텐츠 1개 : 시나리오 N개 · R코드).

## 6. 정리 — 시스템이 지금 하는 판단

1. 채널은 **cid로** 정한다 (폴더 아님).
2. 채널 간 **cid 다름 = 정상**(다른 발행).
3. **cid 불일치 오류 = 한 원고 안에서 cid 섞였을 때만.**
4. **중복 = 파일명 + 채널**이 같을 때만.
5. 예약 기간 다르면 다른 콘텐츠.
