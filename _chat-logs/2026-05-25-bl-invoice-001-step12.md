## ② 이유

- **검증을 1회성 보고가 아닌 영구 자산으로 박은 이유 (헌법 12원칙 12)**: chat-log에 "검증 통과했어요"만 박으면 다음에 누군가 인보이스 시스템 손댈 때 다시 손으로 점검해야 함. 스크립트로 박아두면 변경 후 1번 실행으로 회귀 검증 가능. 향후 CI에 박을 수 있음.
- **API 401 응답을 "PASS"로 박는 이유**: Bearer/x-cron-token 없으면 무조건 차단되어야 함이 정책. 200이 오면 보안 사고. 401 = 정상.
- **사람 손 필요한 영역 분리**: PayPal sandbox 결제는 브라우저 인터랙션이 필수 (PayPal 팝업 → 카드 정보 입력 → 확인). 클로드는 페이지 200 응답 + API 401 차단까지만 자동 검증. 결제 자체는 대표님 5분이면 끝남.
- **CRON_SECRET 환경변수 박지 않고 skip한 이유**: GitHub Secrets에 박혀있어 GitHub Actions에서 실행되며, 로컬 실행 시에만 환경변수 필요. 토큰 메모리에 영구 박지 않음(헌법 11 메모리 사이클 — 운영 진입 시 회전 가능하도록).
- **task status를 done으로 박은 이유**: 100% 도달 = BL 종료. tasks.json `status: in_progress → done` 자동 전환. 별도 BL 분리 항목들(BL-INVOICE-ARCHIVE-DELETE / BL-INVOICE-ENGINE-EXTRACT)은 새 BL로 박혀야 함.
