# 원고 드라이브 폴더 규칙 v1

> 확정일: 2026-07-10
> 대상 드라이브: `1RgCf1M5ujCiX3WP6oahB7tlQrbWvKgsc` (호텔 콘텐츠)
> 관련 작업: `BL-YT-DOCX-INGEST`
> 관련 API: `POST /api/publications`

---

## 0. 왜 필요한가

원고 docx 는 이미지 때문에 3~4MB 다. base64 로 감싸면 4.8MB 가 되어
Vercel 서버리스 payload 한도(4.5MB)를 넘는다. **화면에서 직접 못 올린다.**

대신 **서버가 드라이브를 읽는다.** 에디터는 폴더에 파일을 넣기만 한다.
payload 한도를 아예 타지 않는다. 구글 드라이브 읽기·쓰기는 무료다.

---

## 1. 기존 구조를 바꾸지 않는다

이미 만들어져 있는 것을 그대로 쓴다. **새 폴더 체계를 만들지 않는다.**

```
📁 호텔 콘텐츠
   📁 0. 호텔 컨텐츠 시나리오 규칙      ← 손대지 않음
   📁 1. 나라별 제작 목록               ← 손대지 않음
   📁 1. Scenario                      ← 손대지 않음 (Before / After / Hotel image)
   📁 2. 여행능력자들 쇼츠   (TW)  ⭐ 채널 폴더
   📁 2. 호텔이야           (HT)  ⭐ 채널 폴더
   📁 2. 호텔이곳           (HG)  ⭐ 채널 폴더
```

**채널 폴더 3개가 이미 채널별로 나뉘어 있다.** 여기가 작업 단위다.
`2. 호텔이야` 안에는 이미 `완료` 폴더가 있다. 그 관행을 이어간다.

---

## 2. 채널 폴더 안의 규칙 (3개 채널 동일)

```
📁 2. 호텔이야
   📁 대기            ← 에디터가 원고를 넣는다        (신설)
   📁 완료            ← 처리되면 여기로 옮긴다        (이미 있음)
   📁 확인필요        ← 경고가 있으면 여기로          (신설)
   📁 패키지          ← 업로드용 txt 가 여기 생긴다   (신설)
```

폴더 이름에 번호를 붙이지 않는다. 기존 `완료` 폴더가 번호 없이 만들어졌다.

### 채널 판별

**폴더가 채널을 정한다.** 원고 안의 cid 는 검증용이다.

| 폴더 | channel_code | cid |
|---|---|---|
| `2. 여행능력자들 쇼츠` | TW | 1913282 |
| `2. 호텔이야` | HT | 1932026 |
| `2. 호텔이곳` | HG | 1946819 |

원고 링크의 cid 가 폴더와 다르면 **처리하지 않고 `확인필요` 로 옮긴다.**
어제 `1931762` 를 붙여넣은 사고가 그 자리에서 잡힌다.

---

## 3. 파일이 흐르는 길

```
에디터가 원고 docx 를  📁 대기  에 넣는다
        │
        ▼  (30분마다 자동으로 확인)
   ┌────────────────────────────────┐
   │ 1. 파일명에서 도시·지역·성급 읽기 │
   │ 2. 장부 없으면 실측해서 만들기    │
   │ 3. 제목·본문·키워드·태그 만들기   │
   │ 4. publications 에 저장          │
   └────────────────────────────────┘
        │
        ├─ 경고 0건 ──▶ 📁 패키지 에 txt 생성  →  원고를 📁 완료 로 이동
        │
        └─ 경고 있음 ──▶ 원고를 📁 확인필요 로 이동  +  경고 txt 함께
```

**파일을 옮기는 것이 핵심이다.** 그래야 같은 원고를 두 번 처리하지 않는다.
`대기` 폴더가 비어 있으면 할 일이 없다는 뜻이다.

## 4. 패키지 txt 모양

파일명: `{원고 파일명}_업로드.txt`

```
[제목]
일본 후쿠오카 하카타 호텔 추천 TOP3, 10만원대 후쿠오카 가성비 숙소

[영상 파일명]
후쿠오카호텔_후쿠오카호텔추천_하카타호텔_하카타호텔추천_후쿠오카가성비호텔_호텔이야

[설명란]
(본문 전문 — 아고다 링크 3개 포함)

[키워드 · 415자]
후쿠오카여행, 후쿠오카 여행, ...

[해시태그]
#하카타호텔 #후쿠오카호텔추천 ...

[챕터]
00:00 오늘의 조건
00:09 TOP3 KOKO 호텔 하카타 스테이션
...
```

유튜브 스튜디오에 **그대로 복사·붙여넣기** 하는 파일이다.
`studio.html` 에서도 같은 값을 복사 버튼으로 준다. 둘 중 편한 쪽을 쓴다.

## 5. 확인필요로 가는 경우

| 사유 | 예 |
|---|---|
| cid 가 폴더 채널과 다름 | `2. 호텔이야` 에 cid=1946819 원고 |
| 아고다 링크 3개가 안 나옴 | hid 를 못 뽑음 |
| 파일명 규격 위반 | 도시·성급을 못 읽음 |
| `[원고 확인 필요]` 발생 | 원고에 없는 값 |
| 키워드가 400자 미만 | 실측 후보 부족 |

**지어내지 않는다.** 경고를 파일로 남기고 사람이 본다.

## 6. 서버가 드라이브를 읽는 법

구글 **서비스 계정** 하나를 만들고, 드라이브 폴더를 그 계정에 **뷰어+편집자**로 공유한다.
개인 계정 로그인이 아니라 서버 계정이다. 만료되지 않는다.

```
GOOGLE_SERVICE_ACCOUNT_JSON   Vercel env 에 넣는다
DRIVE_ROOT_FOLDER_ID          1RgCf1M5ujCiX3WP6oahB7tlQrbWvKgsc
```

돈은 들지 않는다. 드라이브 API 는 무료다.

### 도는 주기

`vercel.json` 의 cron 으로 **30분마다** 확인한다.
`대기` 폴더가 비어 있으면 아무 일도 하지 않는다. 호출 비용 0.

## 7. 안 하는 것

- 원고 내용을 고치지 않는다. 읽기만 한다.
- `1. Scenario`, `0. 규칙`, `1. 나라별 제작 목록` 폴더는 건드리지 않는다.
- 처리 실패한 원고를 지우지 않는다. `확인필요` 로 옮길 뿐이다.
- 이미 `published` 인 원고는 다시 처리하지 않는다 (`api/publications.js` 가 409 로 막는다).

---

## 8. 에디터가 하는 일

```
1. 완성한 원고 docx 를  📁 대기  에 넣는다
2. 30분 뒤  📁 패키지  에서 txt 를 받거나,  studio.html 에서 복사 버튼을 누른다
3. 유튜브에 올린다
4. studio.html 에 유튜브 주소를 붙여넣고 [발행 완료] 를 누른다
```

**4번을 누르면 `publications.status` 가 `published` 로 바뀌고,
`videos` 행이 자동 생성되어 호텔 매니저 화면에 뜬다.**
