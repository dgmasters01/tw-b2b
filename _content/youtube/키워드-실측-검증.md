## 2. 실측이 뒤집은 것 두 가지

### 2-1. 골든셋의 호텔명 5개 중 3개가 죽은 키워드였다

문서 §5 규칙("호텔명은 자동완성 검증 후에만 채택")을 코드가 실제로 집행하자 이렇게 나왔다.

| 호텔명 | 자동완성 | 경쟁 | 판정 |
|---|---|---|---|
| 컴포트 호텔 나고야 신칸센구치 | 1위 | 404 | ✅ 채택 |
| 소테츠 프레사 인 나고야 | 1위 | 527 | ✅ 채택 |
| 메이테츠 호텔 나고야 에키마에 | **없음** | 362 | ❌ 제거 |
| 호텔 몬테레 그라스미어 오사카 | **없음** | 379 | ❌ 제거 |
| 히요리 호텔 오사카 난바 스테이션 | 1위 | 626 | ✅ 채택 (+축약형 `히요리호텔 오사카`) |
| 페어필드 바이 메리어트 오사카 난바 | 1위 | 700 | ✅ 채택 (+축약형) |
| 코코 호텔 신주쿠 요츠야 | **없음** | 436 | ❌ 제거 |
| 호텔 마이스테이 니시 신주쿠 | **없음** | 542 | ❌ 제거 |
| 소테츠 프레사 인 히가시 신주쿠 | 1위 | — | ✅ 채택 |

> `호텔 몬테레 그라스미어 오사카` 를 치면 유튜브는 `호텔 힐러리스 신사이바시` 를 권한다.
> 아무도 그렇게 치지 않는다는 뜻이다. 태그란에 넣어도 아무 일도 일어나지 않는다.
> 호텔명은 본문에서 3회 반복(링크·챕터·주변명소)되므로 태그란에서 빠져도 손실이 작다.

**대표님 결정 필요 없음.** 문서 규칙 그대로 집행한 결과다. 다만 골든셋 3개 문서(§8·§9)의
호텔명 줄이 실측과 어긋나 있으니, 다음 원고 작업 때 그 줄은 무시하면 된다.

### 2-2. "3어절 이상은 띄어쓰기가 갈린다"는 틀렸다

나고야 19쌍 전수 측정. **갈린 것은 3쌍뿐.**

| 쌍 | 띄어쓰기 | 붙여쓰기 | 배수 | 판정 |

| 나고야 숙소 추천 | 70,948 | 8,002 | 8.9배 | **갈림** |
| 나고야 가성비 숙소 추천 | 7,723 | 2,326 | 3.3배 | **갈림** |
| 나고야 여행 숙소 | 13,638 | 119,298 | 0.1배 | **갈림** |
| 나고야 호텔 추천 | 55,312 | 55,335 | 1.0배 | 동일 |
| 나고야 호텔 위치 | 5,136 | 5,145 | 1.0배 | 동일 |
| 나고야 호텔 조식 | 4,623 | 4,695 | 1.0배 | 동일 |
| 나고야 가성비 호텔 | 9,791 | 9,607 | 1.0배 | 동일 |
| 나고야 가성비 호텔 추천 | 9,549 | 9,561 | 1.0배 | 동일 |
| 나고야 대욕장 호텔 | 906 | 909 | 1.0배 | 동일 |
| 나고야 숙소 위치 추천 | 2,406 | 2,431 | 1.0배 | 동일 |
| 나고야역 근처 호텔 | 5,413 | 4,148 | 1.3배 | 동일 |

| (외 7쌍) | | | 1.0배 | 동일 |

어절 수로는 알 수 없다. `pair()` 로 재고 나서 넣는다. 문서 두 곳을 고쳤다.
(`키워드-실측.md` §4 / `여행능력자들.md` §7)

### 2-3. 다만 규칙성은 있다 — `가성비` 는 갈린다

4개 도시 53쌍 전수 측정 결과, **`가성비` 가 들어간 어형은 거의 예외 없이 갈렸다.**

| 쌍 | 배수 |
|---|---|
| 오사카 가성비 호텔 / 오사카가성비호텔 | 31.4배 |
| 오사카 호텔 가성비 / 오사카호텔가성비 | 17.0배 |
| 오사카 가성비 숙소 추천 | 6.8배 |
| 도쿄 호텔 가성비 | 5.4배 |
| 도쿄 숙소 가성비 | 3.7배 |
| 후쿠오카 텐진 호텔 가성비 | 3.5배 |
| 도쿄 가성비 호텔 | 3.2배 |
| 시부야 숙소 가성비 | 3.0배 |

반대로 `호텔 추천` `호텔 위치` `근처 호텔` 은 거의 언제나 1.0배다.
그래도 **패턴을 코드에 박지 않았다.** 재는 비용이 1초다. 재고 넣는다.

### 2-4. 갈린다고 다 넣으면 안 된다

`하카타역 대욕장 호텔`(2,246) vs `하카타역대욕장호텔`(**5**) → 449배 갈림.
하지만 경쟁 5는 "그 어형으로 만든 영상이 사실상 없다"는 뜻이고, 자동완성에도 안 뜬다.
아무도 그렇게 치지 않는다. **붙여쓰기형 경쟁 500 미만은 버린다** (문서 §4).

## 3. 실측으로 새로 찾은 어형 (예: 나고야)

슬롯 템플릿에서는 절대 나오지 않는 것들이다.

| 어형 | 자동완성 | 경쟁 | 기회점수 |
|---|---|---|---|
| 나고야 대욕장 호텔 | **1위** | 906 | 3.38 |
| 나고야 대욕장 숙소 | 2위 | 833 | 3.08 |
| 나고야 숙소 위치 추천 | 1위 | 2,406 | 2.96 |
| 나고야역 근처 숙소 | 2위 | 1,531 | 2.83 |
| 나고야 호텔 조식 | 1위 | 4,623 | 2.73 |
| 나고야역 근처 호텔 | 1위 | 5,413 | 2.68 |
| 나고야역 가성비 숙소 | 1위 | 9,300 | 2.52 |

`나고야 대욕장 호텔` 은 자동완성 1위인데 경쟁이 906개다. 사실상 빈 자리다.

## 4. 안전장치 (지어내지 않는다)

장부가 없는 도시(후쿠오카)로 불러본 결과 — 슬롯만 327자를 내고 **전부 경고로 알린다.**

```
· 실측 장부 keywords/후쿠오카.csv 가 없습니다. 슬롯 키워드만으로 냈습니다.
· 호텔명 "테스트 호텔 하카타" 은 실측 장부에 없습니다. 검증 못 했으므로 뺐습니다.
· 3어절 이상 9개는 붙여쓰기 경쟁을 못 재서 넘겼습니다. 어절 수만으로는 갈리는지 알 수 없습니다.
· 키워드가 327자로 하한(400자)에 못 미칩니다. 지어내지 않고 그대로 냅니다.
```

죽은 키워드 회귀 시험: `요츠야 호텔`(자동완성 0, 경쟁 6,545) → **도쿄 결과에 없음.** ✅

## 5. 새 도시를 붙이는 법

```bash
# 1) 자동완성 + 경쟁 수집
python3 api/_lib/kwtool.py harvest "후쿠오카 호텔" --depth 1 > 후보.txt
python3 api/_lib/kwtool.py analyze --file 후보.txt --csv _content/youtube/keywords/후쿠오카.csv

# 2) 3어절 이상은 갈리는지 확인
python3 api/_lib/kwtool.py pair "후쿠오카 가성비 숙소 추천"

# 3) 커밋하면 끝. 코드는 안 고친다.
```

급하면 API 를 `{"live": true}` 로 부르면 그 자리에서 유튜브에 묻는다.
느리다(어형당 1~2초). 평소엔 장부를 쓴다.
