# 유튜브 키워드 실측 표준 v1

> 확정일: 2026-07-10
> 적용: 여행능력자들 · 호텔이야 · 호텔이곳 (한국어 4채널 공통)
> 코드: `api/_lib/kwtool.js` (측정) · `api/_lib/youtube-keywords.js` (조립)
> 원본 CLI: `api/_lib/kwtool.py`

---

## 0. 왜 이 문서가 있나

슬롯 템플릿만으로 만든 키워드는 **300~335자**에서 멈춘다.
규격은 400~450자다. 모자란 100자는 **유튜브 자동완성에 실제로 존재하는 어형**이라
코드가 지어낼 수 없다. 예: `우메다 대욕장 호텔`, `나고야 호텔 조식`, `오사카 숙소 난바`.

그래서 채우는 방법을 여기에 규정한다. **코드는 규칙을 갖지 않는다. 이 문서를 읽는다.**

---

## 1. 두 가지 측정값

| 이름 | 출처 | 의미 | 성질 |
|---|---|---|---|
| 자동완성 순위 | `suggestqueries.google.com/complete/search?client=firefox&ds=yt` | 수요 프록시 | **띄어쓰기를 구분한다** |
| 경쟁 영상 수 | 유튜브 검색결과 HTML의 `"estimatedResults"` | 경쟁 프록시 | **띄어쓰기를 정규화한다** · ±20% 노이즈 |

기회점수 = `수요 ÷ log10(경쟁)`
수요 = `11 - 자동완성순위` (1위=10점, 10위=1점, 없으면 0점)

---

## 2. 살아있는 키워드 / 죽은 키워드

```
살아있다  = 자동완성 목록에 그 어형이 문자 그대로 존재한다
죽었다    = 존재하지 않는다 → 버린다
```

- 예: `요츠야 호텔` = 죽은 키워드 (자동완성 0)
- 예외 1 — **도시 대표어**(`오사카 호텔` 226만): 이길 수 없으나 비용 0이므로 넣는다.
- 예외 2 — **성급 어형**(`나고야 3성급 호텔`): 자동완성이 없어도 AI 검색용으로 1개만 유지한다.

이 두 예외 외에는 죽은 어형을 넣지 않는다.

## 3. 확장 어형 채택 규칙 (100자를 채우는 방법)

### 3-1. 씨앗
```
[도시] 호텔 / [도시] 숙소 / [도시] 여행 / [도시] 호텔 추천 / [도시] 숙소 추천
[역명] 호텔 / [역명] 숙소
[지역N] 호텔 / [지역N] 숙소
```

### 3-2. 허용 토큰 (화이트리스트)

후보 어형의 **모든 어절**이 아래 집합 안에 있어야 채택한다.
하나라도 밖에 있으면 버린다. 원고에 없는 값을 지어내지 않기 위한 장치다.

**원고에서 오는 토큰**: `[나라]` `[도시]` `[지역1~3]` `[역명]` `[N성급]`

**일반 토큰** (이 줄을 고치면 결과가 바뀐다):
```
호텔, 숙소, 추천, 위치, 가성비, 근처, 조식, 예약, 여행, 자유여행, 대욕장, 도보
```

추가로 두 조건을 더 건다.
- **2어절 이상**일 것 (`나고야` 한 단어는 안 넣는다)
- **`[도시]`·`[지역]`·`[역명]` 중 하나를 반드시 포함**할 것
  (`일본 숙소 추천` 은 토큰이 전부 허용이지만 우리 도시가 없다 → 버린다)

### 3-3. 자동 배제 예시

| 후보 | 판정 | 이유 |
|---|---|---|
| `오사카 숙소 난바` | 채택 | 전 어절이 허용 토큰 |
| `나고야 호텔 조식` | 채택 | 전 어절이 허용 토큰 |
| `나고야역 근처 호텔` | 채택 | 전 어절이 허용 토큰 |
| `오사카 5성급 호텔 추천` | 배제 | 원고는 4성급 |
| `오사카 숙소 추천 4인` | 배제 | `4인`이 원고에 없음 |
| `나고야 호텔 리젠트` | 배제 | 남의 호텔 브랜드 |
| `오사카 여행 브이로그` | 배제 | `브이로그`가 허용 밖 |
| `나고야 료칸 숙소` | 배제 | `료칸`이 허용 밖 |

### 3-4. 정렬·투입
기회점수 내림차순으로 하나씩 넣는다. 상한(450자)을 넘기면 그 자리에서 멈춘다.

## 4. 띄어쓰기 / 붙여쓰기 페어

- **2어절**은 유튜브가 같게 취급한다 (`오사카 호텔` = `오사카호텔` = 226만). 하나만 넣어도 되지만 태그란은 표기 변형 보완이 목적이므로 **슬롯이 지시한 페어는 그대로 유지**한다.
- **3어절 이상이라고 전부 갈리지는 않는다.** 재보지 않고 둘 다 넣으면 자리만 버린다.

> ⚠️ v1 정정 (2026-07-10 실측). 여행능력자들.md §7 은 "3어절 이상은 실제로 갈림"이라고 적었으나,
> 나고야 19쌍을 전수 측정한 결과 **갈린 것은 3쌍뿐**이었다. 나머지 16쌍은 1.0배(동일 취급).
>
> | 쌍 | 띄어쓰기 | 붙여쓰기 | 배수 | 판정 |
> |---|---|---|---|---|
> | 나고야 숙소 추천 | 70,948 | 8,002 | 8.9배 | **갈림** |
> | 나고야 가성비 숙소 추천 | 7,723 | 2,326 | 3.3배 | **갈림** |
> | 나고야 여행 숙소 | 13,638 | 119,298 | 0.1배 | **갈림** |
> | 나고야 호텔 추천 | 55,312 | 55,335 | 1.0배 | 동일 |
> | 나고야 가성비 호텔 | 9,791 | 9,607 | 1.0배 | 동일 |
> | 나고야역 근처 호텔 | 5,413 | 4,148 | 1.3배 | 동일 |
>
> 어절 수는 예측력이 없다. **반드시 `pair()` 로 재고 나서 넣는다.**

- 판정: `pair()` 로 두 어형의 경쟁을 재서 **1.5배 이상 벌어지면 갈린 것** → 둘 다 넣는다.
- 다만 붙여쓰기형의 **경쟁이 500 미만이면 넣지 않는다.** 그 어형을 쓴 영상이 사실상 없다는 뜻이고,
  그런 어형은 자동완성에도 안 뜬다. 아무도 그렇게 치지 않는다.
  (예: `하카타역대욕장호텔` 경쟁 5 → 449배 갈렸지만 죽은 어형이다)
- 갈리지 않으면 띄어쓰기형 하나만 넣는다. (자리를 아낀다)
- 측정한 붙여쓰기 어형은 장부에 `자동완성순위 = 붙여쓰기` 행으로 남긴다. 다음번엔 안 재도 된다.

### 관찰된 패턴 (2026-07-10, 4개 도시 · 53쌍)

**`가성비`가 들어간 어형은 거의 항상 갈린다.**

| 쌍 | 배수 |
|---|---|
| 오사카 가성비 호텔 / 오사카가성비호텔 | 31.4배 |
| 오사카 호텔 가성비 / 오사카호텔가성비 | 17.0배 |
| 오사카 가성비 숙소 추천 / 오사카가성비숙소추천 | 6.8배 |
| 도쿄 호텔 가성비 / 도쿄호텔가성비 | 5.4배 |
| 도쿄 숙소 가성비 / 도쿄숙소가성비 | 3.7배 |
| 후쿠오카 텐진 호텔 가성비 / 붙여쓰기 | 3.5배 |
| 도쿄 가성비 호텔 / 도쿄가성비호텔 | 3.2배 |
| 시부야 숙소 가성비 / 시부야숙소가성비 | 3.0배 |

반대로 `호텔 추천` `호텔 위치` `근처 호텔` 은 거의 언제나 1.0배(동일 취급)다.
그래도 **패턴을 규칙으로 굳히지 않는다.** 재는 비용이 1초다. 재고 넣는다.

## 5. 호텔명 규칙

1. `suggest(호텔명)` 결과에 **그 이름이 문자 그대로 있어야** 채택한다.
2. 자동완성이 함께 돌려준 어형 중, **모든 어절이 `호텔명 토큰 ∪ [도시] ∪ [지역] ∪ [역명]` 안에 있는 것**만 축약형으로 추가 채택한다.
3. 그 외는 **다른 호텔**이므로 금지한다.
4. 자동완성이 `[도시] + 호텔명` 어형을 따로 준다면 **그 어형을 우선 채택**한다.
   같은 이름의 다른 호텔과 섞이지 않게 도시가 묶어준다.
   (예: `호텔 라 포레스타` 경쟁 37,813 — 국내 동명 호텔·웨딩홀이 섞인다.
    자동완성이 주는 `후쿠오카 호텔 라 포레스타` 를 쓴다)
5. 원고가 영문으로 적은 이름을 **코드가 한글로 음역하지 않는다.** 그건 지어내기다.
   (예: 원고 `KOKO 호텔 하카타 스테이션` → 유튜브는 `후쿠오카 코코호텔` 을 쓴다.
    코드는 이 연결을 만들 수 없다. 경고를 내고 뺀다. 사람이 원고를 고치면 된다)

| 호텔명 | 자동완성이 준 어형 | 판정 |
|---|---|---|
| 히요리 호텔 오사카 난바 스테이션 | `히요리 호텔 난바 스테이션` | 채택 (토큰 부분집합) |
| 호텔 몬테레 그라스미어 오사카 | `호텔 몬테레이 오사카` | **금지** (`몬테레이`는 다른 호텔) |
| 소테츠 프레사 인 나고야 | `소테츠 프레사 인 오사카` | **금지** (`오사카`는 이 원고의 도시가 아님) |
| 메이테츠 호텔 나고야 에키마에 | (자기 자신 없음) | **금지** (죽은 호텔명) |

호텔명은 본문에서 각 3회 반복되므로 태그란에서 빠져도 손실이 작다.
반대로 **남의 호텔명을 넣으면 그 호텔 영상에 트래픽을 넘긴다.** 금지가 이득이다.

## 6. 조립 순서 (youtube-keywords.js)

```
1. 슬롯 템플릿 렌더            → 기반 (300~335자)
2. 죽은 호텔명 제거            → §5
3. 확장 어형 투입 (기회점수順)  → §3
4. 3어절 페어 보강             → §4
5. 400자 도달 / 450자 정지     → 채널 문서 §5 규칙 1
6. 근거표(evidence) 동봉       → 순위·경쟁·기회점수·채택사유
```

400자에 못 미치면 **경고를 남기고 그대로 낸다.** 억지로 지어내지 않는다.

## 7. 실측 장부

`_content/youtube/keywords/[도시].csv` 에 쌓는다.

| 열 | 뜻 |
|---|---|
| 키워드 | 어형 |
| 자동완성순위 | 1~10, 없으면 `없음` |
| 경쟁영상수 | estimatedResults |
| 기회점수 | 수요 ÷ log10(경쟁) |
| 살아있나 | `○` / `✗ 죽은키워드` |
| 측정일 | YYYY-MM-DD |

- API는 **CSV를 먼저 본다.** 없으면 라이브 측정한다 (`live: true` 또는 `YT_LIVE_MEASURE=1`).
- CSV는 갱신 주기 **90일**. 경쟁 수치는 시점에 따라 크게 흔들린다 (나고야역 호텔: 2026-07-09 7,989 → 2026-07-10 59,998).
- 순위(수요)는 안정적이다. **순위를 믿고, 경쟁은 자릿수만 믿는다.**

## 8. 갱신하는 법

```bash
python3 api/_lib/kwtool.py analyze --file 키워드목록.txt --csv _content/youtube/keywords/오사카.csv
python3 api/_lib/kwtool.py pair "오사카 가성비 호텔"
python3 api/_lib/kwtool.py harvest "오사카 호텔" --depth 1
```
