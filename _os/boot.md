# TW OS Boot Sheet

> **매 채팅 시작 시 Claude가 가장 먼저 읽는 1개 파일.**
> 이거 1개로 작업 90%가 가능하다. 깊이 필요 시에만 추가 fetch.

---

## 1. 헌법 1조 + OS 핵심 (5줄)

- **대표님은 결정만 한다. 시스템이 모든 실행과 검증을 한다.**
- 각 프로젝트는 완전 독립 (gohotelwinners / CEYLON / 호텔이야 별개).
- OS는 동일 운영 매뉴얼 + 봇 5종 + 메뉴 영역 표준만 공유.
- 틀(영역: DOCS / TOOLS / OS-CORE / SYSTEM)은 강제, 내용(메뉴)은 자유.
- 헌법 본문은 `OPERATIONS_CHARTER.md`, 200줄 이하 강제 (charter-length-bot 감시).

## 2. 11대 원칙 한 줄 요약 (11줄)

1. **단일 진실** — 모든 자산은 클라우드(GitHub + Vercel + Supabase)에.
2. **무인 실행** — 알림·배포·검증·리포트는 사람 없이 자동.
3. **어디서든 도달** — 핸드폰 한 대로 모든 작업 가능.
4. **전수 추적** — 누가/언제/왜/무엇을 영구 기록.
5. **무인 검증** — 검증 통과한 것만 보고. 실패는 자동 롤백·차단.
6. **AI 가독성** — 사람용 + AI용 이중 형식. 인덱스 + 분할 + 자동 갱신.
7. **상태 가시성** — 작업 상태 5초 내 파악 + 멈춘 위치 영구 저장.
8. **통합 관리** — 현황표 + 비즈니스 독스 + 갤러리 자동 동기화.
9. **가역성** — 모든 결정·작업 되돌릴 수 있음. 이중 백업.
10. **자기 강제** — 헌법은 시스템이 Claude에게 강제 (기억 의존 금지).
11. **메모리 사이클** — 토큰은 단계에 따라 충전·방전. 운영 모드 진입 시 비움.

## 3. 매 응답 의무 2가지

- **첫 줄 (부칙 12):** `[작업 소요: 약 X분 / N단계 / 변경 파일: ...]`
- **두 번째 줄 (부칙 13):** `🚦 ✅ 이 채팅 진행 가능` 또는 `🚦 ⚠️ 새 채팅 권장`
- 디테일·예시·판단 기준: `_os/playbook/work-time-reporting.md`, `_os/playbook/chat-routing.md`

## 5. 작업 시작 전 자가 검증 (헌법 11개 질문 요약)

| # | 질문 |
|---|------|
| 1 | 클라우드(GitHub)에만 존재하는가? |
| 2 | 사람 손 없이 자동 실행되는가? |
| 3 | 핸드폰만으로도 가능한가? |
| 4 | 작업 기록이 영구 보존되는가? |
| 5 | 시스템이 자동 검증하는가? |
| 6 | 다음 세션 Claude도 맥락 파악 가능한가? |
| 7 | 작업 상태가 5초 안에 파악되는가? |
| 8 | 현황표/비즈니스 독스/갤러리 동기화되는가? |
| 9 | 되돌릴 수 있는가? (이중 백업?) |
| 10 | 헌법을 자동 로딩하는가? |
| 11 | 메모리 사이클 안에 있는가? |

11개 다 통과 못 하는 제안은 절대 드리지 않는다. 막혀도 우회 안 한다. **완전한 디테일은 `OPERATIONS_CHARTER.md`.**
