## 0. 대전제 (반복 금지 — 이미 확정)
- 보는 사람 = **B2B 호텔 담당자/사장** (일반 여행자 ❌). 영어 가능.
- **언어 = 한국어 · 영어만.** 일/중 확장 안 함 (담당자 영어 됨).
- 핵심 가치 = **예약(real)**. 조회수는 보조(vanity). [BUSINESS.md 일치]
- 시장 = 한국 여행자 → 일본·태국·베트남·대만 호텔 (한국 호텔 아님, 87건뿐). 1차 강점.
- 데이터: 예약·노출수 **월 1회 수동(엑셀) 갱신**. 실시간 ❌. 숫자엔 "누적/기준시점" 표기. [D-BOOKING-DATA-FLOW]
- 톤: **"이미 잘 되고 있다"는 자신감.** "성장 중" 같은 약한 말 ❌. 단점은 언급 안 함.
- **기존 라이브 index.html 디자인/구조는 살리고 카피·강조만 교체.** 바닥부터 재작성 ❌.

## 1. 메인 섹션 순서 (마케팅 데이터 검증 = stop→trust→value→objection→CTA)
1. **히어로** — 스크롤 멈추기
   - 헤드라인: "지금도 우리는 호텔의 예약을 만듭니다. 이제, 당신의 호텔 차례입니다." (현재진행+초대)
   - 받침: "누적 900만 도달 · 3,774건 예약 · $854,000 매출"
   - 우측 카드: 누적 성과(도달 9M+/예약 3,774/매출 $854K), "검증됨"
   - 숫자전략: 큰 숫자(9M)로 시선 → 예약·매출로 신뢰 받치기
   - CTA: "내 호텔 등록하기 — $200" / 신뢰배지: 6개월보장·$200 1회·0건 100%환불
2. **실적 슬라이드** (히어로 바로 아래 = 프라임 자리) — 신뢰
   - 다양한 나라·도시·성급 호텔이 **자연스럽게 흐름**(필터/골라보기 ❌, marquee ✅)
   - 표시 = **호텔 실명 + 건수** (도시/성급도 섞어 다양함 과시). "아고다/채널노출" 문구는 슬라이드에 ❌ → 설명 섹션으로 분리.
   - 기준 = **예약 많은 상위 호텔만**(1~2건 제외). [마케팅 검증: 거대 숫자 H1 +6~29%, 구체·실명·수치화가 최강 사회증거]
   - 실명 리스크: 호텔 좋게 보여주는 건 OK(악평 아님). 단 "우리가 만들었다" 단정보다 사실 표기. 동의받은 곳은 안전.
3. **작동 원리** — 가치 설명 (기존 3단계 활용: 가입&인증 → 결제&일정 → 송출). "내가 뭘/너희가 뭘" 명확.
4. **가격·포함 내역** — $200에 영상제작·8채널·6개월 포함 명확(투명성).
5. **환불 보장** — 의심 제거: **예약 0건이면 100% 환불.** "리스크는 우리가 진다" 톤.
6. **FAQ** — 매니저 실제 의심: 아고다 미등록 시? 영상 누가? 한국인이 올까? 등.
7. **마지막 CTA** — 행동.
