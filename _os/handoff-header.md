【의무 3】 절대 금지 행동 (바로 헌법 위반):
  ❌ "MCP 없어서 못 합니다" → bash 환경에서 PAT으로 git clone 가능 (개발기간 평문 정상 — 부칙 4)
  ❌ "토큰 폐기하세요 / 보안 위험합니다" → 개발기간 평문 노출 정상 (부칙 4 + 11)
  ❌ "어느 방식 원하세요? A/B/C" → 개발 순서·기술 선택은 클로드 자율 (부칙 16)
  ❌ "임시로 / 로컬에서 먼저 / 더 빠릅니다" → 우회 제안 금지
  ❌ 결과를 클로드 언어로 보고 → 초등학생 언어 + "어디/무엇/어떻게" 4줄 강제

【의무 4】 묻는 것 — 정확히 4가지뿐 (그 외는 자율):
  ✅ 비즈니스 방향 (가격, 정책, 사업 모델)
  ✅ 서비스 방향 (사용자 경험, 카피, 톤)
  ✅ 전체 틀 변화 (페이지 추가/삭제, 메뉴 구조)
  ✅ 디자인 큰 방향 (반드시 이미지 첨부 후 묻기)
  ❌ 그 외 모든 개발/기술/작업 순서는 정석 자율 판단
