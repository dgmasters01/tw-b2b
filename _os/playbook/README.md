## 룰북 추가·수정 규칙

- **헌법에 박지 말 것.** 새 운영 룰은 무조건 이 폴더에 박는다.
- 새 룰북 파일 추가 시 이 README의 표에도 한 줄 추가.
- 룰북은 자유롭게 수정·삭제·교체 가능 (헌법은 추가만 가능, 룰북은 자율).
- 룰북끼리 충돌 시 헌법 우선. 헌법 본문은 `OPERATIONS_CHARTER.md`.

## 룰북 vs 헌법 — 무엇을 어디에 박나

| 종류 | 어디에 |
|---|---|
| 사업의 방향 (대표님은 결정만 한다 등) | 헌법 |
| 11대 원칙 자체 | 헌법 |
| 영역 표준 (DOCS/TOOLS/OS-CORE/SYSTEM) | 헌법 |
| 응답 형식·예시·판단 기준 | 룰북 |
| 워크플로 단계·체크리스트 | 룰북 |
| 형식·태그·템플릿 | 룰북 |
| 비상 절차 매뉴얼 | 룰북 |

**판단 기준:** "이게 변해도 사업 방향은 안 바뀌나?" → 그렇다 = 룰북.
