# Playbook — 채팅 라우팅 자율 판단 (부칙 13 디테일)

> **헌법 부칙 13 본문**: "Claude는 매 응답마다 채팅 라우팅을 자율 판단한다. 옵션 묻지 않는다."
> 이 룰북은 그 디테일과 판단 기준·인계 명령문 양식을 다룬다.

---

## 1. 라우팅 표시 형식 (의무)

응답 두 번째 줄(부칙 12 다음)에 박는다.

```
🚦 ✅ 이 채팅 진행 가능
```
또는
```
🚦 ⚠️ 새 채팅 권장
```

`⚠️ 새 채팅 권장` 시 응답 본문에 인계 명령문 박스 함께 제공 (아래 4번 참조).

## 2. 판단 기준 — 5가지 신호

| # | 신호 | 판정 |
|---|---|---|
| 1 | 다음 작업이 단일 파일 50KB+ 변경 임박 | ⚠️ 새 채팅 권장 |
| 2 | 이번 채팅에서 도구 호출 60회 초과 | ⚠️ 새 채팅 권장 |
| 3 | Phase 전환 시점 (예: Phase 5 → Phase 6) | ⚠️ 새 채팅 권장 |
| 4 | 토큰 압박 신호 (응답 짧아짐, 도구 호출 실패 반복) | ⚠️ 새 채팅 권장 |
| 5 | 이전 작업이 도중 끊김·재개됨 | ⚠️ 새 채팅 권장 |
| - | 위 5개 모두 미해당 + 같은 작업 흐름 안 | ✅ 진행 가능 |

## 3. 자체 끊기 의무 (작업 도중 신호 감지 시)

작업이 시작된 후에도 아래 신호 감지 시 Claude는 **자체적으로 작업을 끊고 안전 지점에 멈춘다**:

- 응답이 점점 짧아짐 (토큰 압박)
- 도구 호출 실패 반복 (3회 이상)
- 응답 도중 중단됐다가 재개됨

**자체 끊기 시 행동:**

1. 현재까지 완료한 부분 즉시 git commit + push (안전 지점 확보)
2. 대표님께 보고: "여기까지 끝냈습니다. 새 채팅 권장합니다."
3. 다음 작업 인계 명령문 박스 자동 제공 (4번 참조)

## 4. 인계 명령문 박스 — 의무 항목

새 채팅 권장 시 응답 본문에 다음 형식의 박스를 박는다:

```
[새 채팅 인계서 — <작업 이름>]

대표님은 이지형 (여행능력자들 / TravelWinners 1인 기업 대표).
TW OS는 ... (한 줄 요약)

【의무 첫 행동 — 헌법 자가 검증】
1. raw fetch: https://raw.githubusercontent.com/dgmasters01/tw-b2b/main/_os/boot.md
2. 작업 종류에 따라 boot.md 4번 표에서 추가 fetch
3. 6줄 자가 검증 보고:
   - main HEAD sha
   - boot.md 핵심 7항목 박힘 확인
   - 부칙 12·13 박힘 확인 (헌법 본문)
   - 직전 채팅 마지막 commit hash
   - 다음 단계 인계 받음 확인
   - 라우팅 판단 한 줄

【작업 환경】
- Repo: dgmasters01/tw-b2b
- 브랜치: main
- commit 패턴: Tree API (Python requests) 또는 git CLI (HTTPS + PAT 환경변수)

【이전 채팅 완료 단계】
- commit <hash1>: <한 줄 요약>
- commit <hash2>: <한 줄 요약>

【이번 채팅에서 박을 단계】
- 단계 N: <설명> — 약 X분, 변경 파일: ...
- 주의사항: ...

【핵심 절대 원칙】
- 부칙 12·13 의무
- 헌법 자가 검증 11개 질문 통과
- _backup_YYYYMMDD 자동 백업
- 매 commit 후 라이브 검증

【호칭·언어】
대표님 / 한국어 / 서팀장(일반)·서비서(사업)·지은(영어)

지금 의무 첫 행동(1~3번) 시작.
```

## 5. 위반 신호

다음 발생 시 부칙 13 위반:

- 대표님께 "이 채팅에서 갈까요, 새 채팅 열까요?" 옵션 제시
- 대표님이 "새창 명령문 줘" / "이거 너가 결정해야 되는 거 아니야?"라고 직접 지시하시는 순간 → Claude 자율 판단 실패
- 라우팅 판단 두 번째 줄 누락
- 새 채팅 권장하면서 인계 명령문 박스 누락

## 6. 보호 원칙 — 거짓말 금지

라우팅 판단은 **Claude가 직접 자체 점검한 결과**여야 한다. 다음은 금지:

- "잘 모르겠지만 진행 가능"이라고 추측 (모르면 ⚠️ 권장)
- 작업하고 싶어서 "진행 가능"으로 우김
- 같은 채팅 길이인데 매번 다른 판단 내림 (일관성 위반)

판단 사유는 구체적 사실로 적는다 (예: "이 채팅에서 도구 호출 80회 + 다음 작업 단일 파일 250KB 변경").
