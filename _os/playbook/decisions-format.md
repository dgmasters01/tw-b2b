# Playbook — DECISIONS 형식 (D-### 박는 표준)

> **헌법 원칙 4 (전수 추적) 운영 디테일.**
> 모든 비즈니스·시스템 결정은 D-###로 박혀야 한다.

---

## 1. 두 파일의 역할

| 파일 | 형식 | 용도 |
|---|---|---|
| `DECISIONS.md` | 사람용 (스토리 형식) | 자연어 서술, 맥락 보존 |
| `DECISIONS_INDEX.md` | AI용 (인덱스 형식) | 구조화된 표·태그·고정 ID, Claude 즉시 검색 |

**한 쪽이 수정되면 다른 쪽 자동 동기화** (sync-bot 책임).

## 2. DECISIONS.md 박는 형식

```markdown
### 결정 D-### : <한 줄 요약>

**무엇을**: <결정 내용 구체적으로>

**왜**: <배경·이유. 이전 결정과 어떻게 다른지>

**누가**: 이지형 대표님 + Claude

**후속 작업**: <BL-XXX 또는 없음>
```

## 3. DECISIONS_INDEX.md 박는 형식

```markdown
| ID | 날짜 | 카테고리 | 한 줄 요약 | 영향 파일 |
|---|---|---|---|---|
| D-### | 2026-MM-DD | <카테고리> | <요약> | <파일1>, <파일2> |
```

## 4. ID 규칙

- 순번 증가, 절대 재사용 금지
- D-001, D-002, ... D-013, D-014, ... 식
- 결정 취소·변경 시: 원본 D-### 유지 + 새 D-###로 변경 결정 박음 + INDEX에서 원본 D-###는 `~취소선~` 표시

## 5. 카테고리 (INDEX용)

| 카테고리 | 예시 |
|---|---|
| 헌법 | 헌법 부칙 신설·수정 |
| 시스템 구조 | 영역 분리, 단일 진실 매핑 |
| 비즈니스 정책 | 가격, 보장, 메시징 |
| 디자인 | 디자인 시스템, UX/UI 결정 |
| 운영 | 워크플로, 룰북 추가 |
| 인프라 | 인증, 결제, 메일 |

## 6. 발견 경위 의무

각 결정에 **"발견 경위" 1줄 이상**을 박는다 — 왜 이 결정이 필요했는지 미래 Claude가 이해할 수 있게.

예: "2026-05-07 Phase 5 단계 4 후 대표님 통찰 — 옵션 제시는 헌법 1조 위반."
