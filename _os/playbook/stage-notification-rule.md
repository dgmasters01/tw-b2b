# 단계 진입 자동 알림 룰 (Stage Transition Auto-Notification)

> **헌법 보조 룰.**
> 대표님이 작업 단계 사이에서 헤매지 않도록, 클로드가 단계 전환 시 명확히 알린다.
> 트리거: 2026-05-17 대표님 요청 "시스템 완성도 해야 될 때 너가 이야기를 해줘. 그러면 내가 안 헷갈리잖아."

---

## 1. 모든 BL은 단계가 있다

대부분의 BL은 3~5단계로 구성:
- 1단계: 결정·정책 박제 (디자인 가이드)
- 2단계: 와이어프레임 (UI 그림)
- 3단계: 시스템 완성도 (실제 코드)
- 4단계: 검증·배포
- 5단계: 운영 안정화

각 단계는 별도 채팅에서 진행 (컨텍스트 한도 + 작업 집중도).

---

## 2. 클로드의 4가지 알림 의무

### 의무 1: 단계 진입 시작 알림 (채팅 첫 응답)

```
🎯 X단계 시작 — [단계 이름]

이번 채팅에서 할 일:
- [작업 1]
- [작업 2]
- [작업 N]

예상 소요: [시간]
끝나면: [다음 단계 안내]
```

### 의무 2: 단계 진행 중 점검 (매 응답 첫 줄)

```
🟢/🟡/🔴 채팅 잔량 [상태]
[X단계 진행 중] / 작업 [N/M]
```

### 의무 3: 단계 완료 알림 (단계 끝 시점)

```
═══════════════════════════════════════
✅ X단계 완료
═══════════════════════════════════════

이번 채팅 결과:
- [성과 1]
- [성과 2]

박힌 자산:
- [GitHub 경로 1]
- [GitHub 경로 2]

다음 단계: Y단계 ([단계 이름])

대표님 행동:
1. 이 채팅 닫기
2. 새 채팅 열기
3. 아래 인계서 그대로 복사:

[인계서 자동 출력 — 다음 클로드용]

→ 새 클로드가 모든 결정 자동 회수 후 Y단계 시작
═══════════════════════════════════════
```

### 의무 4: 다음 단계 인계서 자동 출력

인계서 표준 형식:
```
[BL ID] [단계 이름] 시작합니다. 헌법대로 부팅하세요.

## 1단계: 부팅
- _os/boot.md 읽고
- tasks.json에서 [BL ID] 확인
- 박힌 자산 자동 회수: [경로 N개]

## 2단계: 회수해야 할 결정
- [문서 1]: [요약]
- [문서 2]: [요약]
- ...

## 3단계: 이번 채팅 작업
[구체 작업 N개]

## 4단계: 헌법 원칙
- 매 응답 첫 줄 잔량 점검
- 위치/구조/배치 질문 금지
- 정석 방안만 제시
- 추측 박지 않기. 박힌 결정만 사용

[첫 작업 명시]
```

## 3. 단계 매핑 표준 (BL 유형별)

### UI/페이지 BL (대부분)
1. 카피·정책 박제
2. 와이어프레임
3. 시스템 완성도 (코드)
4. 라이브 검증
5. 운영 모드

### 데이터·인프라 BL
1. 스키마·정책 박제
2. SQL/API 구현
3. 라이브 검증
4. 운영 모드

### 사업 전략 BL
1. 시장·경쟁 조사
2. 메시지·카피 박제
3. 실행 자료 (페이지/메일/영상)
4. 운영 모드

각 BL 시작 시 단계 수를 명시한다. 단계가 끝날 때마다 위 4가지 의무 실행.

---

## 4. 사고 방지 — 클로드가 잊으면 안 되는 것

### ❌ 사고 케이스
- 단계 완료했는데 대표님께 알리지 않고 다음 작업으로 넘어감
- "이번 채팅에서 5탭 다 그리고 코드까지 박을게요" → 채팅 끊김 → 회수 사고
- 인계서 안 박고 채팅 끝남 → 다음 클로드가 회수 못 함

### ✅ 정석 방안
- 단계는 한 채팅에 1개만 진행
- 단계 완료 시점에 무조건 알림 + 인계서
- 다음 단계는 새 채팅에서 시작 (강제)
- 대표님 헤매면 클로드가 헷갈린 거 (대표님 책임 아님)

## 5. tasks.json에 단계 메타 박기

각 BL의 tasks.json에 다음 필드:

```json
{
  "id": "BL-XXX",
  "stages": [
    {"n": 1, "name": "카피·정책 박제", "status": "done", "completed_at": "2026-05-17"},
    {"n": 2, "name": "와이어프레임", "status": "in_progress"},
    {"n": 3, "name": "시스템 완성도", "status": "pending"},
    {"n": 4, "name": "라이브 검증", "status": "pending"},
    {"n": 5, "name": "운영 모드", "status": "pending"}
  ],
  "current_stage": 2,
  "next_stage_action": "와이어프레임 끝나면 시스템 완성도 (코드) 단계로"
}
```

→ 다음 클로드가 boot.md 읽으면 자동으로 어느 단계인지 회수.
