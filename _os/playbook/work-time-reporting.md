# Playbook — 작업 소요 선보고 (부칙 12 디테일)

> **헌법 부칙 12 본문**: "Claude는 매 응답 첫 줄에 작업 소요 시간 + 단계 수 + 변경 파일을 명시한다."
> 이 룰북은 그 디테일을 다룬다.

---

## 1. 형식 (의무)

```
[작업 소요: 약 X분 / N단계 / 변경 파일: A, B, C]
```

- **약 X분**: 분 단위. 1분 미만은 `30초` 같이.
- **N단계**: 응답에서 처리할 step 수 (조회만이면 1단계).
- **변경 파일**: 실제 수정·생성될 파일 목록. 없으면 `없음 (조회만)` 또는 `없음 (보고만)`.

## 2. 예시

| 작업 종류 | 첫 줄 예시 |
|---|---|
| 단순 조회·답변 | `[작업 소요: 30초 / 1단계 / 변경 파일: 없음 (조회만)]` |
| 시스템 설계 제안만 | `[작업 소요: 약 5분 / 1단계 / 변경 파일: 없음 (제안만)]` |
| 단일 파일 부분 수정 | `[작업 소요: 약 10분 / 3단계 / 변경 파일: admin-status.html]` |
| 다중 파일 + commit | `[작업 소요: 약 30분 / 5단계 / 변경 파일: api/admin-page.js, vercel.json, _admin/admin-business-charter.html]` |
| 큰 단계 작업 (단계 5 등) | `[작업 소요: 약 90분 / 7단계 (A~G) / 변경 파일: _os/boot.md, _os/playbook/ (8개), OPERATIONS_CHARTER.md, ...]` |

## 3. 예외 없음

- 단순 조회·검증·1단계 작업도 첫 줄 의무.
- "예 / 아니오" 같은 짧은 답도 첫 줄 박음.
- 단, **인계서 받은 후 첫 응답은 라우팅 판단(부칙 13)을 첫 줄로 박을 수 있음** — 그 다음 줄이 작업 소요.

## 4. 위반 신호

다음 중 하나 발생 시 부칙 12 위반:

- 첫 줄 없이 작업 시작
- 형식이 다른 (예: "작업 시간: ...", "예상 소요: ...")
- 변경 파일을 누락하거나 일부만 적음

위반 시 즉시 멈춤 → 첫 줄 박은 후 재개.

## 5. 발견 경위

2026-05-07 Phase 5 인계 시점, 대표님 통찰 — 작업 시작 전 규모 파악이 안 되면 채팅 라우팅 판단(부칙 13)도 한 박자 늦음. 첫 줄 의무로 박아 매 작업 시작 시점에 자동 체크.
