---

### 🔵 Tier 2: 1:1 문의 통한 변경 (settings 읽기 전용 + 문의 라우팅)

| 항목 | 이유 |
|---|---|
| 호텔명 | 영상 정보와 연결 |
| 주소 | 영상 정보와 연결 |
| 호텔 등급 | 영상 톤·소개 문구와 연결 |
| Agoda 링크 | 매출 추적 핵심 키 |
| 담당자 교체 | 결제 주체 변경 |
| 가입 이메일 | 계정 정체성 |

**프로세스**:
1. settings에서 해당 항목은 **읽기 전용**으로 표시
2. 옆에 `수정 신청` 버튼
3. 누르면 → **문의 탭에 자동 폼 생성** (현재 값 / 새 값 / 사유 / 증빙 필드)
4. 제출 → 본사 admin 변경 신청 큐에 쌓임
5. 대표님 admin에서 `승인` / `거절+사유` 클릭
6. 승인 → DB 자동 반영 + hotel_change_history 박힘 + 매니저 메일 통보
7. 거절 → 매니저에게 거절 사유 메일

**대표님 1인 운영 부담 최소화**:
- 표준 사유 드롭다운 ("오타 수정" / "리브랜딩" / "주소 정정" / "기타")
- 증빙 첨부 (명함, 사업자등록증 등)
- 1신청당 30초~1분 처리

settings 상단 안내:
```
🌱 초기 셋업 모드 (D-12 / 30일)
호텔 정보·주소·Agoda 링크 수정 신청을 자유롭게 보내실 수 있습니다.
영상 제작 시작 후에는 변경이 영상 재제작으로 이어질 수 있습니다.
```

대표님 입장: 가입 직후 변경 신청은 빠른 승인 (Agoda DB 가져온 기본값 정정용)

### 🚀 운영 모드
**조건**: 영상 노출 시작 후

settings 상단 안내:
```
🚀 운영 모드 (영상 노출 중)
호텔 정보 변경 시 본사 영상팀에 자동 통보됩니다.
주소·이름 변경이 영상 재제작으로 이어질 수 있어 신중하게 부탁드립니다.
```

대표님 입장: 신중 검토 (영상 영향 검토 후 승인)

### 호텔 카드 우측 `이력 보기` 버튼

매니저가 누르면 → 모달 → 시간 역순 표시:
```
2026-05-17  Ms. Priya Silva → 마헤시 퇴사 후 인수인계 (본사 승인)
2026-02-03  Mr. Mahesh Perera · 휴대전화 변경 (셀프)
2025-11-20  Mr. Mahesh Perera · 직책 변경 (셀프)
2025-08-10  Mr. Mahesh Perera 가입 (1차 계약 결제 시점)
```

**효과**:
- 본사: 어느 호텔이 담당자 자주 바뀌는지 안정성 평가
- 재계약 시점: 영업 무기
- 분쟁 시: 명확한 기록
- LTV 분석: 매니저 교체율 ↔ 영상 제작 우선순위
