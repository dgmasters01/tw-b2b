## 1. 원고 넣기 — 두 경로 [이미지 017]

화면 상단 두 블록:

### ① 직접 올리기
- `파일 선택 (docx)…` 입력 + [패키지 만들기] 버튼.

### ② 구글 드라이브 [연결됨 배지]
- `폴더: 유튜브 업로드 자동화`
- `마지막 확인 오늘 11:00 · 다음 확인 16:00` (하루 4번 = 06·11·16·21 KST)
- 카운트 카드 3개: **대기 N · 오늘 처리 N · 확인필요 N**(확인필요>0이면 갈색 강조).
- 원고에 아고다 파트너 링크(cid·hid, word/_rels 하이퍼링크)가 있으면 자동으로 읽음 → 화면에서 링크 넣는 단계 없음.

### 두 순서도(기획 다이어그램, 참고)
- **A. 직접 올리기(6단계)** [이미지 009]: docx 고르고 [패키지 만들기](파일명에 도시·성급·가격대) → 발행 패키지 자동 생성(제목·설명란·태그·파일명) → 아고다 링크 3개 넣기(링크 없을 때만·cid로 채널 확인) → [내가 맡기](찜하기·중복 방지) → 복사→유튜브 업로드(번호 순서대로) → 유튜브 주소→[발행 완료](이 순간이 노출 시작일).
- **B. 구글 드라이브(5단계)** [이미지 010]: 「대기」 폴더에 원고(docx) 넣기(채널별 폴더) → 서버가 자동으로 읽음(대표님 손 안 댐·cid로 채널 판별) → 「올리기」 화면에 카드로 뜸 → 「완료」/「확인필요」 자동 이동 → 이후 [내가 맡기]→복사→[발행 완료](A의 4~6단계와 동일).
  - ※ 이미지 010은 "30분마다"로 적혀 있으나, 상세 화면(017)·D-060은 **하루 4번(06·11·16·21)**. → 하루 4번이 확정.

## 2. 발행 카드 (원고 하나 펼침) [이미지 001·005·011]

### 헤더
- 제목 `{도시/지역} · {성급} · {가격대}` (예: `후쿠오카 하카타 · 3성급 · 10만원대`)
- 서브 `{채널명} · {올린이}@… 님이 맡음 · {날짜 시각}` + 우상단 ⋯ 메뉴
- 발행 후 상태 배지(예: [발행됨] 초록). 대기 상태면 [대기] 갈색. [이미지 006·008]

### 복사 4개 + 번호 (맡아야 열림)
`번호 순서대로 유튜브에 붙여넣으세요` 안내 아래:
1. **제목** — 예: `일본 후쿠오카 하카타 호텔 추천 TOP3, 10만원대…` [Copy]
2. **설명란** — `49줄 · 해시태그 9개 · 챕터 5줄 · 아고다 링크 3개 포함` (챕터·해시태그·링크는 설명란 안에 구조대로 포함) [Copy]
3. **태그** — `415자 · 유튜브 한도 500자` [Copy]
4. **영상 파일명** — 예: `후쿠오카호텔_후쿠오카호텔추천_하카타호텔…` [Copy]
- `해시태그·챕터 따로 보기 ▾` 접기 토글(펼치면 따로 확인).

### 이 영상에 넣은 호텔 (TOP1·2·3)
- 각 행: `TOP{n} {호텔 한글명} / {영문명} · hid {번호}` + 우측 `예약 {N}건` + `{예약 기간}`
  - 예: TOP1 호텔 라 포레스타(hid 746301) 예약 3건 2023-07~2023-11
- **아고다 이름 불일치 표시**: TOP3처럼 원고명과 아고다 링크명이 다르면 빨강 `아고다에는 {아고다명}` + 빨간 경고 배너 `⚠ TOP3 이름이 아고다 링크와 다릅니다. 원고를 확인해 주세요.`
- 노란 배너 `확인 {N}건 · 눌러서 펼치기 ▾` (사람이 확인할 점 목록).

### 발행 입력
- `올린 뒤 주소를 붙여넣으세요` + 입력칸(placeholder `youtube.com/watch?v=…`) + [발행 완료]
- 맡은 카드엔 [놓기](찜 해제) 버튼도.

### 발행 후 「주소 고치기」 [이미지 006]
- `현재 유튜브 주소` {링크} + [✎ 주소 고치기]
- `새 주소로 바꾸기` 입력 + [저장] · 안내 `ⓘ 중복 확인 후, 이 영상이 붙은 호텔 링크도 함께 바뀝니다.`
- `이 원고 변경 이력`: ✎ 주소 수정·{누가}({날짜}, 이전→새 주소) / ✓ 발행·{누가}({날짜}) — (D-058)

## 3. 올리기 목록 — 보기·필터·상태

### 보기 (전체/내작업/미지정) [이미지 011·012·013·014]
- **전체 보기** — 모두 봄, 여러 채널 섞여 나옴.
- **내 작업** — 내가 맡은 것만.
- **미지정** — 아직 아무도 안 맡은 것만.
- **확인필요 (4번째 탭)** — 문제 원고만. [확정: 4번째 보기 탭. 뒤쪽 보강(D-060 §2-A)이 최신. 원고넣기 아래 별도 섹션은 초기안.]

### 필터 (모두 드롭다운, 무한 확장·가로 나열 금지)
- [전체 채널 ▾] / [전체 담당자 ▾](관리자만) / [전체 언어 ▾] [이미지 013·014]

### 각 원고 행에 새기는 정보
- **채널명**(행 왼쪽 라벨) + **언어 뱃지**(한국어/영어/일본어) [이미지 014]
  - 언어 따라 채널명 표기 변형: 영어행 `Hotel Here`, 일본어행 `ホテル`. 호텔명도 언어 반영(`Namba`, `札幌`).
- **출처 아이콘**: ⏥ 자동(드라이브에서 온 원고) / ↥ 수동(화면에서 직접 올린 원고) [이미지 024]
- **올린 사람**: 자동=`드라이브`, 수동=`{○○} 올림` [이미지 025]
- **중복 딱지**: 노란 `중복` 배지 + `겹침` 사유(예: `↥ 수동 · sanjaya@… 올림 · 드라이브 자동본과 겹침`), 행 갈색 강조. [이미지 024·025]
- **상태(3종만)**: 대기 · **맡음** · 발행됨. [이미지 013]
  - 맡음 = `{누가} 맡음 · {경과}`(예: 2시간 전 / 3일째). 3일+면 `방치` 빨강 강조(행 갈색). [이미지 012·013·014·015]
  - **미지정 = `{등록시각/출처} · {경과}`** (예: `드라이브에서 옴 · 2일째 미지정` / `7/10 등록 · 3일째`). 맡음의 "맡은 지 N일째"와 동일 규칙 — ①언제 등록 ②며칠째. 오래되면 `방치` 강조·정렬 위로. [보강 2026-07-13 · 이미지에 없던 것]
  - ⚠ 이미지 012는 `작업 중`으로 표기됐으나, D-060·이미지 013은 **`맡음`이 확정**("작업 중"은 실시간 알 수 없어 폐기).

### 담당자 지정·재배정 (owner+admin 전용) [보강 2026-07-13 · 이미지에 없던 것]
- **미지정 원고**: `[담당자 지정 ▾]` — 사람 골라 배정.
- **이미 맡은 원고**: `[재배정 ▾]` — 뺏어서 다른 사람에게.
- **권한 = 최종관리자(owner) + 관리자(admin)**. 에디터·매니저는 [내가 맡기]로 본인만 집어감(남에게 배정 불가). 권한 없는 사람 화면엔 지정/재배정 버튼 자체가 안 뜸(요소 숨김).
- 지정되면 대상자 「내 작업」에 즉시 반영. 이메일·알림은 후속(BL 후속).

### 상태별 EN 표기 [이미지 015]
- Unassigned / Held by {누가} · {경과} = Taken / Idle(방치). UI만 영어, 채널명·사람 이름 유지.

## 5. 확인필요 (문제로 못 올라온 원고) [이미지 018·021·022]

- 헤더 `확인필요 ({N}) — 아직 손봐야 할 것` + 안내 `⏱ 다음 확인 16:00 · 고쳐서 「대기」에 넣으면 그때 처리되고 여기서 사라집니다`
- 각 항목: `{파일명}.docx` + 우측 채널명 + ⚠ 사유 + [지우기](=드라이브 휴지통 30일)
- **사유 카테고리**:
  - `파일명에 성급이 없습니다 (예: 3성급)` → 파일명 고쳐 「대기」 재투입 시 자동 재처리
  - `아고다 파트너 링크가 없습니다` → cid·hid 링크 넣고 다시
  - `링크 cid가 채널과 안 맞습니다 (호텔이야 링크인데 호텔이곳 폴더)` → 맞는 폴더 or 링크 수정
  - `이미 등록된 원고입니다 (수동으로 올린 듯) · 대기 폴더에서 지워주세요`(파란 강조·중복) [이미지 022]
- 고친 원고 처리되면 리스트에서 자동으로 빠짐. 넣었는지 확인은 드라이브 「대기」 폴더 직접 보기.

## 6. 중복 방지 (양방향) [이미지 023·024·025·026]

- **수동 업로드 중복** [이미지 023]: [패키지 만들기] 누르는 즉시 갈색 경고 `⚠ 이미 등록된 원고입니다` + `{도시·성급·가격대} ({채널})` + `· 드라이브 「대기」 폴더에도 같은 원고가 있습니다 → 중복 방지를 위해 거기서도 지워주세요` + [취소] [그래도 추가].
- **드라이브 유입 중복**: 배치가 "이미 등록됨" 감지 → 확인필요로(§5).
- 일부러 [그래도 추가]한 것은 목록에 `중복` 딱지로 남음(§3).
- **중복 삭제 = 대표님(owner) 전용** [이미지 026]:
  - 👑 대표님 접속: 중복 행에 빨간 [🗑 삭제] 버튼 등장.
  - 👤 직원 접속: 같은 행이 [삭제]·자물쇠 **없이** 여느 원고처럼 보임(중복 딱지만). 권한 판정으로 버튼 렌더 여부만 분기(요소 자체 숨김).

## 7. 어긋남 해소 — 확정 결과 (2026-07-13 대표님)

원칙: **뒤쪽 이미지 > 앞쪽 이미지**(뒤쪽이 계속 보완해 맞춘 최신). 이미지=근거, 스펙(기록)=최신 단일 기준.

1. **탭 구조** → 확정: `전체 보기 · 내 작업 · 미지정 (+ 확인필요)`. `원고 대기·발행 중·발행 이력`은 초기안 폐기. (발행 이력 KPI·기간필터는 후속 흡수 검토)
2. **확인필요 위치** → 확정: **4번째 보기 탭**(뒤쪽 보강 D-060 §2-A). 별도 섹션은 초기안.
3. **수익/수수료 노출** → 확정: **owner(대표님) 전용**. 에디터·매니저 화면엔 렌더 안 함(요소 자체 숨김, 권한 분기). 성과표 `수익 $2,840`은 대표님 접속 때만. 호텔 상세 "수수료 안 보임"은 에디터 기준 표현.
4. **드라이브 주기** → 확정: 하루 4번(06·11·16·21). "30분마다" 폐기.
5. **상태 라벨** → 확정: `맡음`. "작업 중" 폐기.
