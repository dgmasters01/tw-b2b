## 목적 (가장 중요)
호텔 메뉴 = **호텔 하나에 대한 전체를 모아 전략적으로 판단하는 대시보드.** 단순 재사용 관리가 아니라 "이 호텔을 콘텐츠에 어떻게(다시) 쓸지"를 노출·성과·취소·확정률 보고 결정. 취소 분석(예약이 언제 잡히고 단계별로 얼마나 살아남나)이 핵심.

## 1. 목록 [이미지 호텔001·002·003·006·007]

세 가지 표기 변형이 이미지에 존재(같은 목록, 정보 밀도만 다름):
- **성과형** [호텔001]: 검색(`호텔 이름으로 찾기`) + `N곳 노출` / 열 = 호텔(한글+영문) · 채널 · 노출(N회 TOP순위) · 예약(N건) · 확정률(색: 초록↑/빨강↓)
- **재사용형** [호텔002]: 검색(호텔명·hid) + [지금 쓸 수 있는 것만 ▾] / 각 행 = 👁 마지막 노출·영상 · 📅 재사용 가능일 · 🎫 예약 N건 → 우측 상태(`아직 못 씀`/`지금 가능`/`노출 없음`)
- **분류형** [호텔003·006·007]: 필터 [나라▾][도시▾][성급▾][유형▾] / 각 행 = `나라·도시` + `N성급` + `유형(호텔/리조트)` 뱃지 + 호텔명·hid → 상태 + `›`(누르면 상세)

**재사용 상태 3종**: 아직 못 씀(재사용일 전) / 지금 가능(6개월 지남) / 노출 없음(아직 안 쓴 호텔).
**유형**: 호텔 / **리조트**(예: 신나몬 벤토타 비치, 스리랑카). 리조트 뱃지 파란색.

## 재사용 금지 = 6개월 (확정)
- 마지막 노출일 + 6개월 = 재사용 가능일. [호텔002·006·007] 상대표기(예: `마지막 노출 3개월 전 · 재사용 가능 2026-10 (3개월 뒤)`)도 지원.

## 2. 상세 (호텔 클릭) — 위→아래 순서

### 기본 정보 [호텔004·010]
`나라·도시` + `N성급` + `유형` 뱃지 / 호텔명(한글) / `{영문명} · hid {번호} · {재사용상태}(재사용 {가능일})`

### ① 노출 이력 (여러 건) [호텔009·010]
한 호텔은 여러 채널·여러 번 노출. 각 노출 = **정확한 날짜(YYYY-MM-DD)** · 채널 · 영상 제목 · **노출 순위(TOP1/2/3)** + [영상] 링크. 목록으로 쌓임.
- 예: 2026-04-15 호텔이야 후쿠오카 하카타 호텔 TOP3 → TOP1 / 2025-11-20 호텔이곳 규슈 가성비 → TOP2 / 2025-08-03 여행능력자들 후쿠오카 3박4일 → TOP3

### ② 성과 요약 — 드릴다운 3단계 [호텔010·011·012·013]
- **지표 4개**(전체 통합): 조회수 · 클릭 · 예약 · 확정률. (조회수·클릭은 `연동 후`, 예약·확정률은 실데이터)
- **드릴다운(지표 클릭 → 채널별 → 다시 클릭 → 최종)**:
  - **예약** [호텔011]: 전체(9건) → 채널별(호텔이야 5·확정4 80% / 호텔이곳 3·2 67% / 여행능력자들 1·1 100%) → **개별 예약**(예약일·금액·투숙기간·박·인원·객실·국가·상태 = 매니저 형태. 예: 2026-05-02 · 08-02~08-05·3박·성인2·디럭스더블·한국 · 방문 $285)
  - **조회수** [호텔012·013]: 전체(45,000) → 채널별(호텔이야 30,000·호텔이곳 12,000·여행능력자들 3,000) → **영상별 조회수**(영상·게시일·조회수. 예: 후쿠오카 하카타 TOP3·2026-04-15 = 22,000)
  - **클릭** [호텔012]: 전체(320) → 채널별(210·85·25) → **영상별 클릭수**
  - 조회수·클릭은 영상별까지, **예약은 개별 예약까지(한 칸 더 깊음)**
- 선택된 지표/채널은 파란 강조. 각 단계 사이 `↓ 누름` 안내.

### ③ 채널별 예약 [호텔009]
채널마다 예약 건수·확정률(가로 막대바). `채널별 예약 · 총 N건` + [예약 세부 전체 보기 →]. 어느 채널이 이 호텔에 효과적인가.

### ④ 예약·방문·취소·노쇼 [호텔004·010]
전체 집계 4카드: 예약 · 방문(초록) · 취소(빨강) · **노쇼(안 왔지만 매출)** + 방문율(예: 예약9·방문7·취소2·노쇼0 → 방문율 78%) + [세부 전체 보기 →].
- 세부 = 매니저 제공 형태 [호텔005]: 투숙(기간·박) · 인원·객실 · 고객 국가 · 방문 완료일(체크아웃).

### ⑤ 취소·확정률 전략 (리드타임별) [호텔008·010]
- **확정률 정의 = 실투숙 기준**(확정=체크아웃완료, 취소·노쇼 둘 다 미확정).
- 헤더 카드 2개: `이 호텔 확정률 N% (예약 X→방문 Y)` / `취소 N · 먼 예약에서 발생`
- **리드타임별 확정 막대**: 이미지는 **3구간**(임박 0~7일 / 근접 8~30일 / 먼 31일+), 각 구간 확정률%(초록/빨강). 예: 임박 88% · 근접 70% · 먼 30%.
- **판단 힌트**: `확정률 낮은 호텔은 TOP3에 넣을 때 주의. (90일 전 예약은 30%만 확정, 임박 예약은 88% 확정)`

### ⑥ 재사용 [호텔010]
마지막 노출 + 6개월 → 재사용 가능일. (예: 마지막 노출 +6개월 → 2026-10부터 가능)

## 3. 수수료·권한
- **수수료 = 기본 비노출. 대표님(owner) 화면에서만.** 에디터·매니저 화면엔 예약금액만, 수수료 숨김. [호텔001 "예약 (수수료는 여기서 안 보입니다)" = 에디터 기준 표현]
- 성과표와 동일 원칙(§올리기 §8): 수익·수수료 = owner 전용, 요소 자체 숨김(렌더 분기).

## 4. 데이터·연결
- 나라·도시·성급·유형 = hotels
- 노출 이력 = publications ↔ hotels(hid), published_at
- 조회수 = D-059 수집 후 / 클릭 = 추적 링크(gohotel.win/h/…) 후
- 예약·방문·취소·노쇼 = bookings_agoda (booked_at·checkin_date·booking_status·is_cancelled·noshow_count). 매출인정 = 체크아웃완료(+노쇼 별도, 취소 제외)
- 리드타임 = checkin_date − booked_at
- **취소일 컬럼 없음** → 진짜 취소 타이밍(체크인 D-day) 분석은 아고다 파일 업로드 후 취소일 유무 확인 필요

## 5. 확인 필요 (이미지 vs D-062 어긋남 — 대표님 판단)

1. **리드타임 구간 수** — 이미지 = **3구간**(임박0~7 / 근접8~30 / 먼31+). D-062 ⑤-1 = **6구간**(0~3 / 4~7 / 8~14 / 15~30 / 31~60 / 61+). 최종은 3구간(단순·이미지 최신)인가, 6구간(정밀·문서)인가?
2. **전환 퍼널** — D-062 ②는 "조회→클릭→예약→확정 전환율 퍼널"을 명시하나 **이미지엔 퍼널 시각화가 안 보임**(KPI 카드+드릴다운만). 퍼널을 넣을지, 뺄지?
3. **TOP3 배지(라벨)** — D-062 ⑤-3은 확정률→자동 배지(안정≥85 / 보통70~84 / 주의50~69 / 경고<50). 이미지는 확정률 **색상만**(초록/빨강), 라벨 배지는 안 보임. 라벨 배지 추가할지?
4. **⑤ 실데이터 재정리** — D-062 지시: 대표님이 실제 아고다 예약파일 업로드하면 ⑤ 전체를 실데이터로 재정리, 취소일 컬럼 있으면 진짜 취소 타이밍으로 승격. (미래 작업 — 파일 준비되면)

> ①②③은 뒤쪽 이미지 원칙상 "이미지가 최신"으로 볼 수 있으나, D-062에 명시된 정밀 요소라 대표님 확인 후 확정.
