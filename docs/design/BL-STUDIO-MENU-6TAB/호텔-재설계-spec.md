## 목적 (2겹)
1. 스튜디오 호텔 = "이 호텔을 콘텐츠에 어떻게(다시) 쓸지" 전략 판단 대시보드. 취소·확정률·리드타임이 핵심 축.
2. 그 위 사업 목적 = 콘텐츠 보고 소비자가 예약하게 만드는 것.
→ 그래서 "재사용 가능 + 키워드 수요 + 전략 방향 + 성과(돈·전환)"를 묶어 "지금 이 호텔 넣을지" 판단.

## A. 상단 현황표 — 2축 (핵심: 시점과 작업을 분리)

### 축 1 · 재사용 시점 (자동으로 바뀜 — 마지막 노출 +6개월)
카드 4: 지금 쓸 수 있음 / 곧 풀림(30일) / 아직 못 씀 / 노출 안 함

### 축 2 · 작업 상태 (사람이 움직임 — 큐 넣기·발행 시 즉시)
카드 4: 안 건드림 / 큐에 넣음(누가·언제 표시) / 발행됨 / **놓친 것**(피크 지남·되살리기 가능, 빨간 점선)

### 좋은 호텔·커버리지 (보조)
평균 확정률 / 확정률 높음(≥85) / 낮음(<50) / 전체 / 나라·도시 / 호텔·리조트

**업데이트 기준**
- 재사용 시점 = 자동(6개월 경과 시 "지금 가능"으로)
- 작업 상태 = 행동 시(큐에 넣으면·발행하면 즉시)
- 놓친 것 = 피크 월 지나면 자동 만료 → "놓친 것"으로 이동 (판정 기준: 피크 월 경과. 대안=추천 후 N일 미실행 — 확정 필요)

핵심 원칙: **골라서 큐에 넣어도 "지금 쓸 수 있음"에 남음**(재사용은 여전히 가능). 대신 `큐에 넣음` 배지로 안 건드린 것과 구분. 사라지지 않음.

## B. 목록 + 필터
- 검색: 호텔명·hid (검색창은 필터 드롭다운 **뒤/오른쪽**)
- 필터 드롭다운: 나라 · 도시 · 성급 · **유형(호텔/리조트)**
- 각 행: 호텔(나라·도시·성급 뱃지, 리조트=파란) · 키워드 수요·포지션 · 전략 방향·큐 상태 · 확정률
- 정렬: 추천순(수요×전략) 기본 / 키워드 수요순 / 확정률순 / 나라·도시
- "지금 쓸 수 있음 N개" 카드 열면 → 이 목록(추천순)

## C. 행 펼침 (인라인 아코디언, 배경=회색 중립·왼쪽 회색선, 파란색 금지)
행 누르면 그 자리에서 펼침(다른 행 자동 접힘). 담긴 것 위→아래:

### ① 키워드 근거 (D-065 연계 · 조사일 표시)
- 검색량(월) · 3개월 추세(±%↑↓) · 우리 순위
- 검색량 흐름 + 예측 피크 그래프(파랑=실측, 주황=예측)
- 연관 키워드 N개 · 합계 검색량

### ② 전략 근거 (D-066 연계 · 4단 상세 — 비약 금지)
방향 배지 + 근거 4: 1)수요 2)타이밍(제작 6~8주, 지금 만들면 피크 노출) 3)경쟁(순위·여지) 4)호텔 적합. + 결론 한 줄.

### ③ 이 호텔 성과 (돈·전환·노출)
- 돈: 예약금액(모두) · **수수료(대표님만 — 점선+잠금, 에디터·매니저 화면엔 칸 자체 숨김·권한 분기)** · 확정률
- 전환 흐름 = **칸(카드)로**: 조회 › 클릭 › 예약 › 확정, 칸 사이 전환율(%). 확정 칸 초록. (조회·클릭=연동 후, 예약·확정·금액=실데이터)
- 노출(N회·TOP순위) · 리드타임 확정(가장 강한 구간)

### 행동 버튼 2개
- **이 방향으로 콘텐츠 만들기 ↗** = 실행 → 전략 큐(D-066)에 카드 생성. 근거 같이 붙음. 누른 뒤 그 행 = `✓ 큐에 넣음 (누가·날짜)` 배지, 사라지지 않음. 또 누르면 "이미 큐에 있음" 중복 경고(큐에서 보기/그래도 추가). (올리기 맡음·중복 방지 원리)
- **이 호텔 전체 상세 ›** = 더 보기 → **방식 A(같은 창 화면 전환 + `‹ 목록으로` 뒤로가기)**. 상세=D-062 대시보드(노출이력 전체·개별예약 전체·리드타임·채널별·재사용·수수료). (인라인 B·사이드 C 아님 — 모바일·가로넘침금지 D-060)

## D. 놓친 것 (지난 추천)
- 피크 지나 자동 만료된 호텔. **숨기지 않음 — 누구나 열람.**
- 클릭하면 → **C의 행 펼침을 그대로 재사용** + 맨 위 "왜 놓쳤나" 박스.
- **기존 데이터 전부 그대로 표시**(검색량·순위·과거 노출·예약·확정·금액·전환 — 비우지 않음). 놓친 이유·다음 피크만 추가.
- 버튼: **되살리기(= 이 방향으로 콘텐츠 만들기 → 전략 페이지로 이동)** · 다음 피크까지 보류 · 전체 상세.

## E. 라이브 반영 순서 (실데이터 되는 것부터)
1. 현황표 2축(재사용 시점=실데이터 / 작업 상태=큐·발행 연동 후) + 좋은 호텔·커버리지
2. 목록(재사용 상태·나라·도시·성급·유형 필터·검색 뒤로) — 유형/일부 지역은 마스터 보완 필요
3. 행 펼침 성과·전환(예약·확정·금액=실데이터 / 조회·클릭=연동 후)
4. 키워드·전략·큐·놓친것 연계 = 키워드(D-065)·전략(D-066) 메뉴 실데이터 붙은 뒤 채움 (지금은 "연동 후" 자리만)
5. 전체 상세(방식 A 화면 전환) = D-062 대시보드 연결
